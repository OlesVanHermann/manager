# PROMPT_TARGET.GENERAL.API.TXT
# NAV1=General > NAV2=API

## ROUTE
/general/api

## NAVIGATION STATE
```
NAV1: [General*] Bare Metal | Hosted Private | Public Cloud | Web Cloud | Network | IAM
NAV2: General | Mon compte | Facturation | Support | [API*] Carbon
NAV3: [General*] Token
NAV4: [Console*] (when NAV3=General) | [General*] (when NAV3=Token)
SIDECAR: YES (API categories when NAV3=General, Token list when NAV3=Token)
```

## DESCRIPTION
Console API OVHcloud et gestion des tokens d'authentification.
Permet de tester les endpoints API et de gérer les applications autorisées.

## LAYOUT
WITH SIDECAR (280px): content varies by NAV3 selection.

## NAV3 TABS (2)

### 1. General (default)
- API Console / Explorer
- SIDECAR: API categories list
- NAV4: Console

### 2. Token
- Token management
- SIDECAR: Token list
- NAV4: General (token detail)

## SIDECAR CONTENT

### When NAV3=General (API Categories)
- Title: "API"
- Categories list:
  - /me (Account)
  - /domain (Domains)
  - /hosting/web (Web Hosting)
  - /email (Email)
  - /vps (VPS)
  - /cloud (Public Cloud)
  - /dedicated (Dedicated Servers)
  - /ip (IP)
  - /order (Orders)
  - /billing (Billing)

### When NAV3=Token (Token List)
- Title: "Tokens"
- Search bar
- Button: [+ Créer un token]
- Token cards: name, app ID, status
- Filter: status dropdown

## NAV4 CONTENT

### NAV3=General > NAV4=Console
- Endpoint selector (path input)
- Method selector: GET | POST | PUT | DELETE
- Parameters form
- Execute button
- Response viewer (JSON)
- Request history

### NAV3=Token > NAV4=General
- Token detail card
- Name, Application ID
- Created date, Expiry date
- Status badge
- Permissions table (GET/POST/PUT/DELETE per route)
- Last API calls log
- Actions: [Modifier] [Regénérer] [Supprimer]

## COMPONENTS

### API Category Card (sidecar)
- Category name
- Endpoint count
- Expand to show endpoints

### Endpoint Tester
- Method badge (GET=green, POST=blue, PUT=orange, DELETE=red)
- Path input with autocomplete
- Parameters editor
- Headers editor (optional)
- Response viewer with syntax highlighting

### Token Card
- App name
- App ID (truncated)
- Status badge
- Created date
- Actions menu

### Permissions Matrix
- Routes as rows
- Methods as columns (GET/POST/PUT/DELETE)
- Checkmarks for allowed

## STATES
- Default: category selected, endpoints listed
- Testing: loading indicator during API call
- Response: success (green) or error (red)
- Token selected: detail view

## MODALS
- Create token
- Edit token permissions
- Regenerate token secret
- Delete token confirmation
