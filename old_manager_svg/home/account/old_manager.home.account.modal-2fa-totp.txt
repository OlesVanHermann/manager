# Cahier des Charges : home/account - Modal 2FA TOTP

## 1. VUE D'ENSEMBLE
- **Type** : Modal activation 2FA TOTP
- **Déclencheur** : Clic "Activer" sur carte Application Mobile
- **Objectif** : Configurer l'authentification TOTP

## 2. STRUCTURE MODAL
- Header avec titre
- Étape 1 : QR Code + clé manuelle
- Étape 2 : Vérification code
- Description optionnelle
- Footer actions

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur 700x620px
- **OdsQRCode** : Code QR (200x200px)
- **OdsInput** : Code, Description
- **OdsButton** : Copier, Annuler, Activer

## 4. ÉTAPE 1 : QR CODE
| Élément | Contenu |
|---------|---------|
| Titre | "Étape 1 : Scanner le QR code" |
| Description | "Scannez avec Google Authenticator, Authy, etc." |
| QR Code | 200x200px centré |

## 5. CLÉ MANUELLE
| Élément | Contenu |
|---------|---------|
| Label | "Ou entrez manuellement cette clé :" |
| Input (readonly) | 400x40px, monospace |
| Exemple | "JBSW Y3DP EHPK 3PXP" |
| Action | "Copier" (link) |

## 6. ÉTAPE 2 : VÉRIFICATION
| Élément | Contenu |
|---------|---------|
| Titre | "Étape 2 : Vérifier le code" |
| Label | "Code à 6 chiffres affiché dans l'application" |
| Input | 200x44px |
| Placeholder | "• • • • • •" |

## 7. DESCRIPTION
| Élément | Contenu |
|---------|---------|
| Label | "Description (facultatif)" |
| Input | 652x44px |
| Placeholder | "Ex: Mon téléphone personnel" |

## 8. ACTIONS
| Bouton | Style | Position |
|--------|-------|----------|
| Annuler | Secondary | Droite |
| Activer | Primary bleu | Extrême droite |

## 9. APIS & ENDPOINTS
- GET /me/auth/2fa/totp/setup - Obtenir QR/secret
- POST /me/auth/2fa/totp/validate - Valider code
