# Cahier des Charges : home/iam - Modal Ajouter Utilisateur

## 1. VUE D'ENSEMBLE
- **Type** : Modal création utilisateur
- **Déclencheur** : Clic "Ajouter un utilisateur"
- **Objectif** : Créer nouvel utilisateur local

## 2. STRUCTURE MODAL
- Header avec titre
- Info connexion
- Formulaire 5 champs
- Footer actions

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur 600x620px
- **OdsInput** : Champs texte
- **OdsSelect** : Groupe
- **OdsTextarea** : Description
- **OdsButton** : Annuler, Valider

## 4. HEADER
| Propriété | Valeur |
|-----------|--------|
| Titre | "Ajouter un utilisateur" |
| Bouton fermer | × (top right) |

## 5. INFO CONNEXION
| Ligne | Contenu |
|-------|---------|
| 1 | "Afin de se connecter à l'interface, l'utilisateur devra renseigner votre NIC" |
| 2 | "{NIC}-ovh et son nom d'utilisateur." |

## 6. FORMULAIRE
| Champ | Type | Requis | Placeholder |
|-------|------|--------|-------------|
| Identifiant | Input | Oui | "nouveau-utilisateur" |
| Email | Input email | Oui | "user@example.com" |
| Mot de passe | Input password | Oui | "••••••••" |
| Groupe | Select | Oui | "DEFAULT" |
| Description | Textarea | Non | "Description optionnelle..." |

## 7. PREVIEW IDENTIFIANT
Sous le champ identifiant :
"{NIC}-ovh/nouveau-utilisateur"

## 8. ACTIONS
| Bouton | Style | Label |
|--------|-------|-------|
| Annuler | Secondary | "Annuler" |
| Valider | Primary | "Valider" |

## 9. VALIDATION
| Règle | Message |
|-------|---------|
| Identifiant requis | "L'identifiant est obligatoire" |
| Email valide | "Format email invalide" |
| Mot de passe min 8 | "8 caractères minimum" |

## 10. APIS & ENDPOINTS
- POST /me/identity/user - Créer utilisateur
- GET /me/identity/group - Liste groupes
