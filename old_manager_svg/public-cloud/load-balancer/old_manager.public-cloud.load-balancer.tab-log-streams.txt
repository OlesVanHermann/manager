# Cahier des Charges : public-cloud/load-balancer - Onglet Log Streams

## 1. VUE D'ENSEMBLE
- **Page** : Flux de données logs
- **Type** : Page datagrid avec intégration LDP
- **Objectif** : Gérer les flux de logs HAProxy vers Logs Data Platform

## 2. STRUCTURE PAGE
- Header OVHcloud
- Sidebar (Logs actif)
- Lien retour
- Titre "Flux de données"
- Description avec lien documentation
- Sélecteur compte LDP
- Bouton ajouter
- Datagrid
- Pagination

## 3. COMPOSANTS UI
- **OdsLink** : Retour, documentation
- **OdsText** : Titre, description
- **OdsSelect** : Compte LDP
- **OdsButton** : Ajouter un flux
- **OdsDatagrid** : Liste des flux
- **OdsBadge** : Statut (Actif)
- **OdsPagination** : Navigation

## 4. LIEN RETOUR
"← Retour à la page précédente"

## 5. TITRE
"Flux de données"

## 6. DESCRIPTION
"En vous abonnant aux logs haproxy de votre Load Balancer, ils seront transférés dans Logs Data Platform et vous pourrez les centraliser avec vos autres logs."
- Lien : "En savoir plus ↗"

## 12. APIS & ENDPOINTS
- GET /dbaas/logs
  - Liste des comptes LDP
- GET /cloud/project/{projectId}/region/{regionName}/loadbalancing/loadbalancer/{loadBalancerId}/log/subscription
  - Liste des flux de logs
- POST /cloud/project/{projectId}/region/{regionName}/loadbalancing/loadbalancer/{loadBalancerId}/log/subscription
  - Créer un flux
