# Cahier des Charges : public-cloud/databases - Onglet Utilisateurs

## 1. VUE D'ENSEMBLE
- **Page** : Databases - Onglet Utilisateurs
- **Type** : Page onglet listing
- **Objectif** : Gérer les utilisateurs d'un service DB

## 2. STRUCTURE PAGE
- Header OVH
- Sidebar avec Databases actif
- Breadcrumb
- Header service avec badges
- Onglets (Utilisateurs actif)
- Bouton créer utilisateur
- Tableau des utilisateurs
- ActionMenu ouvert

## 3. COMPOSANTS UI
- **OdsBreadcrumb** : Navigation
- **OdsTabs** : Onglets avec Utilisateurs actif
- **OdsButton** : "+ Créer un utilisateur" (primaire)
- **OdsTable** : Liste des utilisateurs
- **OdsBadge** : Rôle (Superuser, Read/Write, Read Only)
- **OdsBadge** : Statut (Actif)
- **OdsActionMenu** : Actions par utilisateur

## 4. TITRE
"prod-postgresql"

## 6. TABLEAU UTILISATEURS
### Colonnes
| Colonne | Type | Description |
|---------|------|-------------|
| Nom d'utilisateur | Texte bold | Identifiant |
| Rôle | Badge | Niveau d'accès |
| Bases autorisées | Texte | Liste des bases |
| Créé le | Date | Date création |
| Statut | Badge pill | Actif/Inactif |
| Actions | Menu ⋮ | ActionMenu |

### Badges Rôle
| Rôle | Style |
|------|-------|
| Superuser | Badge bleu (#DBEAFE, #1E40AF) |
| Read/Write | Badge gris (#F3F4F6) |
| Read Only | Badge gris (#F3F4F6) |

### Données exemple
| Utilisateur | Rôle | Bases | Créé le | Statut |
|-------------|------|-------|---------|--------|
| admin | Superuser | Toutes | 15/12/2024 | Actif |
| app_user | Read/Write | production, staging | 18/12/2024 | Actif |
| readonly_user | Read Only | analytics | 20/12/2024 | Actif |

## 7. ACTION MENU (ouvert)
| Action | Type |
|--------|------|
| Modifier les droits | Standard |
| Réinitialiser mot de passe | Standard |
| Supprimer | Danger (rouge) |

## 8. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/database/service/{serviceId}/user
  - Liste des utilisateurs
- POST /cloud/project/{projectId}/database/service/{serviceId}/user
  - Créer un utilisateur
- PUT /cloud/project/{projectId}/database/service/{serviceId}/user/{userId}
  - Modifier un utilisateur
- DELETE /cloud/project/{projectId}/database/service/{serviceId}/user/{userId}
  - Supprimer un utilisateur
- POST /cloud/project/{projectId}/database/service/{serviceId}/user/{userId}/credentials/reset
  - Réinitialiser mot de passe
