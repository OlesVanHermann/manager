# Cahier des Charges : public-cloud/databases - Modal Ajouter intégration

## 1. VUE D'ENSEMBLE
- **Modal** : Ajouter une intégration
- **Type** : Modal formulaire
- **Objectif** : Créer une intégration entre deux services

## 2. STRUCTURE MODAL
- Overlay sombre
- Conteneur blanc 500x380px
- Header avec titre et description
- Formulaire avec 3 champs select
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur
- **OdsSelect** : Type d'intégration, Service source, Service destination
- **OdsButton** : Annuler, Ajouter

## 4. TITRE
"Ajouter une intégration"

## 5. DESCRIPTION
"Ajouter une intégration entre deux services."

## 6. CHAMPS DU FORMULAIRE
| Champ | Type | Placeholder | Requis |
|-------|------|-------------|--------|
| Type d'intégration | Select | Sélectionnez un type | Oui |
| Service source | Select | Sélectionnez un service | Oui |
| Service destination | Select | Sélectionnez un service | Oui |

### Types d'intégration
| Type | Description |
|------|-------------|
| opensearchLogs | Envoyer les logs vers OpenSearch |
| grafanaDatasource | Ajouter comme datasource Grafana |
| kafkaMirrormaker | Réplication Kafka |
| kafkaLogs | Envoyer logs Kafka |
| kafkaConnect | Connexion Kafka Connect |

## 7. BOUTONS
| Bouton | Style | Position | Action |
|--------|-------|----------|--------|
| Annuler | Outline | Gauche | Fermer modal |
| Ajouter | Primary | Droite | POST integration |

## 8. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/database/service
  - Liste des services disponibles
- GET /cloud/project/{projectId}/database/capabilities/integration
  - Liste des types d'intégration
- POST /cloud/project/{projectId}/database/service/{serviceId}/integration
  - Body: { type, sourceServiceId, destinationServiceId }
  - Créer une intégration
