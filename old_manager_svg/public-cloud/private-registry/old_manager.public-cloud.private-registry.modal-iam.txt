# Cahier des Charges : public-cloud/private-registry - Modal IAM

## 1. VUE D'ENSEMBLE
- **Modal** : Activer l'authentification via OVHcloud IAM
- **Type** : Modal confirmation avec warning
- **Objectif** : Activer SSO OVHcloud sur le registre

## 2. STRUCTURE MODAL
- Modal blanche (550x440px)
- Barre orange en haut (warning)
- Titre + bouton fermer
- Warning important
- Description explicative
- Lien documentation
- Champ saisie CONFIRM
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur
- **OdsMessage** : Warning (fond jaune)
- **OdsText** : Description
- **OdsLink** : "Plus d'informations ici"
- **OdsInput** : Champ confirmation
- **OdsButton** : Annuler (outline), Confirmer (warning/orange)

## 4. HEADER
- **Barre orange** : Indicateur warning
- **Titre** : "Activer l'authentification via OVHcloud IAM"
- **Bouton fermer** : ×

## 5. WARNING
Fond jaune avec icône ⚠ :
"Attention : l'activation de l'authentification OVHcloud IAM sur le registre entraînera la suppression de tous les utilisateurs configurés."

## 6. DESCRIPTION
"La fonctionnalité d'authentification via OVHcloud IAM permet à Harbor d'utiliser les identités gérées par OVHcloud IAM (Identity and Access Management) pour contrôler l'accès au registre.

Les utilisateurs peuvent ainsi se connecter à Harbor avec leurs identifiants SSO OVHcloud."

- **Lien** : "Plus d'informations ici"

## 7. CONFIRMATION
- **Texte** : "Entrez \"CONFIRM\" dans le champ ci-dessous pour confirmer"
- **Input** : Champ texte
- **Placeholder** : "CONFIRM"

## 8. BOUTONS
- Annuler (outline primary)
- Confirmer (warning/orange)

## 9. APIS & ENDPOINTS
- PUT /cloud/project/{projectId}/containerRegistry/{registryId}/iam
  - Body: { enabled: true }
  - Active l'authentification IAM
