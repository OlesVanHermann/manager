# Cahier des Charges : bare-metal/netapp - Créer volume

## 1. VUE D'ENSEMBLE
- **Page** : Création d'un volume
- **Type** : Modal formulaire
- **Objectif** : Créer un nouveau volume sur le service NetApp

## 2. STRUCTURE MODAL
- Header avec titre + bouton fermer
- Formulaire de création
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Modal principale
- **OdsInput** : Nom, description
- **OdsQuantity** : Taille
- **OdsSelect** : Protocole, snapshot policy
- **OdsMessage** : Info montage
- **OdsButton** : Annuler (outline), Créer (primary)

## 4. CHAMPS FORMULAIRE
| Champ | Type | Requis | Description |
|-------|------|--------|-------------|
| Nom | Input | Oui | Nom du volume |
| Description | Input | Non | Description optionnelle |
| Taille | Quantity | Oui | Taille en Go (min 100) |
| Protocole | Select | Oui | NFS v3 / NFS v4.1 |
| Snapshot Policy | Select | Non | Politique de rétention |

## 5. OPTIONS PROTOCOLE
| Option | Description |
|--------|-------------|
| NFS v3 | Network File System v3 |
| NFS v4.1 | Network File System v4.1 (recommandé) |

## 6. OPTIONS SNAPSHOT POLICY
| Option | Description |
|--------|-------------|
| Aucune | Pas de politique |
| default | Quotidien 7 / Hebdo 4 / Mensuel 6 |
| backup-critical | Quotidien 14 / Hebdo 8 / Mensuel 12 |

## 7. MESSAGE INFO
| Propriété | Valeur |
|-----------|--------|
| Type | Info (bleu) |
| Texte | Après création, les informations de montage NFS seront disponibles dans les détails du volume. |

## 8. BOUTONS
| Bouton | Style | Position |
|--------|-------|----------|
| Annuler | Outline | Gauche |
| Créer | Primary | Droite |

## 9. APIS & ENDPOINTS
- POST /storage/netapp/{serviceName}/share
  - Paramètres : name, description, size, protocol, snapshotPolicy
