# Cahier des Charges : bare-metal/dedicated-cloud - Modal User IAM Role

## 1. VUE D'ENSEMBLE

**Type:** Modal de création
**Fonction:** Créer un nouveau rôle IAM OVHcloud pour l'infrastructure
**Contexte:** Accessible depuis l'onglet Utilisateurs/IAM du Dedicated Cloud

## 2. DIMENSIONS

- Modal: 600x440px
- Border-radius: 8px
- Overlay: rgba(0,0,0,0.5)

## 3. COMPOSANTS UI

### Header Modal
- Background: #F9FAFB
- Titre: "Créer un nouveau rôle IAM OVHcloud"
- Close button: ×

### Description
- Texte: "Veuillez renseigner les champs suivants pour ajouter un rôle IAM OVHcloud à votre infrastructure."

### Formulaire

#### Champ Nom du rôle
- Label: "Nom du rôle *"
- Input: 520x40px
- Placeholder: "utilisateur"
- Helper text: "Le nom du rôle ne doit pas contenir de caractères spéciaux."

### Bloc info
- Background: #EFF6FF
- Border: 1px solid #BFDBFE
- Messages (3 lignes):
  - "La gestion des permissions de ce rôle dans vSphere sera"
  - "identique aux utilisateurs de votre infrastructure."
  - "La gestion des politiques et accès s'effectuera depuis l'IAM OVHcloud."

## 4. HEADER MODAL

- Hauteur: 60px
- Titre: "Créer un nouveau rôle IAM OVHcloud"

## 5. CONTENU

### Formulaire
| Champ | Type | Obligatoire | Validation |
|-------|------|-------------|------------|
| Nom du rôle | text | Oui | Alphanumérique, sans caractères spéciaux |

### Message informatif
```
La gestion des permissions de ce rôle dans vSphere sera
identique aux utilisateurs de votre infrastructure.
La gestion des politiques et accès s'effectuera depuis l'IAM OVHcloud.
```

## 6. FOOTER MODAL

- Séparateur: ligne 1px
- Bouton Annuler: btn-secondary (100x36px)
- Bouton Créer: btn-primary (100x36px)

## 7. APIS & ENDPOINTS

```
POST /dedicatedCloud/{serviceName}/iam/role
Body: {
  name: "utilisateur"
}
→ Crée un nouveau rôle IAM

GET /dedicatedCloud/{serviceName}/iam/role
→ Liste les rôles IAM existants
```

## 8. VALIDATION

- Nom obligatoire
- Pas de caractères spéciaux
- Unicité du nom (vérification côté API)
