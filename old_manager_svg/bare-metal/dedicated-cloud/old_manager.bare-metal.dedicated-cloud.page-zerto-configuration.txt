# Cahier des Charges : bare-metal/dedicated-cloud - Page Zerto Configuration

## 1. VUE D'ENSEMBLE

**Type:** Page Wizard (4 étapes)
**Fonction:** Configurer le plan de reprise d'activité (PRA) avec Zerto
**Contexte:** Accessible depuis l'onglet Zerto du Dedicated Cloud

## 2. DIMENSIONS

- Container principal: 1360x750px
- Cartes de choix: 600x200px

- Text: "Créez votre plan de reprise d'activité avec Zerto"
- Font-size: 20px, font-weight: 600

- Ligne 1: "Assurez la protection de votre système d'information en configurant votre plan de reprise d'activité."
- Ligne 2: "Basculez votre infrastructure en 1 clic et quelques minutes en cas de dysfonctionnement ou d'interruption de service."

## 5. ÉTAPE 1 : TYPE DE PRA

### Sous-titre
- Text: "Sélectionnez le type de plan de reprise d'activité"

### Carte 1 - OVH vers OVH (sélectionnée)
- Border: 2px solid #0050D7
- Header background: #EFF6FF
- Titre: "OVH vers OVH"
- Description: "Répliquez votre infrastructure entre deux Hosted Private Cloud OVHcloud. Nécessite un Private Cloud secondaire."
- Note: "Idéal pour une réplication complète dans l'environnement OVHcloud."
- Radio: checked

### Carte 2 - OVH vers On-Premises
- Border: 1px solid #E5E7EB
- Header background: #F9FAFB
- Titre: "OVH vers On-Premises"
- Description: "Répliquez votre infrastructure OVHcloud vers votre datacenter local. Nécessite une connexion VPN."
- Note: "Idéal pour une stratégie hybride cloud/on-premise."
- Radio: unchecked

## 6. FOOTER / NAVIGATION

- Bouton Annuler: btn-secondary (100x40px) - gauche
- Bouton Suivant: btn-primary (120x40px) - droite

## 7. APIS & ENDPOINTS

```
GET /dedicatedCloud/{serviceName}/zerto/configuration
→ Récupère la configuration actuelle

GET /dedicatedCloud/{serviceName}/datacenter
→ Liste les datacenters disponibles

POST /dedicatedCloud/{serviceName}/zerto
Body: {
  type: "ovh-to-ovh" | "ovh-to-onpremise",
  primaryDatacenter: "dc-paris-01",
  secondaryDatacenter: "dc-london-02"
}
→ Configure le PRA Zerto
```
