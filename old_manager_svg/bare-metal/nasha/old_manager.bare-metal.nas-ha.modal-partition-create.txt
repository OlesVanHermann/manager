# Cahier des Charges : bare-metal/nas-ha - Créer partition (complet)

## 1. VUE D'ENSEMBLE
- **Page** : Création de partition (formulaire complet)
- **Type** : Modal formulaire
- **Objectif** : Créer une nouvelle partition NAS-HA

## 2. STRUCTURE MODAL
- Header avec titre + bouton fermer
- Formulaire complet
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Modal principale
- **OdsInput** : Nom, description
- **OdsQuantity** : Taille
- **OdsSelect** : Unité (GB/TB), politique snapshots
- **OdsCheckbox** : Protocoles (NFS, CIFS, FTP)
- **OdsButton** : Annuler, Créer

## 4. CHAMPS FORMULAIRE
| Champ | Type | Requis | Description |
|-------|------|--------|-------------|
| Nom | Input | Oui | Nom de la partition |
| Taille | Quantity | Oui | Taille en Go/To |
| Unité | Select | Oui | GB / TB |
| Protocoles | Checkbox | Oui | NFS, CIFS/SMB, FTP/FTPS |
| Description | Textarea | Non | Description optionnelle |
| Politique snapshots | Select | Non | Quotidien, hebdo, etc. |

## 5. PROTOCOLES (Checkboxes)
| Option | Description |
|--------|-------------|
| NFS | Network File System |
| CIFS / SMB | Windows (Common Internet File System) |
| FTP / FTPS | File Transfer Protocol |

## 6. POLITIQUE SNAPSHOTS
| Option | Description |
|--------|-------------|
| Aucune | Pas de snapshot automatique |
| Quotidien (7j) | Rétention 7 jours |
| Hebdomadaire (4s) | Rétention 4 semaines |
| Mensuel (12m) | Rétention 12 mois |

## 7. VALIDATION
- Caractères alphanumériques et tirets uniquement
- Espace disponible affiché : 1.8 TB

## 8. BOUTONS
| Bouton | Style | Position |
|--------|-------|----------|
| Annuler | Outline | Gauche |
| Créer | Primary | Droite |

## 9. APIS & ENDPOINTS
- POST /dedicated/nasha/{serviceName}/partition
  - Paramètres : partitionName, size, protocol, description
