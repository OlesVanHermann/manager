# Cahier des Charges : bare-metal/nas-ha - Onglet Snapshots

## 1. VUE D'ENSEMBLE
- **Page** : Liste des snapshots NAS-HA
- **Type** : Onglet tableau
- **Objectif** : Gérer les snapshots de toutes les partitions

## 2. STRUCTURE PAGE
- Breadcrumb avec retour
- Titre "Snapshots"
- Onglets navigation
- Toolbar actions
- Tableau snapshots
- Bandeau planification

## 3. COMPOSANTS UI
- **OdsTabs** : Navigation
- **OdsButton** : Créer / Planifier
- **OdsDatagrid** : Tableau snapshots
- **OdsBadge** : Type (Automatique/Manuel)
- **OdsActionMenu** : Actions par ligne
- **OdsMessage** : Info planification

## 4. COLONNES DATAGRID
| Colonne | Type | Description |
|---------|------|-------------|
| Nom | Link | Identifiant snapshot |
| Partition | Text | Partition source |
| Date | Date | Date création |
| Taille | Text | Taille du snapshot |
| Type | Badge | Automatique (bleu) / Manuel (vert) |
| Actions | ActionMenu | Menu ⋮ |

## 5. ACTIONMENU (⋮)
| Action | Style | Description |
|--------|-------|-------------|
| Restaurer | Normal | Restaure le snapshot |
| Cloner vers partition | Normal | Clone les données |
| Monter en lecture | Normal | Monte en RO |
| Supprimer | Danger | Supprime le snapshot |

## 6. TOOLBAR
| Bouton | Style | Action |
|--------|-------|--------|
| Créer un snapshot | Primary | Ouvre modal création |
| Planifier des snapshots | Outline | Ouvre config planning |

## 7. BANDEAU PLANIFICATION
| Propriété | Valeur |
|-----------|--------|
| Background | Bleu clair |
| Titre | Planification active |
| Texte | Snapshots automatiques : Tous les jours à 03:00 - Rétention : 7 jours |
| Lien | Modifier la planification → |

## 8. APIS & ENDPOINTS
- GET /dedicated/nasha/{serviceName}/partition/{partitionName}/snapshot
  - Liste snapshots d'une partition
- POST /dedicated/nasha/{serviceName}/partition/{partitionName}/snapshot
  - Créer snapshot
- DELETE /dedicated/nasha/{serviceName}/partition/{partitionName}/snapshot/{name}
  - Supprimer snapshot
