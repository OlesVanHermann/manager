# Cahier des Charges : network/ip - Modal ARP

## 1. VUE D'ENSEMBLE

**Type:** Modal informatif
**Fonction:** Afficher le statut ARP et les logs de blocage
**Contexte:** Diagnostic ARP spoofing/blocage

## 2. DIMENSIONS

- Modal: 450x350px
- Overlay: fond noir 50%

## 3. LAYOUT & STRUCTURE

### Header modal
- Background: #000E9C (header OVHcloud)
- Titre: "ARP"
- Bouton fermer: × (blanc)

### Contenu
| Champ | Type | Valeur exemple |
|-------|------|----------------|
| Statut | Badge | "Bloquée" (danger) |
| Bloquée depuis | DateTime | 02/01/2025 14:22:45 |
| Temps restant (s) | Nombre | 7200 |

### Zone Logs
- Titre: "Logs :"
- Background: #1F2937 (dark)
- Font: monospace
- Couleur texte: #10B981 (vert terminal)

**Exemple logs:**
```
[2025-01-02 14:22:45] ARP spoofing detected
[2025-01-02 14:22:45] Invalid MAC address broadcast
[2025-01-02 14:22:46] ARP table poisoning attempt
[2025-01-02 14:22:46] Source blocked: 51.178.42.123
[2025-01-02 14:22:46] Auto-unblock in 2 hours
```

## 4. COMPOSANTS UI

### Badge statut
- "Bloquée": bg-danger #FEE2E2, text-danger #DC2626

### Zone logs
- Dimensions: 400x100px
- Border-radius: 4px
- Scroll si dépassement

## 5. ACTIONS & INTERACTIONS

### Bouton Débloquer l'IP
- Style: btn-secondary (130x36px)
- Action: Demande de déblocage ARP

### Bouton Fermer
- Style: btn-secondary (100x30px)
- Position: footer droite
- Action: Ferme le modal

## 6. APIS & ENDPOINTS

```
GET /ip/{ip}/arp/{ipBlocked}
→ Récupère le statut ARP

GET /ip/{ip}/arp/{ipBlocked}/log
→ Récupère les logs ARP

POST /ip/{ip}/arp/{ipBlocked}/unblock
→ Déblocage manuel
```
