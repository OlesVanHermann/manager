# Cahier des Charges : network/load-balancer - Frontends

## 1. VUE D'ENSEMBLE

**Type:** Page avec datagrid
**Fonction:** Gérer les frontends (points d'entrée) du Load Balancer
**Contexte:** Configuration des frontends HTTP/HTTPS/TCP

## 2. HEADER

- Titre: "Frontends" (22px, font-weight: 600)
- Sous-titre: "Configurez les points d'entrée de votre Load Balancer" (13px, #6B7280)

## 3. BARRE D'ACTIONS

### Bouton principal
- "+ Ajouter un frontend" (btn-outline, text-link #0050D7)
- Dimensions: 160x36px

## 4. DATAGRID

### Colonnes
| Colonne | Largeur | Description |
|---------|---------|-------------|
| Frontend | 220px | Nom et ID du frontend |
| Protocole | 170px | Type et port (ex: HTTPS : 443) |
| Datacenter | 180px | Zone de déploiement |
| Farm par défaut | 200px | Nom de la ferme associée |
| Actif | 100px | Toggle ON/OFF |
| ⋮ | 50px | Menu actions |

### Données affichées
- Frontend: Nom cliquable (link) avec ID entre parenthèses
- Protocole: HTTP, HTTPS, TCP avec numéro de port
- Datacenter: Nom avec code (Gravelines (GRA))
- Farm: Nom de la ferme de serveurs par défaut

### Toggle Actif
| État | Couleur |
|------|---------|
| ON | #0050D7 (cercle à droite) |
| OFF | #E5E7EB (cercle à gauche) |

## 5. MENU ACTIONS (ActionMenu)

- Voir les détails
- Modifier
- Supprimer (rouge #DC2626)

## 6. PAGINATION

- Format: "Affichage 1-X sur Y frontends"
- Position: Bas du tableau

## 7. APIS & ENDPOINTS

```
GET /ipLoadbalancing/{serviceName}/http/frontend
GET /ipLoadbalancing/{serviceName}/http/frontend/{frontendId}
POST /ipLoadbalancing/{serviceName}/http/frontend
PUT /ipLoadbalancing/{serviceName}/http/frontend/{frontendId}
DELETE /ipLoadbalancing/{serviceName}/http/frontend/{frontendId}
GET /ipLoadbalancing/{serviceName}/tcp/frontend
```
