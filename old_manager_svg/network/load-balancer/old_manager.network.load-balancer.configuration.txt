# Cahier des Charges : network/load-balancer - Configuration

## 1. VUE D'ENSEMBLE

**Type:** Page avec datagrid
**Fonction:** Appliquer les changements de configuration aux datacenters
**Contexte:** Gestion du déploiement des configurations Load Balancer

## 2. HEADER

- Titre: "Configuration" (22px, font-weight: 600)
- Sous-titre: "Appliquez les changements de configuration aux datacenters" (13px, #6B7280)

## 3. BARRE D'ACTIONS

### Bouton principal
- "Appliquer les changements" (btn-outline)
- Position: En haut du datagrid

## 4. DATAGRID

### Colonnes
| Colonne | Largeur | Description |
|---------|---------|-------------|
| ☐ | 40px | Checkbox de sélection |
| Datacenter | 300px | Nom du datacenter (ex: Gravelines (GRA)) |
| Progression | 300px | Barre de progression ou texte |
| Statut | 150px | Badge du nombre de changements |
| ⋮ | 50px | Menu actions |

### Données affichées
- Nom du datacenter avec code (GRA, RBX, SBG)
- Progression: Barre de progression avec pourcentage ou "Changements appliqués"
- Badge statut selon nombre de changements

### Badges statut
| État | Background | Text |
|------|------------|------|
| 0 changement | #DCFCE7 | #16A34A |
| N changements | #FEF3C7 | #92400E |

### Barre de progression
- Track: #E5E7EB (gris)
- Fill: #0050D7 (bleu) pour en cours
- Pourcentage affiché à droite

## 5. PAGINATION

- Format: "Affichage 1-X sur Y datacenters"
- Position: Bas du tableau

## 6. APIS & ENDPOINTS

```
GET /ipLoadbalancing/{serviceName}/zone
GET /ipLoadbalancing/{serviceName}/pendingChanges
POST /ipLoadbalancing/{serviceName}/refresh
GET /ipLoadbalancing/{serviceName}/task
```
