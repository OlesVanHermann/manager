# Cahier des Charges : web-cloud/email-exchange - Modal Politique d'envoi

## 1. VUE D'ENSEMBLE
- **Modal** : Configurer la politique d'envoi
- **Type** : Formulaire de paramètres
- **Objectif** : Définir les restrictions et limites d'envoi d'emails

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Bandeau explicatif
- Limites d'envoi
- Restrictions
- Boutons Annuler / Enregistrer

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (500x420)
- **OdsMessage** : Info bleu
- **OdsInput** : Limites numériques
- **OdsCheckbox** : Restrictions
- **OdsButton** : Annuler, Enregistrer

## 4. TITRE
"Politique d'envoi"

## 5. BANDEAU INFO
Fond bleu (#EFF6FF) :
"Configurez les restrictions d'envoi pour contrôler qui peut envoyer des emails depuis cette plateforme."

## 6. LIMITES D'ENVOI
| Champ | Type | Valeur | Description |
|-------|------|--------|-------------|
| Messages par heure | Input number | 1000 | Limite horaire |
| Destinataires par message | Input number | 500 | Max destinataires |

## 7. RESTRICTIONS
| Option | État | Description |
|--------|------|-------------|
| Bloquer l'envoi vers l'externe | ☑ | Interdit les envois hors organisation |
| Autoriser uniquement les domaines vérifiés | ☐ | Limite aux domaines configurés |
| Exiger l'authentification SMTP | ☐ | Force l'auth SMTP |
| Bloquer les pièces jointes dangereuses | ☑ | Filtre les fichiers à risque |

## 8. BOUTONS
| Bouton | Type | Action |
|--------|------|--------|
| Annuler | Secondaire | Ferme sans sauvegarder |
| Enregistrer | Primaire bleu | Applique la politique |

## 9. APIS & ENDPOINTS
- GET /email/exchange/{org}/service/{service}/sendPolicy : Récupérer politique
- PUT /email/exchange/{org}/service/{service}/sendPolicy : Modifier politique
