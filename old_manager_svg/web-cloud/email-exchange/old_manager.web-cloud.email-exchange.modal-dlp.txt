# Cahier des Charges : web-cloud/email-exchange - Modal Politique DLP

## 1. VUE D'ENSEMBLE
- **Modal** : Création d'une politique DLP (Data Loss Prevention)
- **Type** : Formulaire de création
- **Objectif** : Configurer la protection contre la fuite de données

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Bandeau info explicatif
- Formulaire avec nom et types de données
- Action à effectuer
- Boutons Annuler / Créer

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (550x460)
- **OdsMessage** : Bandeau info bleu
- **OdsInput** : Nom de la politique
- **OdsCheckbox** : Types de données à détecter
- **OdsSelect** : Action
- **OdsButton** : Annuler, Créer

## 4. TITRE
"Politique de prévention des pertes de données"

## 5. BANDEAU INFO
Fond bleu (#EFF6FF) :
"Les politiques DLP détectent et protègent les données sensibles (numéros de carte, données personnelles...)."

## 6. CHAMPS
| Champ | Type | Obligatoire |
|-------|------|-------------|
| Nom de la politique | Input text | Oui (*) |
| Types de données | Checkboxes | Au moins 1 |
| Action | Select dropdown | Oui |

## 7. TYPES DE DONNÉES À DÉTECTER
| Type | État | Description |
|------|------|-------------|
| Numéros de carte bancaire | ☑ | Visa, Mastercard, etc. |
| IBAN / Coordonnées bancaires | ☑ | Comptes bancaires |
| Numéros de sécurité sociale | ☐ | NIR français |
| Données médicales | ☐ | Informations de santé |

## 8. ACTIONS DISPONIBLES
| Action | Description |
|--------|-------------|
| Bloquer et notifier l'admin | Bloque l'email, alerte l'administrateur |
| Avertir l'expéditeur | Envoie un avertissement |
| Journaliser uniquement | Log sans blocage |
| Ajouter un en-tête | Marque l'email comme sensible |

## 9. BOUTONS
| Bouton | Type | Action |
|--------|------|--------|
| Annuler | Secondaire | Ferme la modal |
| Créer | Primaire bleu | Crée la politique DLP |

## 10. APIS & ENDPOINTS
- POST /email/exchange/{org}/service/{service}/dlp : Créer politique DLP
- GET /email/exchange/{org}/service/{service}/dlp : Lister politiques DLP
