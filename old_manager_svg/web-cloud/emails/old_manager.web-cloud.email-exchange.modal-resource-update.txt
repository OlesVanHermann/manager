# Cahier des Charges : web-cloud/email-exchange - Modal Modifier une ressource

## 1. VUE D'ENSEMBLE
- **Modal** : Modifier une ressource Exchange
- **Type** : Formulaire de modification
- **Objectif** : Mettre à jour les informations d'une salle/équipement

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Adresse email (readonly)
- Type de ressource (badge readonly)
- Nom de la ressource
- Capacité et emplacement
- Option acceptation automatique
- Boutons Annuler / Modifier

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (500x460)
- **OdsInput** : Email (readonly), Nom, Capacité, Emplacement
- **OdsBadge** : Type de ressource
- **OdsCheckbox** : Acceptation auto
- **OdsButton** : Annuler, Modifier

## 4. TITRE
"Modifier la ressource"

## 5. CHAMPS DU FORMULAIRE
| Champ | Type | État | Valeur exemple |
|-------|------|------|----------------|
| Adresse e-mail | Input readonly | Grisé | salle-everest@example.com |
| Type | Badge bleu | Readonly | Salle de réunion |
| Nom de la ressource * | Input text | Éditable | Salle Everest |
| Capacité | Input number | Éditable | 10 |
| Emplacement | Input text | Éditable | Bâtiment A, 2ème étage |

## 6. OPTION ACCEPTATION
☑ "Accepter automatiquement les réservations"

## 7. BOUTONS
| Bouton | Type | Action |
|--------|------|--------|
| Annuler | Secondaire | Ferme sans sauvegarder |
| Modifier | Primaire bleu | Sauvegarde les modifications |

## 8. APIS & ENDPOINTS
- GET /email/exchange/{org}/service/{service}/resourceAccount/{email} : Récupérer ressource
- PUT /email/exchange/{org}/service/{service}/resourceAccount/{email} : Modifier ressource
