# Cahier des Charges : web-cloud/email-exchange - Modal Diagnostic SRV

## 1. VUE D'ENSEMBLE
- **Modal** : Résultat diagnostic enregistrement SRV
- **Type** : Modal informative avec action
- **Objectif** : Afficher l'absence de SRV et proposer une correction

## 2. STRUCTURE MODAL
- Header jaune (warning)
- Domaine concerné
- Message d'avertissement
- Enregistrement à créer
- Boutons Fermer / Configurer auto

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (500x320)
- **OdsMessage** : Avertissement jaune
- **OdsInput** : Enregistrement SRV (readonly)
- **OdsButton** : Fermer, Configurer auto

## 4. TITRE
"Enregistrement SRV manquant"
- Header fond jaune (#FEF3C7)
- Texte orange (#92400E)

## 5. DOMAINE
Affiché en gras : "example.com"

## 6. MESSAGE D'AVERTISSEMENT
Fond jaune (#FEF3C7) avec icône :
| Élément | Valeur |
|---------|--------|
| Icône | ! (cercle jaune) |
| Titre | "Autodiscover non configuré" |
| Message | "L'enregistrement SRV permet la configuration automatique d'Outlook et des clients mail." |

## 7. ENREGISTREMENT À CRÉER
| Champ | Valeur exemple |
|-------|----------------|
| Label | "Enregistrement à créer" |
| Type | _autodiscover._tcp.example.com |
| Priorité/Poids/Port | 0 0 443 |
| Cible | ex1.mail.ovh.net |

## 8. BOUTONS
| Bouton | Type | Action |
|--------|------|--------|
| Fermer | Secondaire | Ferme la modal |
| Configurer auto | Primaire bleu | Crée automatiquement le SRV |

## 9. APIS & ENDPOINTS
- GET /email/exchange/{org}/service/{service}/domain/{domain}/dnsDiagnostic : Récupérer diagnostic
- POST /email/exchange/{org}/service/{service}/domain/{domain}/updateDns : Corriger DNS automatiquement
