# Cahier des Charges : web-cloud/zimbra - Modal suppression compte

## 1. RESUME EXECUTIF

### Perimetre
Modal de confirmation de suppression d'un compte Zimbra.

### Objectif fonctionnel
- Double confirmation de suppression
- Avertir de l'irreversibilite
- Supprimer le compte

---

## 2. LAYOUT

### Structure modale avec header danger + contenu + footer

---

## 3. COMPOSANTS UI

### Modal header
| Element | Description |
|---------|-------------|
| Icone | Warning rouge |
| Titre | Supprimer le compte |
| Bouton fermer | X |

### Contenu
| Element | Description |
|---------|-------------|
| Question | Etes-vous vraiment sur(e) de vouloir supprimer ce compte email {email} ? |

### Message warning
| Element | Description |
|---------|-------------|
| Type | Warning (jaune) |
| Texte | Attention cette action est irreversible et entraine la suppression des donnees |

### Footer
| Element | Description |
|---------|-------------|
| Bouton annuler | Secondary |
| Bouton supprimer | Danger (rouge) |

---

## 4. APIS & DONNEES

### Endpoints
```
DELETE /zimbra/platform/{platformId}/account/{accountId}
```

### Modele
```typescript
interface DeleteAccountResponse {
  taskId: string;
  status: 'pending' | 'done' | 'error';
}
```

## 6. REGLES METIER

- Double confirmation requise
- Suppression irreversible
- Toutes les donnees perdues (emails, contacts, calendrier)
- Slot libere apres suppression
