# Cahier des Charges : web-cloud/zimbra - Ajouter repondeur

## 1. RESUME EXECUTIF

### Perimetre
Formulaire d'ajout d'un repondeur automatique Zimbra.

### Objectif fonctionnel
- Configurer un message d'absence
- Definir la periode d'activation
- Optionnellement copier les messages

## 3. COMPOSANTS UI

### Navigation
| Element | Description |
|---------|-------------|
| Breadcrumb | Zimbra > Repondeurs > Ajouter un repondeur |
| Lien retour | Retour vers mes repondeurs |
| Titre | Ajouter un repondeur |

### Formulaire
| Champ | Type | Description |
|-------|------|-------------|
| Adresse liee * | Input + Select domain | Compte email concerne |
| Duree du repondeur * | Radio | Temporaire / Permanente |
| Du * | Date picker | Date debut |
| Jusqu'au * | Date picker | Date fin |
| Envoyer une copie | Checkbox | Option copie |
| Message * | Textarea | Contenu du repondeur |

### Note information
| Element | Description |
|---------|-------------|
| Texte | Veuillez noter que seuls les caracteres latins sont supportes |

### Bouton
| Element | Description |
|---------|-------------|
| Type | Primary |
| Texte | Confirmer |

## 4. APIS & DONNEES

### Endpoints
```
GET /zimbra/platform/{platformId}/account
GET /zimbra/platform/{platformId}/domain
POST /zimbra/platform/{platformId}/autoReply
```

### Modele
```typescript
interface CreateAutoReply {
  account: string;
  duration: 'temporary' | 'permanent';
  from?: string; // date ISO
  to?: string; // date ISO
  copyTo?: string;
  message: string;
}
```

## 6. REGLES METIER

- Temporaire = dates obligatoires
- Permanent = pas de date de fin
- Message en caracteres latins uniquement
- Copie vers adresse externe optionnelle
