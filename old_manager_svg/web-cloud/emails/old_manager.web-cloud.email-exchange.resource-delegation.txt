# Cahier des Charges : web-cloud/email-exchange - Délégation ressource

## 1. VUE D'ENSEMBLE
- **Modal** : Gérer les délégations d'une ressource
- **Type** : Modal liste éditable
- **Objectif** : Définir qui peut réserver et gérer une ressource

## 2. STRUCTURE MODAL
- Header avec email ressource et bouton fermer (×)
- Bandeau info
- Bouton Ajouter délégué
- Tableau des délégués
- Bouton Fermer

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (650x440)
- **OdsMessage** : Info bleu
- **OdsButton** : Ajouter délégué
- **OdsTable** : Liste des délégués
- **OdsText** : Permission
- **OdsButton** : Supprimer (×)

## 4. TITRE
"Délégation - {email_ressource}"
Exemple: "Délégation - salle-everest@example.com"

## 5. BANDEAU INFO
Fond bleu (#EFF6FF) :
"Les délégués peuvent réserver et gérer les rendez-vous de cette ressource."

## 6. BOUTON D'ACTION
| Bouton | Position | Action |
|--------|----------|--------|
| Ajouter délégué | En haut à droite | Ouvre modal d'ajout |

## 7. COLONNES DU TABLEAU
| Colonne | Description |
|---------|-------------|
| Compte | Email du délégué |
| Permission | Type d'accès |
| - | Bouton supprimer (×) |

## 8. TYPES DE PERMISSIONS
| Permission | Description |
|------------|-------------|
| Accès complet | Gestion totale |
| Réserver | Peut réserver uniquement |
| Envoyer en tant que | Peut envoyer au nom de la ressource |

## 9. EXEMPLES DE DONNÉES
| Compte | Permission |
|--------|------------|
| admin@example.com | Accès complet |
| assistant@example.com | Réserver |
| secretary@example.com | Envoyer en tant que |

## 10. BOUTON
| Bouton | Type | Action |
|--------|------|--------|
| Fermer | Secondaire | Ferme la modal |

## 11. APIS & ENDPOINTS
- GET /email/exchange/{org}/service/{service}/resourceAccount/{email}/delegate : Lister délégués
- POST /email/exchange/{org}/service/{service}/resourceAccount/{email}/delegate : Ajouter
- DELETE /email/exchange/{org}/service/{service}/resourceAccount/{email}/delegate/{id} : Supprimer
