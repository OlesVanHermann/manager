# Cahier des Charges : web-cloud/zimbra - Modal ajout redirection

## 1. RESUME EXECUTIF

### Perimetre
Modal d'ajout d'une redirection email Zimbra.

### Objectif fonctionnel
- Definir l'adresse source
- Definir l'adresse de destination
- Creer la redirection

---

## 2. LAYOUT

### Structure modale avec header + formulaire + footer

---

## 3. COMPOSANTS UI

### Modal header
| Element | Description |
|---------|-------------|
| Titre | Ajouter une redirection |
| Bouton fermer | X |

### Description
| Element | Description |
|---------|-------------|
| Texte | Merci de completer les informations pour la redirection |

### Formulaire
| Champ | Type | Description |
|-------|------|-------------|
| De l'adresse * | Input + @ + Select | Adresse source (domaine Zimbra) |
| Vers l'adresse * | Input | Adresse destination (externe) |

### Footer
| Element | Description |
|---------|-------------|
| Bouton annuler | Secondary |
| Bouton confirmer | Primary |

---

## 4. APIS & DONNEES

### Endpoints
```
GET /zimbra/platform/{platformId}/domain
POST /zimbra/platform/{platformId}/redirection
```

### Modele
```typescript
interface CreateRedirection {
  from: string;
  fromDomain: string;
  to: string;
}
```

## 6. REGLES METIER

- Source = domaine de la plateforme Zimbra
- Destination = adresse externe quelconque
- Pas de copie locale (redirection pure)
- Quota de 500 redirections par plateforme
