# Cahier des Charges : web-cloud/email-pro - Modal Filtre anti-spam

## 1. VUE D'ENSEMBLE
- **Modal** : Configuration du filtre anti-spam
- **Type** : Modal formulaire
- **Objectif** : Définir le niveau de filtrage et l'action sur les spams

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Compte (lecture seule)
- Niveau de filtrage (radio)
- Action sur les spams (select)
- Boutons Annuler / Enregistrer

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (500x390)
- **OdsInput** : Compte (désactivé)
- **OdsRadio** : Niveau de filtrage
- **OdsSelect** : Action sur les spams
- **OdsButton** : Annuler, Enregistrer

## 4. TITRE
"Filtre anti-spam"

## 5. COMPTE
Champ désactivé avec l'email du compte

## 6. NIVEAU DE FILTRAGE
| Option | État | Description |
|--------|------|-------------|
| ○ Désactivé | - | Aucun filtrage |
| ● Normal | Sélectionné | Recommandé |
| ○ Agressif | - | Filtrage renforcé |

## 7. ACTION SUR LES SPAMS
| Option | Description |
|--------|-------------|
| Déplacer vers le dossier Spam | Par défaut |
| Supprimer | Suppression directe |
| Marquer comme lu | Garder dans boîte |

## 8. BOUTONS
| Bouton | Type | Action |
|--------|------|--------|
| Annuler | Secondaire | Ferme la modal |
| Enregistrer | Primaire bleu | Sauvegarde la configuration |

## 9. APIS & ENDPOINTS
- GET /email/pro/{service}/account/{email}/spamFilter : Récupérer config
- PUT /email/pro/{service}/account/{email}/spamFilter : Configurer
  - level: "off" | "normal" | "aggressive"
  - action: "move" | "delete" | "mark"
