# Cahier des Charges : web-cloud/private-database - Modal Événements OOM

## 1. VUE D'ENSEMBLE
- **Modal** : Détails des événements Out of Memory
- **Type** : Modal informative avec tableau
- **Objectif** : Afficher l'historique des saturations mémoire (OOM)

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Description avec seuil
- Tableau des événements OOM
- Bandeau info recommandation
- Boutons "+ de RAM" / Fermer

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (700x550)
- **OdsText** : Description
- **OdsTable** : Événements OOM
- **OdsMessage** : Info bleu
- **OdsButton** : + de RAM, Fermer

## 4. TITRE
"Événements Out of Memory (OOM)"

## 5. DESCRIPTION
"Événements OOM sur les 7 derniers jours. Maximum 3 événements tolérés."

## 6. COLONNES DU TABLEAU
| Colonne | Description | Exemple |
|---------|-------------|---------|
| Date de l'événement | Timestamp complet | 22 déc. 2024 03:45:12 |
| Taille atteinte | RAM utilisée | 548 Mo |
| Uptime | Durée depuis démarrage | 4j 12h 30m |

## 7. EXEMPLES DONNÉES
| Date | Taille | Uptime |
|------|--------|--------|
| 22 déc. 2024 03:45:12 | 548 Mo | 4j 12h 30m |
| 20 déc. 2024 18:22:05 | 530 Mo | 2j 8h 15m |
| 18 déc. 2024 09:10:33 | 520 Mo | 1j 4h 45m |

## 8. BANDEAU INFO
Fond bleu (#EFF6FF) avec bordure (#3B82F6) :
- "ℹ Vous avez atteint le seuil critique d'événements OOM."
- "Nous vous recommandons d'augmenter la RAM de votre serveur."

## 9. BOUTONS
| Bouton | Style | Action |
|--------|-------|--------|
| + de RAM | Primaire bleu | Ouvre modal upgrade RAM |
| Fermer | Outline | Ferme la modal |

## 10. APIS & ENDPOINTS
- GET /hosting/privateDatabase/{serviceName}/oom
  - Retourne: events[] (date, memorySize, uptime)
