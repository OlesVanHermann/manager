# Cahier des Charges : web-cloud/hosting - Modal Delete FTP

## 1. VUE D'ENSEMBLE

**Type:** Modal confirmation danger
**Fonction:** Supprimer un utilisateur FTP secondaire
**Contexte:** Action depuis l'onglet FTP-SSH

## 2. DIMENSIONS

- Modal: 500x220px
- Overlay: rgba(0,0,0,0.5)

## 3. HEADER MODAL

- Background: #FEE2E2 (rouge pâle)
- Hauteur: 56px
- Titre: "Supprimer l'utilisateur FTP" (couleur #DC2626)
- Bouton fermer: × (position droite)

## 4. CONTENU

### Message principal
- Texte: "Vous êtes sur le point de supprimer l'utilisateur :"
- Nom utilisateur: "monsite-deploy" (font-weight 600)

### Bannière info (452x40px)
- Background: #DBEAFE (bleu info)
- Icône: i
- Texte: "Les fichiers de l'hébergement ne seront pas supprimés."
- Couleur texte: #1E40AF

## 5. FOOTER MODAL

- Bouton gauche: "Annuler" (btn-outline gris, 100x36px)
- Bouton droit: "Supprimer" (btn-danger #DC2626, 100x36px)

## 6. COMPORTEMENT

- L'utilisateur principal ne peut pas être supprimé
- Seuls les utilisateurs secondaires peuvent être supprimés
- Les fichiers restent intacts après suppression

## 7. APIS & ENDPOINTS

```
# Supprimer utilisateur FTP
DELETE /hosting/web/{serviceName}/user/{login}
→ Tâche de suppression

# Note: L'utilisateur principal n'est pas supprimable
# GET /hosting/web/{serviceName}/user/{login}
# → primaryLogin: true indique l'utilisateur principal
```
