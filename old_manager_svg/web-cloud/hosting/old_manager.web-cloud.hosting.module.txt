# Cahier des Charges : web-cloud/hosting - Modules 1 clic

## 1. RESUME EXECUTIF

### Perimetre
Page de gestion des modules CMS (WordPress, PrestaShop...).

### Objectif fonctionnel
- Lister les modules installés
- Installer de nouveaux modules
- Gérer les accès admin

### Structure titre + description + bouton + tableau + galerie

## 4. APIS & DONNEES

### Endpoints
```
GET /hosting/web/{serviceName}/module
POST /hosting/web/{serviceName}/module
DELETE /hosting/web/{serviceName}/module/{id}
POST /hosting/web/{serviceName}/module/{id}/changePassword
```

### Modele
```typescript
interface InstalledModule {
  id: number;
  moduleId: string;
  name: string;
  version: string;
  path: string;
  adminLogin: string;
  database: string;
  state: 'ok' | 'error';
  url: string;
  adminUrl: string;
}

interface AvailableModule {
  id: string;
  name: string;
  icon: string;
  color: string;
  latestVersion: string;
}
```

## 6. REGLES METIER

- Installation auto = BDD + fichiers + config
- Version la plus récente par défaut
- Chemin doit être vide ou inexistant
- Suppression = fichiers + BDD optionnel
