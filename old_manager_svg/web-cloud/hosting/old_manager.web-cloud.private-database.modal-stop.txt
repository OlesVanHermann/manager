# Cahier des Charges : web-cloud/private-database - Modal Arrêter le serveur (critique)

## 1. VUE D'ENSEMBLE
- **Modal** : Arrêt du serveur avec confirmation saisie
- **Type** : Modal danger avec input confirmation
- **Objectif** : Arrêter le serveur CloudDB avec double validation

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (✕)
- Icône stop (carré rouge) dans cercle rose
- Info service
- Bandeau erreur/danger
- Champ confirmation saisie nom service
- Boutons Annuler / Arrêter le serveur

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (500x520)
- **OdsIcon** : ◼ (stop square)
- **OdsCard** : Info service (fond gris)
- **OdsMessage** : Erreur rouge
- **OdsInput** : Confirmation saisie
- **OdsButton** : Annuler, Arrêter le serveur (danger)

## 4. TITRE
"Arrêter le serveur"

## 5. ICÔNE
- Cercle fond rose (#FEE2E2)
- Carré rouge (#DC2626) (icône stop)

## 6. INFO SERVICE
Fond gris (#F9FAFB) :
- Label: "Service"
- Valeur: "private-sql-abc123.database.cloud.ovh.net"

## 7. BANDEAU DANGER
Fond rouge (#FEE2E2) :
- "⚠️ Action critique"
- "L'arrêt du serveur rendra toutes vos bases de"
- "données inaccessibles jusqu'au redémarrage manuel."
- "La facturation continuera pendant l'arrêt."

## 8. CONFIRMATION SAISIE
- Label: "Pour confirmer, tapez le nom du service"
- Placeholder: "private-sql-abc123"
- Validation: Doit correspondre exactement au serviceName

## 9. BOUTONS
| Bouton | Style | État | Action |
|--------|-------|------|--------|
| Annuler | Outline | Toujours actif | Ferme la modal |
| Arrêter le serveur | Danger rouge | Actif si saisie correcte | Arrête le serveur |

## 10. APIS & ENDPOINTS
- POST /hosting/privateDatabase/{serviceName}/stop
  - Retourne: task object
