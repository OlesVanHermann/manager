# Cahier des Charges : web-cloud/hosting - Modal Runtime par défaut

## 1. VUE D'ENSEMBLE
- **Type** : Modal confirmation
- **URL** : N/A (composant modal)
- **Objectif** : Définir un runtime comme runtime par défaut pour Cloud Web

## 2. DIMENSIONS
| Propriété | Valeur |
|-----------|--------|
| Largeur | 500px |
| Hauteur | 360px |

## 3. COMPOSANTS UI
- **OdsModal** : Container
- **OdsText** : Question confirmation
- **OdsCard** : Détails runtime (fond gris)
- **OdsMessage** : Warning impact (jaune)
- **OdsButton** : Actions

## 4. CONTENU
| Élément | Description |
|---------|-------------|
| Titre | "Définir comme runtime par défaut" |
| Question | "Souhaitez-vous définir ce runtime comme runtime par défaut ?" |

## 5. DÉTAILS AFFICHÉS
| Champ | Exemple |
|-------|---------|
| Nom | Mon Runtime Node.js |
| Langage | nodejs22 |
| Répertoire public | public |
| Environnement | production |

## 6. AVERTISSEMENT
| Type | Message |
|------|---------|
| Warning | "⚠ Ce runtime sera utilisé par défaut pour tous les nouveaux multisites créés sur cet hébergement Cloud Web." |

## 7. BOUTONS
| Bouton | Style | Action |
|--------|-------|--------|
| Annuler | Secondary | Fermer modal |
| Confirmer | Primary | Définir comme défaut |

## 8. APIS & ENDPOINTS
- POST /hosting/web/{serviceName}/runtime/{id}/setDefault - Définir par défaut
