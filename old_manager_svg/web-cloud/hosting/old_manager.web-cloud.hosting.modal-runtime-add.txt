# Cahier des Charges : web-cloud/hosting - Modal Runtime Add

## 1. VUE D'ENSEMBLE

**Type:** Modal formulaire
**Fonction:** Ajouter un runtime PHP ou Node.js
**Contexte:** Action depuis l'onglet Runtimes

## 2. DIMENSIONS

- Modal: 600x480px
- Overlay: rgba(0,0,0,0.5)

## 3. HEADER MODAL

- Background: #F9FAFB
- Hauteur: 56px
- Titre: "Ajouter un runtime"
- Bouton fermer: × (position droite)

## 4. FORMULAIRE

### Champ Nom du runtime
- Label: "Nom du runtime"
- Input: 552x40px
- Placeholder: "Ex: php82-production"

### Type de runtime (2 cartes radio, 270x60px)

| Type | Description | État |
|------|-------------|------|
| PHP | Standard web hosting | Sélectionné (border primaire, radio coché) |
| Node.js | JavaScript runtime | - |

### Carte sélectionnée
- Border: #0050D7, 2px
- Radio: cercle plein #0050D7

### Champ Version
- Label: "Version"
- Select: 552x40px
- Valeur: "PHP 8.2 (recommandé)"
- Options selon type sélectionné

### Champ Chemin de l'application
- Label: "Chemin de l'application"
- Input: 552x40px
- Valeur: "/www"

## 5. BANNIÈRE INFO (552x40px)

- Background: #DBEAFE
- Texte: "Les runtimes permettent d'exécuter différentes versions de PHP ou Node.js sur votre hébergement."
- Couleur texte: #1E40AF

## 6. FOOTER MODAL

- Bouton gauche: "Annuler" (btn-outline, 100x36px)
- Bouton droit: "Créer" (btn-primary, 100x36px)

## 7. VERSIONS DISPONIBLES

### PHP
- PHP 7.4
- PHP 8.0
- PHP 8.1
- PHP 8.2 (recommandé)
- PHP 8.3

### Node.js
- Node.js 14 LTS
- Node.js 16 LTS
- Node.js 18 LTS (recommandé)
- Node.js 20 LTS

## 8. APIS & ENDPOINTS

```
# Lister runtimes existants
GET /hosting/web/{serviceName}/runtime
→ Liste des runtimes configurés

# Créer un runtime
POST /hosting/web/{serviceName}/runtime
Body: {
  name: "php82-production",
  type: "phpfpm" | "nodejs",
  version: "8.2",
  appBootstrap: null,
  appEnv: "production"
}
→ Tâche de création

# Versions PHP disponibles
GET /hosting/web/availablePhpVersions

# Versions Node.js disponibles
GET /hosting/web/availableNodeVersions
```
