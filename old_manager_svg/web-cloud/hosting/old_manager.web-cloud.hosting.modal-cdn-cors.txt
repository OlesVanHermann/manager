# Cahier des Charges : web-cloud/hosting - Modal Configuration CORS

## 1. VUE D'ENSEMBLE
- **Modal** : Configuration Cross-Origin Resource Sharing
- **Type** : Formulaire complet
- **Objectif** : Gérer les politiques CORS pour le CDN

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (✕)
- Bandeau info explicatif
- Formulaire avec plusieurs sections
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur avec overlay
- **OdsMessage** : Bandeau info bleu
- **OdsToggle** : Activer/Désactiver CORS
- **OdsInput** : Saisie origines, headers
- **OdsButton** : Ajouter origine
- **OdsTag** : Tags origines avec suppression (×)
- **OdsCheckbox** : Méthodes HTTP autorisées
- **OdsButton** : Annuler, Enregistrer

## 4. BANDEAU INFO
"ℹ️ CORS (Cross-Origin Resource Sharing) permet à votre site de partager des ressources avec d'autres domaines de manière sécurisée."

## 5. FORMULAIRE
### Activer CORS
- Toggle switch (on/off)
- État : Activé (bleu)

### Origines autorisées
- Input : https://example.com
- Bouton : Ajouter
- Tags existants :
  - https://api.example.com (×)
  - https://cdn.example.com (×)
  - * (tous) (×)

### Méthodes HTTP autorisées
| Méthode | État |
|---------|------|
| GET | ✓ Coché |
| POST | ✓ Coché |
| PUT | ✓ Coché |
| DELETE | Non coché |
| PATCH | Non coché |
| OPTIONS | ✓ Coché |

### En-têtes autorisés
- Input : "Content-Type, Authorization, X-Requested-With"

### En-têtes exposés
- Input : "Ex: X-Custom-Header"

### Durée de cache (max-age)
- Input : 3600
- Label : "secondes (1 heure)"
- Description : Durée pendant laquelle le navigateur peut mettre en cache la réponse preflight

## 6. BOUTONS FOOTER
- Annuler (secondaire)
- Enregistrer (primaire)

## 7. APIS & ENDPOINTS
- GET /hosting/web/{serviceName}/cdn/{domain}/cors : Config CORS actuelle
- PUT /hosting/web/{serviceName}/cdn/{domain}/cors : Mettre à jour CORS
