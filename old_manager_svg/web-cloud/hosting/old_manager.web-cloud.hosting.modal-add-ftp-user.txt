# Cahier des Charges : web-cloud/hosting - Modal Créer utilisateur FTP

## 1. VUE D'ENSEMBLE
- **Modal** : Création d'un nouvel utilisateur FTP
- **Type** : Formulaire simple
- **Objectif** : Ajouter un accès FTP/SSH pour un utilisateur

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Zone formulaire
- Bandeau info (serveur FTP)
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur avec overlay sombre
- **OdsInput** : Nom d'utilisateur avec préfixe, mots de passe, répertoire
- **OdsCheckbox** : Activer l'accès SSH
- **OdsMessage** : Bandeau info (adresse serveur FTP)
- **OdsButton** : Annuler, Créer

## 4. FORMULAIRE
| Champ | Type | Obligatoire | Format |
|-------|------|-------------|--------|
| Nom d'utilisateur | Préfixe + Input | Oui | hosting123- + utilisateur |
| Mot de passe | Password avec œil | Oui | 8-30 caractères |
| Confirmer MDP | Password | Oui | - |
| Répertoire home | Input | Oui | /www |
| Accès SSH | Checkbox | Non | Coché par défaut |

## 5. PRÉFIXE
- Format : `{serviceName}-`
- Exemple : `hosting123-`
- Affiché dans zone grisée non-éditable

## 6. RÈGLES DE VALIDATION
### Nom d'utilisateur
- Entre 1 et 32 caractères

### Mot de passe
- 8 à 30 caractères
- Au moins une majuscule
- Au moins une minuscule
- Au moins un chiffre

### Répertoire home
- L'utilisateur sera limité à ce répertoire et ses sous-répertoires

## 7. SECTION ACCÈS SSH
- Checkbox : "Activer l'accès SSH"
- Description : "Permet à l'utilisateur de se connecter via SSH en plus du FTP."

## 8. BANDEAU INFO
- "ℹ️ Le nouvel utilisateur pourra se connecter avec l'adresse : ftp.cluster042.hosting.ovh.net"

## 9. BOUTONS FOOTER
- Annuler (secondaire)
- Créer (primaire)

## 10. APIS & ENDPOINTS
- POST /hosting/web/{serviceName}/user : Créer utilisateur FTP
- GET /hosting/web/{serviceName}/userLogsToken : Récupérer token accès logs
