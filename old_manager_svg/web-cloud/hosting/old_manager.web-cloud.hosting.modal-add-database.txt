# Cahier des Charges : web-cloud/hosting - Modal Créer base de données

## 1. VUE D'ENSEMBLE
- **Modal** : Création d'une nouvelle base de données
- **Type** : Formulaire simple
- **Objectif** : Configurer une nouvelle BDD MySQL ou PostgreSQL

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Zone formulaire
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur avec overlay sombre
- **OdsRadio** : Sélection type (MySQL/PostgreSQL)
- **OdsSelect** : Version
- **OdsInput** : Nom BDD avec préfixe, utilisateur avec préfixe, mot de passe
- **OdsButton** : Annuler, Créer

## 4. FORMULAIRE
### Type de base de données
| Option | Description | État |
|--------|-------------|------|
| MySQL | Base de données relationnelle la plus populaire | Sélectionné |
| PostgreSQL | Base de données relationnelle avancée | Non sélectionné |

### Champs
| Champ | Type | Obligatoire | Format |
|-------|------|-------------|--------|
| Version | Select | Oui | MySQL 8.0, 5.7... |
| Nom de la base | Préfixe + Input | Oui | hosting123_ + nom_base |
| Nom d'utilisateur | Préfixe + Input | Oui | hosting123_ + utilisateur |
| Mot de passe | Password | Oui | ••••••••• avec icône œil |
| Confirmer MDP | Password | Oui | ••••••••• |

## 5. PRÉFIXES
- Format : `{serviceName}_`
- Exemple : `hosting123_`
- Affiché dans une zone grisée non-éditable

## 6. RÈGLES DE VALIDATION
### Nom de la base
- Entre 1 et 200 caractères alphanumériques

### Mot de passe
- 8 à 30 caractères
- Au moins une majuscule
- Au moins une minuscule
- Au moins un chiffre

## 7. BOUTONS FOOTER
- Annuler (secondaire, bordure)
- Créer (primaire bleu)

## 8. APIS & ENDPOINTS
- POST /hosting/web/{serviceName}/database : Créer base de données
- GET /hosting/web/{serviceName}/databaseAvailableVersion : Versions disponibles
