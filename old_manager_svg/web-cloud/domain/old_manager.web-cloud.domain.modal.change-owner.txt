# Cahier des Charges : web-cloud/domain - Modal Change Owner

## 1. VUE D'ENSEMBLE

**Type:** Modal procédure officielle
**Fonction:** Initier un changement de propriétaire de domaine
**Contexte:** Action depuis la page contacts ou informations générales

## 2. DIMENSIONS

- Modal: 700x600px
- Position: centrée avec overlay

## 3. STRUCTURE MODAL

### Header
- Background: #F9FAFB
- Titre: "Changer de propriétaire - exemple.com"
- Bouton fermer: × (coin droit)

### Content
- Padding: 24px

### Footer
- Séparateur: ligne #E5E7EB

## 4. CONTENU

### Warning banner
- Background: #FEF3C7
- Dimensions: 652x55px
- Titre: "Procédure officielle de changement de propriétaire"
- Description: "Cette procédure est soumise aux règles du registre. Des justificatifs peuvent être requis."

### Propriétaire actuel
- Label: "Propriétaire actuel"
- Box info: 652x45px, background #F9FAFB
- Contenu: NIC handle, Nom complet, Email

### Nouveau propriétaire - Type
- Label: "Nouveau propriétaire"
- Type cards (3 colonnes):

- Label: "NIC Handle du nouveau propriétaire"

## 7. APIS & ENDPOINTS

```
GET /domain/{serviceName}/owner
→ Propriétaire actuel

POST /domain/{serviceName}/trade
Body: {
  ownerType: "individual" | "corporation" | "association",
  newOwner: "xy98765-ovh"
}
→ Initie la procédure de changement

GET /domain/{serviceName}/trade
→ Statut de la procédure en cours

POST /domain/{serviceName}/trade/{tradeId}/document
Body: FormData (fichier)
→ Upload un justificatif

# La procédure nécessite:
# - Validation email ancien propriétaire
# - Validation email nouveau propriétaire
# - Éventuellement documents selon TLD
```
