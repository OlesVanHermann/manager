# Cahier des Charges : web-cloud/domain - Modal Ajouter enregistrement DYNHOST

## 1. VUE D'ENSEMBLE
- **Modal** : Création d'un enregistrement DNS de type DynHost
- **Type** : Formulaire de création
- **Objectif** : Ajouter un enregistrement DynHost pour IP dynamique

## 2. STRUCTURE MODAL
- Header avec titre, badge DYNHOST et bouton fermer (✕)
- Indication champs obligatoires
- Bandeau info explicatif
- Champ sous-domaine
- Champ IP actuelle
- Champ TTL
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (700x480)
- **OdsMessage** : Bandeau info bleu
- **OdsInput** : Sous-domaine, IP, TTL
- **OdsBadge** : Type DYNHOST (jaune)
- **OdsButton** : Annuler, Ajouter

## 4. TITRE
"Ajouter une entrée DNS" avec badge [DYNHOST]

## 5. INDICATION OBLIGATOIRE
"* Champs obligatoires"

## 6. BANDEAU INFO
Fond bleu (#E8F0FE), bordure bleue :
- "ℹ️ DynHOST permet de pointer un sous-domaine vers une IP dynamique."
- "L'IP sera mise à jour automatiquement à chaque changement."

## 7. CHAMP SOUS-DOMAINE
| Élément | Valeur |
|---------|--------|
| Label | "Sous-domaine" |
| Obligatoire | Oui (*) |
| Input | "home" (placeholder) |
| Suffixe (addon gris) | ".example.com" |

## 8. CHAMP IP ACTUELLE
| Élément | Valeur |
|---------|--------|
| Label | "Adresse IP actuelle" |
| Obligatoire | Oui (*) |
| Input | "192.168.1.1" (placeholder) |
| Aide | "Cette IP sera mise à jour dynamiquement par la suite" |

## 9. CHAMP TTL
| Élément | Valeur |
|---------|--------|
| Label | "TTL" |
| Input | 60 |
| Unité | "secondes" |

## 10. BOUTONS FOOTER
- Annuler (secondaire)
- Ajouter (primaire)

## 11. APIS & ENDPOINTS
- POST /domain/zone/{zoneName}/dynHost/record : Créer enregistrement DynHost
  - subDomain: string
  - ip: IPv4
  - ttl: number (court, ex: 60s)
