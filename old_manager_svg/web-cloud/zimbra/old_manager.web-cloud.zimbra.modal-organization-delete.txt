# Cahier des Charges : web-cloud/zimbra - Modal suppression organisation

## 1. RESUME EXECUTIF

### Perimetre
Modal de confirmation de suppression d'une organisation Zimbra.

### Objectif fonctionnel
- Confirmer la suppression de l'organisation
- Verifier qu'aucun domaine n'est associe
- Bloquer si domaines existants

---

## 2. LAYOUT

### Structure modale avec header danger + contenu + footer

---

## 3. COMPOSANTS UI

### Modal header
| Element | Description |
|---------|-------------|
| Icone | Warning rouge |
| Titre | Supprimer l'organisation |
| Bouton fermer | X |

### Contenu
| Element | Description |
|---------|-------------|
| Question | Etes-vous sur(e) de vouloir supprimer l'organisation {name} ? |

### Message erreur (si domaines associes)
| Element | Description |
|---------|-------------|
| Type | Danger (rouge) |
| Titre | Au moins un domaine est encore associe a cette organisation |
| Texte | Veuillez supprimer les domaines associes avant de supprimer votre organisation |

### Footer
| Element | Description |
|---------|-------------|
| Bouton annuler | Secondary |
| Bouton supprimer | Danger (disabled si domaines) |

---

## 4. APIS & DONNEES

### Endpoints
```
GET /zimbra/platform/{platformId}/organization/{orgId}/domain
DELETE /zimbra/platform/{platformId}/organization/{orgId}
```

### Modele
```typescript
interface OrganizationDeleteCheck {
  canDelete: boolean;
  associatedDomains: number;
  blockingReason?: string;
}
```

## 6. REGLES METIER

- Impossible de supprimer si domaines associes
- Cascade: organisation → domaines → comptes
- Bouton disabled si blocage
- Derniere organisation non supprimable
