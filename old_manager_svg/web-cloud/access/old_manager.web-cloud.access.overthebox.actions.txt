# Cahier des Charges : web-cloud/access/overthebox - Actions

## 1. VUE D'ENSEMBLE
- **Écran** : Onglet Actions d'une OverTheBox
- **Objectif** : Afficher l'historique des actions exécutées sur l'appareil
- **Contexte** : Suivi des opérations (reboot, update, reset...)

## 2. LAYOUT & STRUCTURE
- Carte blanche avec titre + bouton refresh
- Tableau des actions avec pagination

## 3. COMPOSANTS UI
- **OdsCard** : Container principal
- **OdsButton** : Refresh (icône ↻)
- **OdsTable** : Liste des actions
- **OdsBadge** : Statut action
- **OdsPagination** : Navigation pages

## 4. DONNÉES & COLONNES
| Colonne | Type | Description |
|---------|------|-------------|
| Nom | text | Type action (reboot, updateFirmware, resetConfiguration...) |
| ID | text | Identifiant action (action-xxxxx) |
| Créé le | date | Date création |
| Mis à jour | date | Dernière mise à jour |
| Statut | badge | État de l'action |

### Statuts possibles
| Statut | Couleur | Description |
|--------|---------|-------------|
| Terminé | vert (#D1FAE5) | Action complétée avec succès |
| En cours | bleu (#DBEAFE) | Action en exécution (spinner) |
| En attente | bleu (#DBEAFE) | Action planifiée |
| Erreur | rouge (#FEE2E2) | Action échouée |

## 5. ACTIONS & INTERACTIONS
- Bouton ↻ → Rafraîchir la liste
- Pagination → Naviguer entre pages

## 6. APIS & ENDPOINTS
- GET /overthebox/{serviceName}/device/actions : Liste actions
- Paramètres : page, pageSize
- Réponse : {data: Array<Action>, totalCount: number}

## 7. TYPES D'ACTIONS
- reboot : Redémarrage appareil
- updateFirmware : Mise à jour système
- resetConfiguration : Réinitialisation config
- backupConfig : Sauvegarde configuration
- syncDevices : Synchronisation appareils
