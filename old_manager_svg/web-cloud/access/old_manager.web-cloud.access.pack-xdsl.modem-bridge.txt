# Cahier des Charges : web-cloud/access/pack-xdsl - Modem Bridge

## 1. VUE D'ENSEMBLE
- **Écran** : Configuration du mode Bridge du modem
- **Objectif** : Activer/désactiver le mode bridge
- **Contexte** : Configuration avancée pour utilisateurs avec routeur externe

## 2. LAYOUT & STRUCTURE
- 2 colonnes : Configuration | Comparaison modes
- Warning confirmation si activation

## 3. COMPOSANTS UI
- **OdsCard** : 2 cartes (config + explication)
- **OdsToggle** : Mode bridge on/off
- **OdsMessage** : Warning confirmation (jaune)

## 4. SECTION CONFIGURATION
- Titre : "Mode Bridge"
- Description :
  - "Le mode bridge désactive le routeur intégré au modem."
  - "Utilisez ce mode si vous avez votre propre routeur."
- Toggle : Désactivé/Activé

## 5. WARNING CONFIRMATION
- Titre : "⚠ Êtes-vous sûr ?"
- Message : "L'activation du mode bridge :"
- Liste impacts :
  - "Désactivera le WiFi et le DHCP du modem"
  - "Nécessitera un routeur externe pour accéder à Internet"

## 6. COMPARAISON DES MODES

### Mode Routeur (par défaut) - Fond vert
- WiFi et DHCP intégrés actifs
- NAT/PAT et firewall fonctionnels
- Configuration simplifiée
- Idéal pour la plupart des utilisateurs

### Mode Bridge - Fond bleu
- Modem = simple "pont" vers Internet
- WiFi et DHCP du modem désactivés
- Nécessite un routeur externe
- Pour utilisateurs avancés avec leur propre équipement réseau

## 7. APIS & ENDPOINTS
- GET /xdsl/{serviceName}/modem/bridge : État actuel
- PUT /xdsl/{serviceName}/modem/bridge : Activer/désactiver
