# Cahier des Charges : web-cloud/domain - Modal Ajouter redirection

## 1. VUE D'ENSEMBLE
- **Modal** : Création d'une redirection web
- **Type** : Wizard 4 étapes
- **Objectif** : Configurer une redirection HTTP pour un sous-domaine

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Stepper horizontal 4 étapes
- Formulaire sous-domaine
- Checkbox www
- Bandeau info
- Panneau aide types
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur avec overlay
- **OdsStepper** : 4 étapes
- **OdsInput** : Sous-domaine
- **OdsCheckbox** : Option www
- **OdsMessage** : Bandeau info bleu
- **OdsCard** : Panneau types redirection
- **OdsButton** : Annuler, Suivant

## 4. ÉTAPES WIZARD
| Étape | Nom | État |
|-------|-----|------|
| 1 | Sous-domaine | Actif |
| 2 | Type de redirection | À venir |
| 3 | Cible | À venir |
| 4 | Résumé | À venir |

## 5. DESCRIPTION ÉTAPE 1
"Indiquez le sous-domaine que vous souhaitez rediriger."

## 6. CHAMP SOUS-DOMAINE
| Élément | Valeur |
|---------|--------|
| Input | "www" |
| Suffixe fixe | .example.com (fond gris) |

## 7. CHECKBOX WWW
☑ "Créer aussi la redirection pour www.www.example.com"

## 8. BANDEAU INFO
Fond bleu (#E8F0FE) :
- "ℹ️ Informations"
- "Laissez le champ vide pour rediriger le domaine principal (example.com)."
- "Les sous-domaines existants peuvent être écrasés par cette redirection."

## 9. PANNEAU TYPES DE REDIRECTION
Fond gris avec descriptions :

### Redirection visible
L'URL affichée dans le navigateur change vers l'adresse de destination.

### Redirection invisible (iframe)
L'URL reste celle du domaine source mais affiche le contenu de la cible.

### Redirection permanente (301)
Indique aux moteurs de recherche que la redirection est définitive.

## 10. BOUTONS FOOTER
- Annuler (secondaire)
- Suivant → (primaire)

## 11. APIS & ENDPOINTS
- POST /domain/zone/{zoneName}/redirection : Créer redirection
