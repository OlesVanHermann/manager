# Cahier des Charges : web-cloud/domain - Modal Modifier contact

## 1. VUE D'ENSEMBLE
- **Modal** : Modification des informations du contact
- **Type** : Formulaire multi-sections
- **Objectif** : Éditer les informations du contact propriétaire/admin/technique

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (✕)
- Section Identité
- Section Adresse
- Section Contact
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (700x600)
- **OdsInput** : Champs de formulaire
- **OdsButton** : Annuler, Enregistrer

## 4. TITRE
"Modifier les informations du contact"

## 5. SECTION IDENTITÉ
| Champ | Type | Obligatoire | Valeur exemple |
|-------|------|-------------|----------------|
| Prénom | Input texte | Oui (*) | Jean |
| Nom | Input texte | Oui (*) | Dupont |

Layout : 2 colonnes (310px + 316px)

## 6. SECTION ADRESSE
| Champ | Type | Obligatoire | Valeur exemple |
|-------|------|-------------|----------------|
| Adresse | Input texte | Oui (*) | 123 Rue de la Paix |
| Code postal | Input texte | Oui (*) | 75001 |
| Ville | Input texte | Oui (*) | Paris |
| Pays | Input/Select | Oui (*) | France |

Layout : 
- Adresse : pleine largeur (652px)
- Code postal / Ville / Pays : 3 colonnes (200px chacune)

## 7. SECTION CONTACT
| Champ | Type | Obligatoire | Valeur exemple |
|-------|------|-------------|----------------|
| E-mail | Input email | Oui (*) | jean.dupont@example.com |
| Téléphone | Input tel | Non | +33 1 23 45 67 89 |

Layout : 2 colonnes (310px + 316px)

## 8. BOUTONS FOOTER
- Annuler (secondaire, à gauche)
- Enregistrer (primaire, à droite)

## 9. APIS & ENDPOINTS
- PUT /domain/{serviceName}/contact : Mettre à jour contact
