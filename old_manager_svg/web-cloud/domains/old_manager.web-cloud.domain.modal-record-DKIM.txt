# Cahier des Charges : web-cloud/domain - Modal Ajouter enregistrement DKIM

## 1. VUE D'ENSEMBLE
- **Modal** : Création d'un enregistrement DNS de type DKIM
- **Type** : Formulaire avancé de création
- **Objectif** : Ajouter un enregistrement DKIM (DomainKeys Identified Mail)

## 2. STRUCTURE MODAL
- Header bleu OVH avec titre, badge DKIM et bouton fermer (✕)
- Champ sous-domaine (selector._domainkey)
- Champ TTL
- Options version et granularité
- Algorithmes de hashage
- Algorithme de clé
- Champ notes
- Champ clé publique (textarea)
- Option révocation
- Types de services (radio)
- Mode test (radio)
- Mode strict (radio)
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Grande modal (700x750)
- **OdsInput** : Sous-domaine, TTL, Granularité, Notes
- **OdsTextarea** : Clé publique
- **OdsCheckbox** : Version DKIM1, SHA-1, SHA-256, RSA, Révocation
- **OdsRadio** : Services, Mode test, Mode strict
- **OdsBadge** : Type DKIM (violet)
- **OdsButton** : Annuler, Ajouter l'entrée

## 4. TITRE
"Ajouter une entrée DNS - DKIM" avec badge [DKIM]

## 5. CHAMP SOUS-DOMAINE
| Élément | Valeur |
|---------|--------|
| Label | "Sous-domaine" |
| Input | "selector._domainkey" |
| Suffixe (addon gris) | ".exemple.com" |

## 6. CHAMP TTL
| Élément | Valeur |
|---------|--------|
| Label | "TTL" |
| Input | 3600 |

## 7. OPTIONS VERSION
| Élément | État |
|---------|------|
| ☑ DKIM1 | Coché |

## 8. CHAMP GRANULARITÉ
| Élément | Valeur |
|---------|--------|
| Label | "Granularité" |
| Input | "*" |

## 9. ALGORITHMES DE HASHAGE
| Élément | État |
|---------|------|
| ☑ SHA-1 | Coché |
| ☑ SHA-256 | Coché |

## 10. ALGORITHME DE CLÉ
| Élément | État |
|---------|------|
| ☑ RSA | Coché |

## 11. CHAMP NOTES
| Élément | Valeur |
|---------|--------|
| Label | "Notes" |
| Placeholder | "Optionnel" |

## 12. CHAMP CLÉ PUBLIQUE
| Élément | Valeur |
|---------|--------|
| Label | "Clé publique (encodée base64)" |
| Obligatoire | Oui (*) |
| Type | Textarea |
| Valeur exemple | "MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA...wIDAQAB" |

## 13. OPTION RÉVOCATION
| Élément | État |
|---------|------|
| ☐ Révoquer cette clé | Non coché |

## 14. TYPES DE SERVICES (RADIO)
| Option | État |
|--------|------|
| ● Tous (*) | Sélectionné |
| ○ Email uniquement | Non sélectionné |
| ○ Aucun | Non sélectionné |

## 15. MODE TEST (FLAG Y)
| Option | État |
|--------|------|
| ● Non (production) | Sélectionné |
| ○ Oui (test) | Non sélectionné |

## 16. MODE STRICT (FLAG S)
| Option | État |
|--------|------|
| ○ Oui (strict) | Non sélectionné |
| ● Non (relaxed) | Sélectionné |

## 17. BOUTONS FOOTER
- Annuler (secondaire)
- Ajouter l'entrée (primaire)

## 18. APIS & ENDPOINTS
- POST /domain/zone/{zoneName}/record : Créer enregistrement DKIM
  - fieldType: "TXT" (DKIM utilise TXT)
  - subDomain: "selector._domainkey"
  - target: valeur DKIM formatée
  - ttl: number
