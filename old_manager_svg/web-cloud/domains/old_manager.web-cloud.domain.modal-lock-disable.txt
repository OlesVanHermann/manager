# Cahier des Charges : web-cloud/domain - Modal Désactiver protection

## 1. VUE D'ENSEMBLE
- **Modal** : Désactivation de la protection du domaine
- **Type** : Modal de confirmation avec aide contextuelle
- **Objectif** : Désactiver le verrouillage transfert du domaine

## 2. STRUCTURE MODAL
- Header bleu OVH avec titre et bouton fermer (✕)
- Question de confirmation
- Bandeau info (accès AUTH)
- Bandeau warning (DNSSEC)
- Panneau aide latéral
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (700x480)
- **OdsMessage** : Bandeau info bleu, Bandeau warning jaune
- **OdsCard** : Panneau aide
- **OdsButton** : Annuler, Valider

## 4. TITRE
"Désactiver la protection du domaine"

## 5. QUESTION
"Souhaitez-vous réellement désactiver la protection pour votre domaine **exemple.com** ?"

## 6. BANDEAU INFO
Fond bleu (#DBEAFE), bordure bleue :
- "Lorsque la protection sera désactivée, vous aurez accès au code AUTH/INFO permettant le transfert de domaine."

## 7. BANDEAU WARNING
Fond jaune (#FEF3C7), bordure orange :
- "Votre domaine utilise le DNSSEC, vérifiez que votre nouveau prestataire puisse correctement le gérer."

## 8. PANNEAU AIDE
Fond gris (#F9FAFB), contenu :

### Protection du domaine
"Activer la protection permet de vous protéger contre le vol de domaine."

### Qu'est-ce que le vol ?
"Le transfert est effectué sans vérification d'identité suffisante."

### Comment OVH protège ?
"Aucun transfert ne peut être demandé à votre insu."

### Note
"À noter : désactivez pour transférer vers un autre bureau d'enregistrement."

## 9. BOUTONS FOOTER
- Annuler (secondaire)
- Valider (primaire)

## 10. APIS & ENDPOINTS
- PUT /domain/{serviceName}/lock : Désactiver protection (locked: false)
