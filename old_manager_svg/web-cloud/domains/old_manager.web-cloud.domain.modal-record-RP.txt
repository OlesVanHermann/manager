# Cahier des Charges : web-cloud/domain - Modal Ajouter enregistrement RP

## 1. VUE D'ENSEMBLE
- **Modal** : Création d'un enregistrement DNS de type RP
- **Type** : Formulaire de création
- **Objectif** : Ajouter un enregistrement RP (Responsible Person)

## 2. STRUCTURE MODAL
- Header bleu OVH avec titre, badge RP et bouton fermer (✕)
- Champ sous-domaine
- Champ TTL
- Champ boîte mail du responsable
- Champ pointeur TXT
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (650x450)
- **OdsInput** : Sous-domaine, TTL, Mbox, TXT-dname
- **OdsBadge** : Type RP (violet)
- **OdsButton** : Annuler, Ajouter l'entrée

## 4. TITRE
"Ajouter une entrée DNS - RP" avec badge [RP]

## 5. CHAMP SOUS-DOMAINE
| Élément | Valeur |
|---------|--------|
| Label | "Sous-domaine" |
| Input | "admin" |
| Suffixe (addon gris) | ".exemple.com" |

## 6. CHAMP TTL
| Élément | Valeur |
|---------|--------|
| Label | "TTL" |
| Input | 3600 |

## 7. CHAMP BOÎTE MAIL
| Élément | Valeur |
|---------|--------|
| Label | "Boîte mail du responsable (mbox)" |
| Obligatoire | Oui (*) |
| Input | "admin.exemple.com." |
| Aide | "Format: le @ est remplacé par un point (admin@exemple.com → admin.exemple.com.)" |

## 8. CHAMP POINTEUR TXT
| Élément | Valeur |
|---------|--------|
| Label | "Pointeur vers enregistrement TXT (txt-dname)" |
| Obligatoire | Oui (*) |
| Input | "rp-info.exemple.com." |

## 9. BOUTONS FOOTER
- Annuler (secondaire)
- Ajouter l'entrée (primaire)

## 10. APIS & ENDPOINTS
- POST /domain/zone/{zoneName}/record : Créer enregistrement RP
  - fieldType: "RP"
  - subDomain: string
  - target: "mbox txt-dname"
  - ttl: number
