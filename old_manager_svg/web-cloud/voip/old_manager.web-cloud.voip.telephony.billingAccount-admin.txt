# Cahier des Charges : telecom/telephony - Administration groupe

## 1. RESUME EXECUTIF

### Perimetre
Page d'administration d'un groupe de facturation telephonie.

### Objectif fonctionnel
- Acceder aux options du groupe
- Gerer le regroupement de lignes
- Suivre les portabilites en cours
- Supprimer le groupe

---

## 2. LAYOUT

### Structure page avec grille de cartes actions et section portabilites

## 3. COMPOSANTS UI

### Header page
| Element | Description |
|---------|-------------|
| Retour | ← Retour a l'administration de groupe |
| Breadcrumb | Telecom / VoIP / billing-xxx / Administration |
| Titre | Administration |

### Cartes actions (grille 3x2)
| Carte | Description | Action |
|-------|-------------|--------|
| Options du groupe | Configurez options generales | Configurer → |
| Regroupement de lignes | Gerez lignes et appartenance | Gerer → |
| Ajouter un groupe | Creez nouveau groupe facturation | Creer → |
| Portabilites en cours | Suivez demandes portabilite | Voir les portabilites → |
| Manager marque blanche | Personnalisez interface | Personnaliser → |
| Supprimer le groupe | Supprimez definitivement | Supprimer → |

### Carte action standard
| Element | Description |
|---------|-------------|
| Icone | Emoji descriptif |
| Titre | Nom action |
| Description | 2 lignes texte gris |
| Lien | Action → (bleu) |

### Carte suppression
| Element | Description |
|---------|-------------|
| Fond | Rouge clair |
| Bordure | Rouge |
| Texte | Rouge |

### Badge portabilites
| Etat | Style |
|------|-------|
| En cours | Fond jaune, texte orange |
| Validee | Fond vert, texte vert |

### Section Portabilites en cours
| Element | Description |
|---------|-------------|
| Titre section | Portabilites en cours |
| Liste | Lignes portabilite |

### Ligne portabilite
| Element | Description |
|---------|-------------|
| Numero | Format 0033XXXXXXXXX |
| Operateur | Nom operateur source |
| Statut | Badge En attente / Validee |
| Date prevue | JJ/MM/AAAA |
| Actions | Annuler / Relancer ou Attacher / Supprimer |

### Bouton nouvelle portabilite
| Element | Description |
|---------|-------------|
| Type | Primary |
| Texte | + Nouvelle portabilite |

## 4. APIS & DONNEES

### Endpoints
```
GET /telephony/{billingAccount}
DELETE /telephony/{billingAccount}
GET /telephony/{billingAccount}/portability
POST /telephony/{billingAccount}/portability
DELETE /telephony/{billingAccount}/portability/{id}
```

### Modele
```typescript
interface BillingAccount {
  id: string;
  name: string;
  status: string;
}

interface Portability {
  id: string;
  number: string;
  operator: string;
  status: 'pending' | 'validated' | 'cancelled';
  scheduledDate: string;
}
```
