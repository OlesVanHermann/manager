# Cahier des Charges : telecom/telephony - Composant Identite du titulaire

## 1. RESUME EXECUTIF

### Perimetre
Composant reutilisable pour les informations d'identite dans les formulaires de commande.

### Objectif fonctionnel
- Collecter les informations d'identite du titulaire
- Gerer les champs particulier vs entreprise
- Valider les informations SIRET/APE

---

## 2. LAYOUT

### Structure carte avec champs sur 2 colonnes

---

## 3. COMPOSANTS UI

### Titre section
| Element | Description |
|---------|-------------|
| Titre | Identite du titulaire |
| Note | * Champs obligatoires |

### Champs Particulier (toujours visibles)
| Champ | Type | Obligatoire |
|-------|------|-------------|
| Nom | Input | Oui |
| Prenom | Input | Oui |
| Email | Input | Non |
| Telephone | Input | Non |

### Separateur
| Element | Description |
|---------|-------------|
| Ligne | Separateur horizontal |
| Note | Champs supplementaires pour les entreprises |

### Champs Entreprise
| Champ | Type | Obligatoire |
|-------|------|-------------|
| Organisation | Input | Oui (si entreprise) |
| Denomination sociale | Input | Oui (si entreprise) |
| SIRET | Input | Oui (si entreprise) |
| Code APE | Input | Oui (si entreprise) |

---

## 4. APIS & DONNEES

### Validation
| Champ | Regle |
|-------|-------|
| Nom | 2-100 caracteres, lettres et tirets |
| Prenom | 2-100 caracteres, lettres et tirets |
| Email | Format email valide |
| Telephone | Format international |
| SIRET | 14 chiffres, algorithme Luhn |
| Code APE | Format XXXX[A-Z] |

### Structure donnees
```json
{
  "identity": {
    "lastName": "Dupont",
    "firstName": "Jean",
    "email": "jean.dupont@email.com",
    "phone": "+33612345678"
  },
  "company": {
    "organisation": "Ma Societe",
    "legalName": "MA SOCIETE SARL",
    "siret": "12345678901234",
    "ape": "6201Z"
  }
}
```

## 6. REGLES METIER

- Nom et prenom toujours obligatoires
- Champs entreprise affiches si type=entreprise
- SIRET verifie via algorithme Luhn
- Code APE verifie contre liste INSEE
- Utilisable dans tous les formulaires de commande
