# Cahier des Charges : telecom/line - Depannage

## 1. RESUME EXECUTIF

### Perimetre
Assistant de depannage guide pour resoudre les problemes de telephone.

### Objectif fonctionnel
- Identifier le type de probleme
- Guider l'utilisateur vers la solution
- Proposer des actions correctives

---

## 2. LAYOUT

### Structure avec image telephone et liste de problemes

---

## 3. COMPOSANTS UI

### Zone information
| Element | Description |
|---------|-------------|
| Info box | Lien vers guide de depannage en ligne |

### Section selection probleme
| Element | Description |
|---------|-------------|
| Titre | Quel est votre probleme ? |

### 2 colonnes
| Colonne | Contenu |
|---------|---------|
| Gauche | Image telephone + infos |
| Droite | Liste radio des problemes |

### Infos telephone
| Element | Valeur |
|---------|--------|
| Image | Placeholder telephone |
| Modele | Cisco SPA504G |
| MAC | 00:1A:2B:3C:4D:5E |

### Liste des problemes (radio buttons)
| Probleme |
|----------|
| Le telephone ne s'allume pas |
| Pas de tonalite |
| Impossible de recevoir des appels |
| Impossible de passer des appels |
| Qualite audio degradee |
| Afficheur ne fonctionne pas |
| Autre probleme |

### Boutons action
| Bouton | Type | Action |
|--------|------|--------|
| Ca fonctionne ! | Primary | Probleme resolu |
| Ca ne fonctionne toujours pas | Default | Etape suivante |

---

## 4. APIS & DONNEES

### Endpoints
```
GET /telephony/{billingAccount}/line/{serviceName}/phone
    Response: { brand, model, macAddress }

GET /telephony/{billingAccount}/line/{serviceName}/phone/troubleshooting/problems
    Response: [{ id, label, solutions }]

POST /telephony/{billingAccount}/line/{serviceName}/phone/troubleshooting/diagnose
    Body: { problemId }
```
