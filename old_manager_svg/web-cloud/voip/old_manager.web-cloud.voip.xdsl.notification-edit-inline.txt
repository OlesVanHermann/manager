# Cahier des Charges : telecom/xdsl - Edition inline notification

## 1. RESUME EXECUTIF

### Perimetre
Mode d'edition en ligne pour modifier une notification existante.

### Objectif fonctionnel
- Modifier une notification sans ouvrir de modal
- Edition rapide directement dans le tableau

## 3. COMPOSANTS UI

### Ligne en mode edition
| Element | Description |
|---------|-------------|
| Fond | Bleu clair (#EFF6FF) |
| Bordure top | Bleu (#0050D7) |

### Champs editables inline
| Colonne | Composant |
|---------|-----------|
| Type | Select (SMS/Email) |
| Compte | Select ou -- |
| Contact | Input text |
| Frequence | Select |
| Notification incidents | Select (Oui/Non) |
| Delai | Stepper numerique |

### Stepper numerique compact
| Element | Description |
|---------|-------------|
| Bouton - | Decremente |
| Valeur | Affichage centre |
| Bouton + | Incremente |

### Boutons action
| Bouton | Type | Action |
|--------|------|--------|
| ✓ | Success (vert) | Sauvegarde les modifications |
| ✕ | Default | Annule l'edition |

### Encart info
| Element | Description |
|---------|-------------|
| Type | Info (fond bleu) |
| Message | Mode edition en ligne : Modifiez les valeurs... |

---

## 4. APIS & DONNEES

### Endpoints
```
PUT /xdsl/{serviceName}/monitoringNotifications/{notificationId}
    Body: {
      type, smsAccount, phone, email,
      frequency, allowIncident, downThreshold
    }
```

---

## 5. TRADUCTIONS (i18n)

```json
  "xdsl_notif_edit_save": "Sauvegarder",

  "xdsl_notif_edit_hint": "Mode edition en ligne : Modifiez les valeurs directement et cliquez sur ✓ pour sauvegarder ou ✕ pour annuler."

```

---

## 6. REGLES METIER

- Une seule ligne editable a la fois
- Clic sur ligne passe en mode edition
- Validation locale avant soumission
- Retour auto au mode lecture apres sauvegarde
- ESC pour annuler l'edition
