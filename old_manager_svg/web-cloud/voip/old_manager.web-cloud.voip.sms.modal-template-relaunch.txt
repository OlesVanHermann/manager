# Cahier des Charges : telecom/sms - Modal relancer validation modèle

## 1. VUE D'ENSEMBLE
- **Type** : Modal de resoumission
- **Contexte** : Page modèles SMS
- **Objectif** : Resoumettre un modèle refusé pour validation

## 2. DIMENSIONS
| Propriété | Valeur |
|-----------|--------|
| Modal largeur | 550px |
| Modal hauteur | 480px |

## 3. COMPOSANTS UI
- **OdsModal** : Container principal
- **OdsMessage** : Avertissement refus
- **OdsInput** : Nom (readonly), Description, Message
- **OdsButton** : Annuler, Soumettre à nouveau

## 4. HEADER MODAL
| Propriété | Valeur |
|-----------|--------|
| Titre | "Relancer la validation du modèle" |

## 5. MESSAGE WARNING
| Propriété | Valeur |
|-----------|--------|
| Type | Warning (jaune) |
| Icône | ⚠ |
| Texte | "Ce modèle a été refusé. Modifiez-le et relancez la validation." |

## 6. CHAMPS FORMULAIRE
| Champ | Type | Requis | Note |
|-------|------|--------|------|
| Nom du modèle | Input (readonly) | - | Non modifiable |
| Description | Textarea | Oui * | 31-249 caractères |
| Message | Textarea | Oui * | 31-249 caractères |

## 7. FOOTER MODAL
| Bouton | Style | Action |
|--------|-------|--------|
| Annuler | Secondary | Ferme modal |
| Soumettre à nouveau | Primary | Resoumet le modèle |

## 8. APIS & ENDPOINTS
- PUT /sms/{serviceName}/templates/{templateId} - Modifier et resoumettre
