# Cahier des Charges : web-cloud/voip/alias - Config Scheduler

## 1. VUE D'ENSEMBLE

**Type:** Page calendrier interactif
**Fonction:** Configurer les plages horaires d'ouverture et fermetures exceptionnelles
**Contexte:** Configuration file d'appels du numéro alias

## 2. NAVIGATION

### Breadcrumb
Télécom › Groupe VoIP › Numéro alias 0033972XXXXXX › Plages horaires et fermetures

### Retour
← Plages horaires et fermetures

## 3. DESCRIPTION

"Configurez les plages horaires d'ouverture et les jours de fermeture exceptionnels."

## 5. VUE SEMAINE (1100x500px)

### Colonne horaire
- Heures: 08:00, 09:00, 10:00... 15:00
- Largeur: 60px
- Position: gauche

### En-têtes jours (header 40px)
- Background: #F3F4F6
- Format: "Lun 6", "Mar 7", "Mer 8", "Jeu 9", "Ven 10", "Sam 11", "Dim 12"

### Grille
- Colonnes verticales: 7 jours
- Lignes horizontales: toutes les heures
- Border: #E5E7EB

### Événements ouverts (bleu)
- Background: #DBEAFE
- Texte: "Ouvert" + horaires
- Exemple: "08:00-12:00", "14:00-18:00"

### Événements fermés (rouge)
- Background: #FEE2E2 opacity 0.5
- Texte: "Fermé"
- Samedi/Dimanche entiers

## 6. LÉGENDE

- ☐ Bleu: "Plage ouverte"
- ☐ Rouge: "Fermeture exceptionnelle"

## 8. APIS & ENDPOINTS

```
# Configuration conditions temporelles
GET /telephony/{billingAccount}/easyHunting/{serviceName}/timeConditions
→ Liste des conditions

# Créer une condition
POST /telephony/{billingAccount}/easyHunting/{serviceName}/timeConditions
Body: {
  slot: "slot1",
  policy: "available",
  timeFrom: "0800",
  timeTo: "1200",
  day: "monday"
}

# Modifier une condition
PUT /telephony/{billingAccount}/easyHunting/{serviceName}/timeConditions/{conditionId}
Body: { timeFrom: "0900", timeTo: "1300" }

# Supprimer une condition
DELETE /telephony/{billingAccount}/easyHunting/{serviceName}/timeConditions/{conditionId}

# Fermetures exceptionnelles
GET /telephony/{billingAccount}/easyHunting/{serviceName}/exceptionalClosure
POST /telephony/{billingAccount}/easyHunting/{serviceName}/exceptionalClosure
Body: {
  dateStart: "2025-12-25",
  dateEnd: "2025-12-26",
  reason: "Noël"
}
```
