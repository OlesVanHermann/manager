## 1. RÉSUMÉ EXÉCUTIF

### Périmètre
Page dashboard d'un service FreeFax individuel.

### Objectif fonctionnel
Afficher les informations générales et permettre la configuration d'un fax.

### Utilisateurs cibles
Clients OVHcloud disposant d'un service FreeFax.

## 3. PAGES & NAVIGATION

### Route
| Propriété | Valeur |
|-----------|--------|
| State | freefaxes.freefax |
| URL | /freefax/:serviceName |
| Component | ovhManagerFreefaxComponent |

### Paramètres
| Param | Type | Description |
|-------|------|-------------|
| serviceName | string | Identifiant du fax |

### Breadcrumb
Affiche serviceName

### Titre page
freefax_page_title : "Freefax {{name}}"

---

## 4. COMPOSANTS UI

### Structure page
La page est composée de plusieurs sections (tiles) :

| Section | Module | Description |
|---------|--------|-------------|
| Informations générales | information | Infos du service |
| Configuration fax | faxConfiguration | Paramètres envoi fax |
| Configuration répondeur | voicemailConfiguration | Paramètres répondeur |
| Crédits | credit | Commande de fax |
| Notifications | notifications | Alertes email |

### Binding
```javascript
bindings: {
  serviceName: '<',
}
```

---

## 5. APIs & DONNÉES

### Endpoint principal
| Méthode | Endpoint | Description |
|---------|----------|-------------|
| GET | /freefax/{serviceName} | Détails du fax |

### Modèle de données
```typescript
interface Freefax {
  number: string;
  fromName: string;
  fromEmail: string;
  faxQuality: 'normal' | 'high' | 'best';
  faxMaxCall: number;
  faxTagLine: string;
}
```

## 7. RÈGLES MÉTIER

### États du service
| État | Affichage |
|------|-----------|
| Actif | Dashboard complet |
| Expiré | Message freefax_expired_error |
| Erreur chargement | Message freefax_detail_error |

### Sous-modules
Chaque sous-module gère sa propre logique :
- credit : Commande de crédits fax
- faxConfiguration : Configuration envoi (voir TXT dédié)
- notifications : Gestion alertes email
- voicemailConfiguration : Paramètres répondeur
