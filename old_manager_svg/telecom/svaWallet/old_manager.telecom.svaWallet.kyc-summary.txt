# Cahier des Charges : telecom/svaWallet - KYC Recapitulatif

## 1. RESUME EXECUTIF

### Perimetre
Page recapitulative du dossier KYC pour portefeuille SVA.

### Objectif fonctionnel
- Afficher la synthese du dossier KYC
- Visualiser les statuts de validation
- Acceder aux modifications par section
- Identifier les elements manquants

## 4. APIS & DONNEES

### Endpoint
```
GET /svaWallet/{walletId}/kyc/summary
    Response: {
      identity: {
        legalName: "MA SOCIETE SAS",
        siret: "12345678901234",
        legalForm: "SAS",
        address: "10 rue de la Paix, 75001 Paris, France"
      },
      bankAccount: {
        accountHolder: "MA SOCIETE SAS",
        iban: "FR76123456789012345678901xx"
      },
      beneficiaries: [
        { name: "Jean DUPONT", birthDate: "1975-03-15", nationality: "France", isLegalRep: true },
        { name: "Marie MARTIN", birthDate: "1980-08-22", nationality: "France", isLegalRep: false }
      ],
      documents: {
        idCard: { status: "validated" },
        kbis: { status: "pending" },
        rib: { status: "validated" },
        proofAddress: { status: "not_provided" }
      },
      overallStatus: "pending"
    }
```

### Statuts KYC
| Statut | Description |
|--------|-------------|
| incomplete | Dossier incomplet |
| pending | En cours de validation |
| validated | Dossier valide |
| rejected | Dossier refuse |

## 6. REGLES METIER

- Synthese read-only avec liens vers editions
- Statut global = minimum des statuts individuels
- Dossier valide = tous documents valides
- Reversements actives apres validation complete
- Mise a jour en temps reel des statuts
