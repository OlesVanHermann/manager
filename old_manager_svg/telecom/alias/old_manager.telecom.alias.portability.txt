## 4. COMPOSANTS UI

### Wizard portabilite
1. Saisie RIO (Releve d'Identite Operateur)
2. Selection operateur actuel
3. Date de portage souhaitee
4. Upload documents (mandat, facture)
5. Validation

### Composants
- oui-stepper : Wizard multi-etapes
- oui-file : Upload documents
- oui-calendar : Selection date

--------------------------------------------------------------------------------

## 5. APIs & DONNEES

GET /telephony/portability
POST /telephony/portability
Body: {
  rio: string,
  desiredDate: string,
  billingAccount: string
}

### Statuts
- todo : En attente traitement
- doing : En cours
- done : Termine
- error : Erreur
- cancelled : Annule
