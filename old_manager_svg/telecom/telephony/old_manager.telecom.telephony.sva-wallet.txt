# Cahier des Charges : telecom/telephony - Portefeuille SVA

## 1. RESUME EXECUTIF

### Perimetre
Page de gestion du portefeuille SVA (Services a Valeur Ajoutee) pour KYC.

### Objectif fonctionnel
- Gerer l'identite entreprise
- Configurer le compte bancaire
- Declarer les beneficiaires effectifs
- Soumettre les documents justificatifs

## 4. APIS & DONNEES

### Endpoints
```
GET /telephony/svaWallet
PUT /telephony/svaWallet/identity
PUT /telephony/svaWallet/bankAccount
GET /telephony/svaWallet/beneficiaries
POST /telephony/svaWallet/beneficiaries
DELETE /telephony/svaWallet/beneficiaries/{id}
GET /telephony/svaWallet/documents
POST /telephony/svaWallet/documents
```

### Modeles
```typescript
interface SvaWallet {
  status: 'pending' | 'verified' | 'rejected';
  identity: CompanyIdentity;
  bankAccount: BankAccount;
  beneficiaries: Beneficiary[];
}

interface CompanyIdentity {
  name: string;
  siret: string;
  address: Address;
  legalRepresentative: string;
  legalRepresentativeRole: string;
}

interface BankAccount {
  iban: string;
  bic: string;
  holder: string;
  verified: boolean;
}

interface Beneficiary {
  id: string;
  firstName: string;
  lastName: string;
  role: string;
  percentage: number;
  verified: boolean;
}
```

## 6. REGLES METIER

- KYC obligatoire pour reversements SVA
- SIRET doit etre valide
- Beneficiaires >25% declares
- Documents: Kbis, CNI representant, RIB
- Delai verification ~48h
