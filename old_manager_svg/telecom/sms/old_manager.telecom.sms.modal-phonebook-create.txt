--------------------------------------------------------------------------------
## 2. CONTEXTE D AFFICHAGE
--------------------------------------------------------------------------------

S affiche quand :
- Aucun carnet n existe (message "Aucun carnet d adresses n a ete trouve")
- Clic sur bouton ajout

--------------------------------------------------------------------------------
## 3. FORMULAIRE
--------------------------------------------------------------------------------

| Champ | Type | Validation |
|-------|------|------------|
| Nom du carnet | Input text | Requis, alphanumerique, max 50 car |

**Placeholder:** Nom du carnet d adresses

**Erreurs:**
- Requis : "Le champ Nom du carnet d adresses est obligatoire."
- Pattern : "Le nom du carnet doit etre compose uniquement de caracteres alphanumeriques."
- Maxlength : "Le nom du carnet d adresses doit faire moins de 50 caracteres."

--------------------------------------------------------------------------------
## 4. BOUTONS
--------------------------------------------------------------------------------

- Valider / Creer (primary)
- Annuler (secondary)

--------------------------------------------------------------------------------
## 5. API
--------------------------------------------------------------------------------

| Methode | Endpoint | Description |
|---------|----------|-------------|
| POST | /sms/{serviceName}/phonebooks | Creer carnet |

**Body:**
```json
{
  "name": "string"
}
```

--------------------------------------------------------------------------------
## 6. MESSAGES RETOUR
--------------------------------------------------------------------------------

- Loading : (spinner)
- Succes : Redirection vers le carnet cree
- Erreur : Message API
