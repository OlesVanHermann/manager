================================================================================
CAHIER DES CHARGES - old_manager.telecom.common.editable-service-name
================================================================================

1. IDENTIFICATION
-----------------
Chemin     : telecom/common
Composant  : Nom de service editable
Type       : Composant inline editable
Framework  : AngularJS 1.7.x
Module     : telecom-universe-components

2. DESCRIPTION FONCTIONNELLE
----------------------------
Composant permettant d afficher et modifier le nom d un service.
Affiche le titre en mode lecture avec bouton edition.
En mode edition, affiche un champ texte avec boutons confirmer/annuler.

3. STRUCTURE UI
---------------
Mode lecture :
+------------------------------------------------------------------+
| Mon Service Telecom                              [crayon]        |
| 0033123456789 | Pack Pro                                         |
+------------------------------------------------------------------+

Mode edition :
+------------------------------------------------------------------+
| Veuillez inserer le nouveau nom de ce service                    |
| [Mon Nouveau Service_________] [v] [x]                           |
| 0033123456789 | Pack Pro                                         |
+------------------------------------------------------------------+

4. MODES
--------
| Mode     | Description                                   |
|----------|-----------------------------------------------|
| Lecture  | Titre + bouton crayon pour editer             |
| Edition  | Input texte + boutons confirmer/annuler       |
| Loading  | Sauvegarde en cours                           |

5. ATTRIBUTS COMPOSANT
----------------------
| Attribut           | Type     | Description                      |
|--------------------|----------|----------------------------------|
| title              | string   | Titre affiche (nom service)      |
| serviceName        | string   | Identifiant technique du service |
| serviceNameAppendix| string   | Suffixe optionnel (ex: Pack Pro) |
| disabled           | boolean  | Desactive le bouton edition      |
| maxlength          | number   | Longueur max du nom              |
| onSave             | function | Callback lors de la sauvegarde   |

6. VALIDATIONS
--------------
- Pattern : Pas de caracteres < ou > (regex /^[^<>]+$/)
- Longueur max configurable via maxlength

7. ICONES UTILISEES
-------------------
- oui-icon-pen_concept : Bouton edition (crayon)
- oui-icon-success : Bouton confirmer (check)
- oui-icon-close : Bouton annuler (croix)

8. COMPOSANTS UTILISES
----------------------
- oui-icon : Icones OVH UI Kit

9. TRADUCTIONS FR
-----------------
tuc_editableServiceName_change: "Changer le nom du service"
tuc_editableServiceName_label: "Veuillez inserer le nouveau nom de ce service"
tuc_editableServiceName_confirm: "Confirmer"
tuc_editableServiceName_cancel: "Annuler"

10. ACCESSIBILITE
-----------------
- Labels sr-only pour lecteurs d ecran
- Formulaire avec validation HTML5
- Boutons avec aria-hidden sur icones

11. EXEMPLE D UTILISATION
-------------------------
<tuc-editable-service-name
    title="Mon Service"
    service-name="0033123456789"
    service-name-appendix="Pack Pro"
    on-save="$ctrl.saveServiceName(newName)"
    maxlength="50"
></tuc-editable-service-name>
