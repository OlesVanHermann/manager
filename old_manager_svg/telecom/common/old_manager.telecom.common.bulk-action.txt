1. DESCRIPTION FONCTIONNELLE
================================================================================
Système permettant d'appliquer une action en masse sur plusieurs services
téléphonie du même groupe de facturation. Affiche une modal de sélection
des services cibles.

Cas d'usage :
- Appliquer la même configuration à plusieurs lignes
- Modifier les options de plusieurs services
- Actions groupées sur un billing account

================================================================================

================================================================================
4. SERVICE TucTelephonyBulkAction
================================================================================
Méthodes :
- getServiceList(billingAccount, serviceType)
- filterCompatibleServices(services, currentService)
- applyAction(services, actionFn)
- getBulkActionStatus()

================================================================================
5. FLUX D'EXECUTION
================================================================================
1. Utilisateur configure un service
2. Clic sur "Appliquer à plusieurs services"
3. Chargement des services compatibles du billing account
4. Sélection des services cibles
5. Application de l'action sur chaque service sélectionné
6. Affichage du résultat (succès/erreurs)

================================================================================
6. FILTRAGE DES SERVICES
================================================================================
Services compatibles :
- Même billing account
- Même type de service (line, fax, alias, etc.)
- Exclut le service source
- Optionnel : mêmes capacités/options

================================================================================
7. GESTION DES ERREURS
================================================================================
- Exécution continue en cas d'erreur sur un service
- Résumé final avec succès/échecs
- Détail des erreurs par service

================================================================================

9. EXEMPLE D'INTEGRATION REACT
================================================================================
interface BulkActionProps<T> {
  billingAccount: string;
  currentService: string;
  serviceType: 'line' | 'fax' | 'alias';
  actionFn: (serviceName: string, config: T) => Promise<void>;
  config: T;
  buttonLabel?: string;
}

interface BulkActionResult {
  success: string[];
  errors: Array<{ service: string; error: Error }>;
}
