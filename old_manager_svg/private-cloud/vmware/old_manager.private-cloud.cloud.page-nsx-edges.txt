# Cahier des Charges : private/cloud - Page NSX Edges

## 1. VUE D'ENSEMBLE

**Type:** Page listing avec datagrid
**Fonction:** Gérer les passerelles réseau NSX Edge
**Contexte:** Configuration réseau virtuel VMware NSX

## 2. DIMENSIONS

- Page: 1100x650px
- Datagrid: 1040x480px

## 3. LAYOUT & STRUCTURE

### Header page
- Titre: "NSX Edges"
- Sous-titre: "Gérez vos passerelles réseau NSX"

### Bouton action
"+ Ajouter un Edge" (btn-primary, 160x40px)

## 4. DATAGRID NSX EDGES

### Colonnes
| Colonne | Largeur | Description |
|---------|---------|-------------|
| Nom | 160px | Nom de l'edge (link) |
| Taille | 100px | Medium, Large, X-Large |
| Datacenter | 140px | DC1-Paris, DC2-Londres |
| IP publique | 140px | Adresse IP externe |
| Services | 190px | Badges des services actifs |
| Statut | 120px | Badge Actif |
| Facturation | 130px | Horaire/Mensuel |
| Actions | 40px | Menu ⋮ |

### Badges services
- Background: #EFF6FF
- Text: #0050D7
- Services: NAT, FW, VPN, LB

## 5. MENU ACTIONS

### Dropdown (180x150px)
- "Configurer NAT"
- "Configurer Firewall"
- "Passer en mensuel" (si horaire)
- "Supprimer" (text-danger)

## 6. COMPOSANTS UI

### Badge service
- Height: 22px

- Font-size: 10px

### Badge statut
- Actif: bg #DCFCE7, text #16A34A

## 7. APIS & ENDPOINTS

```
GET /dedicatedCloud/{serviceName}/datacenter/{datacenterId}/nsx/edge
→ Liste des NSX Edges

GET /dedicatedCloud/{serviceName}/datacenter/{datacenterId}/nsx/edge/{edgeId}
→ Détails d'un edge

PUT /dedicatedCloud/{serviceName}/datacenter/{datacenterId}/nsx/edge/{edgeId}
Body: { billingType: "monthly" }
→ Change la facturation

DELETE /dedicatedCloud/{serviceName}/datacenter/{datacenterId}/nsx/edge/{edgeId}
→ Supprime l'edge

GET /dedicatedCloud/{serviceName}/datacenter/{datacenterId}/nsx/edge/{edgeId}/natRule
→ Règles NAT

GET /dedicatedCloud/{serviceName}/datacenter/{datacenterId}/nsx/edge/{edgeId}/firewallRule
→ Règles firewall
```
