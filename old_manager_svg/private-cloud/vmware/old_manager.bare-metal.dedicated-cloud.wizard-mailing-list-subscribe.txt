# Cahier des Charges : bare-metal/dedicated-cloud - Wizard Mailing List Subscribe

## 1. VUE D'ENSEMBLE

**Type:** Wizard modal 2 étapes
**Fonction:** Abonner une adresse email à la mailing list Hosted Private Cloud
**Contexte:** Inscription aux actualités et mises à jour de l'infrastructure

## 2. DIMENSIONS

- Overlay: 1440x600px (fond noir 50%)
- Modal wizard: 700x440px (centrée)

## 3. LAYOUT & STRUCTURE

### Header modal
- Background: #F9FAFB
- Titre: "Abonner à la mailing list"
- Bouton fermer: × (coin droit)
- Hauteur: 60px

### Stepper (2 étapes)
| Étape | Label | État |
|-------|-------|------|
| 1 | Email | Active (bleu) |
| 2 | Confirmation | Inactive (gris) |

## 4. ÉTAPE 1 : EMAIL

### Label du champ
- Ligne 1: "Entrez l'adresse email à abonner à la mailing list"
- Ligne 2: "Hosted Private Cloud (pcc@ml.ovh.net) :"

### Champ email
- Type: email input
- Dimensions: 620x44px
- Placeholder: adresse email
- Valeur exemple: "admin@example.com"

### Texte d'aide
- Couleur: text-muted
- Texte: "Vous recevrez les actualités et les mises à jour concernant votre infrastructure."

## 5. COMPOSANTS UI

### Stepper modal
- Step active: cercle bleu #0050D7 avec numéro blanc
- Step inactive: cercle blanc avec bordure #E5E7EB
- Ligne connexion: #E5E7EB, 2px

### Footer modal
- Séparateur: ligne #E5E7EB
- Boutons alignés à droite

## 6. ACTIONS & INTERACTIONS

### Bouton fermer (×)
- Action: Ferme le modal sans action

### Bouton Annuler
- Style: btn-secondary (100x36px)
- Action: Ferme le modal

### Bouton Suivant
- Style: btn-primary (100x36px)
- Action: Valide l'email et passe à l'étape 2
- Disabled si email invalide

### Validation email
- Format email valide requis
- Affiche erreur si format incorrect

## 7. ÉTAPE 2 : CONFIRMATION

### Contenu (non visible dans le SVG)
- Résumé de l'email saisi
- Bouton "Confirmer" pour finaliser l'inscription
- Bouton "Précédent" pour modifier l'email

## 8. APIS & ENDPOINTS

```
GET /dedicatedCloud/{serviceName}/mailingList
→ Liste les emails déjà abonnés

POST /dedicatedCloud/{serviceName}/mailingList/subscribe
Body: {
  email: "admin@example.com"
}
→ Abonne l'email à la mailing list

DELETE /dedicatedCloud/{serviceName}/mailingList/unsubscribe
Body: {
  email: "admin@example.com"
}
→ Désabonne l'email
```
