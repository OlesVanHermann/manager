# Cahier des Charges : bare-metal/dedicated-cloud - Tab Users IAM

## 1. VUE D'ENSEMBLE

**Type:** Page de configuration avec onglets
**Fonction:** Gérer l'intégration IAM OVHcloud sur l'infrastructure
**Contexte:** Onglet IAM OVHcloud dans la section Utilisateurs

## 2. DIMENSIONS

- Container principal: 1360x550px

## 3. LAYOUT & STRUCTURE

### Breadcrumb
- Hosted Private Cloud VMware > pcc-192-0-2-1 > Utilisateurs > **IAM OVHcloud**

### Onglets
- Utilisateurs vSphere (inactif)
- **IAM OVHcloud** (actif - souligné bleu)

### Zone principale
1. Titre
2. Description (4 lignes)
3. Message avertissement (si applicable)
4. Statut IAM
5. Boutons actions (droite)
6. Bloc info

## 4. COMPOSANTS UI

### Titre
- Text: "IAM OVHcloud"
- Font-size: 18px

### Description
4 lignes explicatives sur l'IAM OVHcloud:
- "L'activation de l'IAM OVHcloud délègue la gestion des accès au service IAM OVHcloud."
- "La gestion des rôles associés et leurs autorisations dans vSphere s'effectue depuis cette page."
- "La gestion des politiques et accès s'effectue depuis l'IAM OVHcloud."
- "L'IAM OVHcloud est intégré pour le produit vSphere uniquement."

### Message avertissement
- Background: #FEF3C7
- Texte: "Les infrastructures bénéficiant d'options de sécurité renforcée ou d'un service certifié (HDS, PCI-DSS ou SNC) ne peuvent actuellement pas utiliser l'IAM OVHcloud."

### Statut
- Label: "Statut"
- Badge: "Activé" (bg-success #DCFCE7)

### Boutons actions (droite)
1. "Désactiver l'IAM OVHcloud" (btn-secondary)
2. "Gérer les politiques et accès ↗" (btn-secondary, lien externe)

### Bloc info
- Background: #EFF6FF
- Icône: i (cercle bleu)
- Texte: "Une fois l'IAM OVHcloud activé, vous pouvez créer des rôles IAM depuis l'onglet 'Utilisateurs vSphere' et gérer leurs politiques d'accès depuis le tableau de bord IAM OVHcloud."

## 5. ACTIONS & INTERACTIONS

### Désactiver l'IAM OVHcloud
- Ouvre modal de confirmation

### Gérer les politiques et accès
- Ouvre le dashboard IAM OVHcloud (nouvel onglet)

## 6. APIS & ENDPOINTS

```
GET /dedicatedCloud/{serviceName}/iam
→ Récupère le statut IAM

POST /dedicatedCloud/{serviceName}/iam/enable
→ Active l'IAM OVHcloud

POST /dedicatedCloud/{serviceName}/iam/disable
→ Désactive l'IAM OVHcloud
```
