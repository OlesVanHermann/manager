# Cahier des Charges : bare-metal/nas-ha - Créer snapshot

## 1. VUE D'ENSEMBLE
- **Page** : Créer un snapshot manuel
- **Type** : Modal formulaire
- **Objectif** : Créer un snapshot d'une partition

## 2. STRUCTURE MODAL
- Header avec titre + bouton fermer
- Champ partition (lecture seule)
- Champ nom snapshot
- Message info rétention
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Modal principale
- **OdsInput** : Partition (disabled), nom snapshot
- **OdsMessage** : Info rétention
- **OdsButton** : Annuler, Créer

## 4. CHAMPS FORMULAIRE
| Champ | Type | Requis | Description |
|-------|------|--------|-------------|
| Partition | Input disabled | - | Nom partition (backup) |
| Nom | Input | Oui | Nom du snapshot (ex: snapshot-2024-12-29) |

## 5. MESSAGE INFO
| Propriété | Valeur |
|-----------|--------|
| Type | Info (bleu) |
| Icône | ℹ |
| Texte | Les snapshots sont conservés 48h automatiquement |

## 6. BOUTONS
| Bouton | Style | Position |
|--------|-------|----------|
| Annuler | Outline | Gauche |
| Créer | Primary | Droite |

## 7. APIS & ENDPOINTS
- POST /dedicated/nasha/{serviceName}/partition/{partitionName}/snapshot
  - Paramètres : name (string)
