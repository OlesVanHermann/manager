# Cahier des Charges : bare-metal/nas-ha - Modal créer partition

## 1. VUE D'ENSEMBLE
- **Type** : Page formulaire de création
- **Contexte** : NAS-HA Dedicated
- **Objectif** : Créer une nouvelle partition sur le NAS-HA

## 2. DIMENSIONS
| Propriété | Valeur |
|-----------|--------|
| Largeur | 600px |
| Hauteur | 500px |

## 3. COMPOSANTS UI
- **OdsText** : Titre, labels, aides
- **OdsInput** : Nom, taille, description
- **OdsRadio** : Sélection protocole
- **OdsCard** : Options de protocole
- **OdsButton** : Annuler, Créer

## 4. HEADER
| Propriété | Valeur |
|-----------|--------|
| Titre | "Créer une partition" |

## 5. FORMULAIRE

### Champ Nom de la partition
| Propriété | Valeur |
|-----------|--------|
| Label | "Nom de la partition" |
| Type | OdsInput |
| Exemple | "backup-data" |
| Aide | "Lettres, chiffres, tirets uniquement. 3-32 caractères." |

### Champ Taille
| Propriété | Valeur |
|-----------|--------|
| Label | "Taille (Go)" |
| Type | OdsInput (number) avec spinner |
| Min | 10 Go |
| Max | 3000 Go |
| Aide | "Min: 10 Go - Max: 3000 Go" |

### Champ Description
| Propriété | Valeur |
|-----------|--------|
| Label | "Description (optionnel)" |
| Type | OdsInput |
| Placeholder | "Description de la partition..." |
| Aide | "255 caractères maximum" |

## 6. SÉLECTION PROTOCOLE

### Option NFS (sélectionnée)
| Propriété | Valeur |
|-----------|--------|
| Style | Carte avec bordure bleue |
| Radio | Sélectionné |
| Titre | "NFS" |
| Description | "Network File System - Compatible Linux/Unix" |

### Option CIFS
| Propriété | Valeur |
|-----------|--------|
| Style | Carte avec bordure grise |
| Radio | Non sélectionné |
| Titre | "CIFS" |
| Description | "Common Internet File System - Compatible Windows" |

## 7. FOOTER
| Bouton | Style | Action |
|--------|-------|--------|
| Annuler | Secondary | Retour |
| Créer | Primary (bleu) | Crée la partition |

## 8. APIS & ENDPOINTS
- GET /dedicated/nasha/{serviceName} - Infos NAS-HA
- POST /dedicated/nasha/{serviceName}/partition - Créer partition
