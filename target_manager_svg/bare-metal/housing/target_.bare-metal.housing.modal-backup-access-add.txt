# Cahier des Charges : bare-metal/housing - Ajouter accès backup

## 1. VUE D'ENSEMBLE
- **Page** : Ajout d'un accès
- **Type** : Modal formulaire
- **Objectif** : Ajouter un accès IP au backup storage

## 2. STRUCTURE MODAL
- Header avec titre + bouton fermer
- Texte explicatif
- Sélection bloc IP
- Checkboxes protocoles
- Message info
- Résumé droits
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Modal principale
- **OdsSelect** : Bloc IP concerné
- **OdsCheckbox** : CIFS, FTP, NFS
- **OdsMessage** : Info (bleu)
- **OdsText** : Résumé droits appliqués
- **OdsButton** : Annuler (outline), Ajouter (primary)

## 4. SÉLECTION BLOC IP
| Propriété | Valeur |
|-----------|--------|
| Label | Block concerné : |
| Type | Select |
| Options | Liste des blocs IP du housing |

## 5. PROTOCOLES
| Checkbox | Description |
|----------|-------------|
| CIFS | Common Internet File System (Windows) |
| FTP | File Transfer Protocol |
| NFS | Network File System (Linux) |

## 6. MESSAGE INFO
| Propriété | Valeur |
|-----------|--------|
| Type | Info (bleu) |
| Icône | ℹ |
| Texte 1 | Vous devez au moins autoriser un protocole car par défaut, les blocks d'IP n'ayant pas de règle sont refusées. |
| Texte 2 | La modification peut prendre quelques minutes. |

## 7. RÉSUMÉ
Block 123.45.67.0/24 → CIFS ✓ | FTP ✓ | NFS ✗

## 8. BOUTONS
| Bouton | Style | Position |
|--------|-------|----------|
| Annuler | Outline | Gauche |
| Ajouter | Primary | Droite |

## 9. APIS & ENDPOINTS
- POST /dedicated/housing/{serviceName}/features/backupFTP/access
  - Paramètres : ipBlock (string), cifs (boolean), ftp (boolean), nfs (boolean)
