# Cahier des Charges : dedicated - Onglet Interventions

## 1. VUE D'ENSEMBLE
- **Page** : Interventions serveur dédié
- **Type** : Historique avec demande
- **Objectif** : Gérer les interventions matérielles datacenter

## 2. STRUCTURE PAGE
- Carte demande intervention avec bouton
- Tableau historique interventions

## 3. COMPOSANTS UI
- **OdsCard** : Container sections
- **OdsButton** : Nouvelle intervention
- **OdsDatagrid** : Tableau historique
- **OdsBadge** : Statut intervention

## 4. HEADER SECTION
| Élément | Valeur |
|---------|--------|
| Titre | Demander une intervention |
| Description | Intervention matérielle en datacenter (remplacement disque, RAM, etc.) |
| Bouton | Nouvelle intervention |

## 5. COLONNES TABLEAU
| Colonne | Type | Description |
|---------|------|-------------|
| N° Ticket | Link | Référence intervention |
| Date | Date | Date de demande |
| Type | Text | Type d'intervention |
| Description | Text muted | Détail du problème |
| Statut | Badge | État intervention |
| Actions | Link | Détails |

## 6. TYPES D'INTERVENTION
| Type | Description |
|------|-------------|
| Remplacement disque | Disque défaillant |
| Ajout RAM | Extension mémoire |
| Diagnostic matériel | Vérification composants |
| Remplacement alimentation | PSU défaillante |
| Remplacement carte mère | Motherboard |

## 7. BADGES STATUT
| Statut | Couleur |
|--------|---------|
| Terminé | Vert |
| En cours | Bleu |
| Planifié | Orange |

## 9. APIS & ENDPOINTS
- GET /dedicated/server/{serviceName}/intervention - Liste interventions
- POST /dedicated/server/{serviceName}/intervention - Nouvelle demande
- GET /dedicated/server/{serviceName}/intervention/{interventionId} - Détail
