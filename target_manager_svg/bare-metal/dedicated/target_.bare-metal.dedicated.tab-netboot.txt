# Cahier des Charges : bare-metal/dedicated - Onglet Netboot

## 1. VUE D'ENSEMBLE
- **Onglet** : Netboot
- **Type** : Page formulaire configuration boot
- **Objectif** : Configurer le mode de démarrage du serveur

## 2. STRUCTURE PAGE
- Breadcrumb + titre
- Onglets navigation
- Titre section + info netboot actuel
- Formulaire dans une carte

## 3. COMPOSANTS UI
- **OdsTabs** : Navigation
- **OdsCard** : Conteneur formulaire
- **OdsRadio** : Options de boot (3)
- **OdsSelect** : Rescue disponible, Email
- **OdsMessage** : Info rescue (bleu)
- **OdsButton** : Annuler (outline), Suivant (primary)

## 4. INFO NETBOOT ACTUEL
- **Texte** : "Netboot actuellement configuré sur votre serveur ns1234567.ip-12-34-56.eu :"
- **Valeur** : "Disque dur" (lien)

## 5. OPTIONS DE BOOT
| Option | Description | État |
|--------|-------------|------|
| Booter sur le disque dur | Boot normal | ✅ Sélectionné |
| Booter en mode rescue | Mode dépannage | - |
| Booter en mode network | PXE/iPXE | - |

## 6. MESSAGE INFO RESCUE
Fond bleu clair :
- "ℹ Le rescue peut être utile pour diagnostiquer des dysfonctionnements matériels sur un serveur dédié."
- Lien : "Accéder au guide de diagnostic matériel."

## 7. CHAMPS RESCUE
| Champ | Valeur |
|-------|--------|
| Rescue disponible | rescue-customer ▼ |
| Email identifiants | ab12345-ovh@ovh.net ▼ |

## 8. BOUTONS
- **Annuler** : Outline, annule les modifications
- **Suivant** : Primary, applique la configuration

## 9. APIS & ENDPOINTS
- GET /dedicated/server/{serviceName}/boot
  - Liste des modes de boot disponibles
- GET /dedicated/server/{serviceName}/boot/{bootId}
  - Détails d'un mode de boot
- PUT /dedicated/server/{serviceName}
  - Body: { bootId }
  - Change le mode de boot
- POST /dedicated/server/{serviceName}/reboot
  - Redémarre avec le nouveau boot
