# Cahier des Charges : bare-metal/dedicated - Modal Configuration Monitoring

## 1. VUE D'ENSEMBLE
- **Modal** : Configuration du monitoring
- **Type** : Modal formulaire complet
- **Objectif** : Configurer la surveillance OVH du serveur

## 2. STRUCTURE MODAL
- Modal blanche (700x680px)
- Header avec titre + bouton fermer
- Toggle activation monitoring
- Services à surveiller (checkboxes)
- Seuils d'alerte (inputs)
- Notifications (email + SMS)
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur
- **OdsToggle** : Activation monitoring
- **OdsCheckbox** : Services (6)
- **OdsInput** : Seuils (CPU, Mémoire, Disque, Temps réponse)
- **OdsInput** : Email
- **OdsCheckbox** : Alertes SMS
- **OdsButton** : Annuler (outline), Enregistrer (primary)

## 4. HEADER
- **Titre** : "Configuration du monitoring"
- **Bouton fermer** : ×

## 5. ACTIVATION MONITORING
- **Label** : "Monitoring OVH"
- **Description** : "OVH surveillera votre serveur et vous alertera en cas de problème"
- **Toggle** : ✅ Activé (ON)

## 6. SERVICES À SURVEILLER
| Service | État |
|---------|------|
| Ping (ICMP) | ✅ Coché |
| SSH (port 22) | ✅ Coché |
| HTTP (port 80) | ☐ Non coché |
| HTTPS (port 443) | ✅ Coché |
| SMTP (port 25) | ☐ Non coché |
| Port personnalisé | ☐ Non coché |

## 7. SEUILS D'ALERTE
| Métrique | Seuil | Unité |
|----------|-------|-------|
| CPU | 90 | % |
| Mémoire | 85 | % |
| Disque | 80 | % |
| Temps de réponse | 5000 | ms |

## 8. NOTIFICATIONS
- **Email** : admin@example.com
- **Bouton** : "+ Ajouter un email"
- **Checkbox** : ✅ Recevoir un SMS en cas d'alerte critique

## 9. BOUTONS
- **Annuler** : Outline, ferme la modal
- **Enregistrer** : Primary, sauvegarde la configuration

## 10. APIS & ENDPOINTS
- GET /dedicated/server/{serviceName}/serviceMonitoring
  - Configuration actuelle du monitoring
- PUT /dedicated/server/{serviceName}/serviceMonitoring
  - Body: { enabled, services[], thresholds, notifications }
  - Met à jour la configuration
