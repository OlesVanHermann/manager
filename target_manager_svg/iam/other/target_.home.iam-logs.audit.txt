# Cahier des Charges : home/iam - Logs Audit

## 1. VUE D'ENSEMBLE
- **Type** : Page logs audit sécurité
- **URL** : /iam/logs/audit
- **Objectif** : Voir actions sécurité compte

## 2. STRUCTURE PAGE
- Titre + bouton Guides
- Onglets (actif: Audit)
- Description multi-lignes
- Console live tail
- Lien Data Streams

## 3. COMPOSANTS UI
- **OdsTabs** : Types de logs
- **OdsCard** : Console terminal
- **OdsLink** : Data Streams

## 4. ONGLETS
| Onglet | Active |
|--------|--------|
| Logs des politiques d'accès | Non |
| Logs d'Activités | Non |
| Logs d'Audit | Oui |

## 5. DESCRIPTION
L1: "Vous trouverez ci-dessous les logs en temps réel des actions liées à la sécurité de votre compte"
L2: "client : action de connexion des utilisateurs, changement de mot de passe, création et"
L3: "modification d'utilisateurs et de groupes d'utilisateurs."

## 6. TYPES D'ÉVÉNEMENTS AUDIT
| Type | Couleur | Description |
|------|---------|-------------|
| [LOGIN] | #60A5FA bleu | Connexion réussie |
| [LOGIN_FAIL] | #F87171 rouge | Échec connexion |
| [LOGOUT] | #60A5FA bleu | Déconnexion |
| [PASSWORD] | #F59E0B orange | Changement MDP |
| [USER_CREATE] | #34D399 vert | Création utilisateur |
| [USER_DISABLE] | #FB923C orange | Désactivation |
| [GROUP_UPDATE] | #A78BFA violet | Modification groupe |

## 7. FORMAT LOG AUDIT
```
{timestamp} [{EVENT_TYPE}] user:{user} ip:{ip} / by:{admin} target:{user}
```

| Label | Destination |
|-------|-------------|
| "Configurer les Data Streams →" | /iam/logs/data-streams |

## 9. APIS & ENDPOINTS
- GET /me/iam/logs/audit - Stream logs audit
- WebSocket /me/iam/logs/audit/live - Temps réel
