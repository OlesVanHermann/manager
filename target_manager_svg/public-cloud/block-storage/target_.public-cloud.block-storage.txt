# Cahier des Charges : public-cloud/block-storage - Liste des Volumes

## 1. VUE D'ENSEMBLE
- **Page** : Block Storage - Liste des volumes
- **Type** : Dashboard avec datagrid
- **Objectif** : Gérer les volumes de stockage bloc

## 2. STRUCTURE PAGE
- Header OVHcloud
- Sidebar (Block Storage actif dans STORAGE)
- Breadcrumb
- Titre + bouton créer
- Barre de recherche
- Datagrid volumes

## 3. COMPOSANTS UI
- **OdsBreadcrumb** : Navigation
- **OdsButton** : Créer volume
- **OdsInput** : Recherche
- **OdsDatagrid** : Liste volumes
- **OdsBadge** : Type, Statut
- **OdsActionMenu** : Menu contextuel

## 4. BREADCRUMB
"Public Cloud / Mon Projet Cloud / Block Storage"

## 5. TITRE ET BOUTON
- **Titre** : "Block Storage"
- **Bouton** : "+ Créer un volume" (primary bleu)

## 6. COLONNES DATAGRID

| Colonne | Type | Description |
|---------|------|-------------|
| Nom | Text link bleu | Nom du volume |
| ID | Text small | UUID tronqué |
| Localisation | Text | Région (GRA9, SBG5...) |
| Type | Badge | Classic (bleu) / High-speed (orange) |
| Taille | Text | Capacité en Go/To |
| Instance attachée | Text link / "-" | Nom instance ou vide |
| Statut | Badge | in-use (vert) / available (gris) |
| Actions | ActionMenu | ⋮ |

## 7. DONNÉES EXEMPLE

| Nom | ID | Localisation | Type | Taille | Instance | Statut |
|-----|-----|--------------|------|--------|----------|--------|
| volume-data-prod | a1b2c3d4... | GRA9 | High-speed | 500 Go | web-server-prod | in-use |
| backup-volume | f7g8h9i0... | SBG5 | Classic | 1 To | - | available |
| db-storage | m3n4o5p6... | GRA11 | High-speed | 200 Go | db-server-01 | in-use |

## 8. BADGES

| Type | Couleur |
|------|---------|
| Classic | Bleu (#DBEAFE) |
| High-speed | Orange (#FEF3C7) |
| in-use | Vert (#DEF7EC) |
| available | Gris (#E5E7EB) |

## 9. ACTION MENU

| Option | Style |
|--------|-------|
| Modifier | Normal |
| Attacher à une instance | Normal |
| Détacher | Normal |
| Créer un snapshot | Normal |
| Supprimer | Danger rouge |

## 10. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/volume
- POST /cloud/project/{projectId}/volume
- DELETE /cloud/project/{projectId}/volume/{volumeId}
- POST /cloud/project/{projectId}/volume/{volumeId}/attach
- POST /cloud/project/{projectId}/volume/{volumeId}/detach
