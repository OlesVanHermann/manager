# Cahier des Charges : public-cloud/volume-snapshot - Créer Volume

## 1. VUE D'ENSEMBLE
- **Page** : Créer un volume depuis le snapshot
- **Type** : Page formulaire
- **Objectif** : Créer un nouveau volume à partir d'un snapshot existant

## 2. STRUCTURE PAGE
- Header OVHcloud
- Sidebar (Volume Snapshot actif)
- Breadcrumb
- Titre
- Carte formulaire
- Boutons d'action

## 3. COMPOSANTS UI
- **OdsBreadcrumb** : Navigation
- **OdsCard** : Conteneur formulaire
- **OdsMessage** : Info snapshot source (bleu)
- **OdsInput** : Nom du volume, Taille
- **OdsButton** : Annuler (secondary), Créer le volume (primary)

## 4. HEADER
- **Titre** : "Créer un volume depuis le snapshot"

## 5. INFORMATIONS SNAPSHOT SOURCE
Fond bleu clair avec infos :
| Champ | Valeur |
|-------|--------|
| Snapshot | snap-prod-daily |
| Région | GRA11 |
| Taille | 100 GB |
| Type | high-speed |

## 6. FORMULAIRE

### Nom du volume
- **Input** : Texte
- **Valeur** : "snap-prod-daily-restored"

### Taille du volume (GB)
- **Input** : Nombre
- **Valeur** : 100
- **Helper** : "Minimum: 100 GB (taille du snapshot)"

## 7. ESTIMATION COÛT
- **Titre** : "Estimation du coût"
- **Prix** : "**0,04 €** /heure soit environ **29,20 €** /mois HT"

## 8. BOUTONS
- Annuler (secondary)
- Créer le volume (primary)

## 9. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/volume/snapshot/{snapshotId}
  - Détails du snapshot source
- POST /cloud/project/{projectId}/volume
  - Body: { name, size, snapshotId }
  - Crée le volume depuis le snapshot
