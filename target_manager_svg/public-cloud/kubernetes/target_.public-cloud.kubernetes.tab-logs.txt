# Cahier des Charges : public-cloud/kubernetes - Onglet Logs

## 1. VUE D'ENSEMBLE
- **Page** : Onglet Logs du cluster (audit logs)
- **Type** : Page avec empty state et intégration LDP
- **Objectif** : Configurer les flux de logs vers Logs Data Platform

## 2. STRUCTURE PAGE
- Header avec nom cluster et badge READY
- Onglets : Service, Restrictions, Node pools, Logs (actif)
- Description des audit logs
- Lien documentation
- Banner info bleu (avantages LDP)
- Section "Flux de données"
- Empty state ou liste des flux

## 3. COMPOSANTS UI
- **OdsTabs** : Navigation onglets
- **OdsMessage** : Banner info (bleu clair)
- **OdsButton** : "+ Ajouter un flux de données"
- **OdsCard** : Empty state
- **OdsLink** : "En savoir plus ↗"

## 4. DESCRIPTION
"Vous trouverez ci-dessous les journaux d'audit de votre cluster Kubernetes fournissant un ensemble chronologique d'enregistrements pertinents pour la sécurité et documentant la séquence d'actions dans votre cluster."

Lien : "En savoir plus ↗"

## 5. BANNER INFO
- Fond : Bleu clair (#E0F2FE)
- Bordure : Bleu (#0284C7)
- Contenu :
  - "ℹ En vous abonnant aux audit logs de votre cluster Kubernetes, ils seront transférés dans Logs Data Platform."
  - "Vous pourrez les centraliser avec vos autres logs, effectuer des recherches avancées, gérer leurs rétentions et définir des alertes."

## 6. SECTION FLUX DE DONNÉES
- Titre : "Flux de données"
- Bouton : "+ Ajouter un flux de données"

## 7. EMPTY STATE
- Fond : Blanc avec bordure grise
- Texte : "Aucun flux de données configuré"
- Sous-texte : "Ajoutez un flux de données pour commencer à collecter les logs de votre cluster Kubernetes."

## 8. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/kube/{kubeId}/log/subscription
  - Liste des abonnements logs
- POST /cloud/project/{projectId}/kube/{kubeId}/log/subscription
  - Créer un abonnement
