# Cahier des Charges : public-cloud/ai-apps - Modal Synchronisation des données

## 1. VUE D'ENSEMBLE
- **Modal** : Synchronisation manuelle des données
- **Type** : Modal formulaire avec radio buttons
- **Objectif** : Synchroniser manuellement les données entre l'app et les conteneurs

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Type de synchronisation (radio cards Pull/Push)
- Bandeau info (répertoire de montage)
- Bandeau warning (durée opération)
- Boutons Annuler / Synchroniser

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (600x440)
- **OdsRadio** : Pull/Push (style card)
- **OdsMessage** : Info bleu, Warning orange
- **OdsButton** : Annuler, Synchroniser

## 4. TITRE
"Synchronisation manuelle des données"

## 5. TYPE DE SYNCHRONISATION
Description: "Vous pouvez choisir entre une synchronisation de type push, vers votre conteneur de données ou de type pull, dans l'autre sens."

| Option | Description | État |
|--------|-------------|------|
| Pull | Depuis le conteneur vers l'app | Sélectionné (bordure bleue, fond bleu clair) |
| Push | Depuis l'app vers le conteneur | Non sélectionné |

## 6. BANDEAU INFO
Fond bleu (#E8F0FE) :
- "ℹ️ La synchronisation sera effectuée sur le répertoire de montage"
- "/workspace/data (model-weights)" (bold)

## 7. BANDEAU WARNING
Fond jaune (#FEF3C7) :
- "⚠️ Attention"
- "La synchronisation peut prendre plusieurs minutes selon la taille"
- "des données. L'application restera disponible pendant ce temps."

## 8. BOUTONS
| Bouton | Style | Action |
|--------|-------|--------|
| Annuler | Outline | Ferme la modal |
| Synchroniser | Primaire bleu | Lance la synchronisation |

## 9. APIS & ENDPOINTS
- POST /cloud/project/{projectId}/ai/app/{appId}/datasync
  - type: "push" ou "pull"
  - alias: string (nom du volume)
