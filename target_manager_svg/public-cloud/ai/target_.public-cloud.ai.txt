# Cahier des Charges : public-cloud/ai - Dashboard AI Platform

## 1. VUE D'ENSEMBLE
- **Page** : Dashboard AI Platform (vue unifiée)
- **Type** : Page tableau de bord avec onglets
- **Objectif** : Vue d'ensemble de toutes les ressources AI (Notebooks, Jobs, Apps)

## 2. STRUCTURE PAGE
- Header OVH
- Sidebar
- Breadcrumb
- Titre + description
- Onglets (Notebooks actif, Jobs, Apps)
- Cartes statistiques (3 KPI)
- Section Notebooks avec cartes

## 3. COMPOSANTS UI
- **OdsBreadcrumb** : Navigation
- **OdsTabs** : Notebooks, Jobs, Apps
- **OdsCard** : 3 cartes KPI, cartes notebooks
- **OdsButton** : Nouveau notebook (primaire)
- **OdsBadge** : Statuts (RUNNING, STOPPED)

## 4. TITRE
"AI Platform"
- Sous-titre: "Machine Learning et entrainement de modeles"

## 5. ONGLETS
| Onglet | État |
|--------|------|
| Notebooks | Actif (souligné bleu) |
| Jobs | Inactif |
| Apps | Inactif |

## 6. CARTES KPI
3 cartes statistiques côte à côte:

| Titre | Valeur | Icône (couleur fond) |
|-------|--------|---------------------|
| Notebooks | 5 | Violet (#EDE9FE) |
| Jobs | 12 | Jaune (#FEF3C7) |
| Apps deployées | 3 | Vert (#D1FAE5) |

## 7. SECTION NOTEBOOKS
- Titre: "Notebooks" avec badge compteur (5)
- Bouton: "Nouveau notebook" (primaire bleu)

### Cartes Notebooks (3 colonnes)
Format carte:
- Bande couleur en haut (vert=running, gris=stopped)
- Indicateur statut (cercle)
- Nom du notebook
- Détails: "JupyterLab | GPU A100"
- Badge statut

### Exemples
| Nom | Éditeur/GPU | Statut |
|-----|-------------|--------|
| data-exploration | JupyterLab / GPU A100 | RUNNING |
| model-training-v2 | JupyterLab / GPU V100 | RUNNING |
| test-env | JupyterLab / CPU | STOPPED |

## 8. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/ai/notebook
  - Liste des notebooks avec statut
- GET /cloud/project/{projectId}/ai/job
  - Liste des jobs
- GET /cloud/project/{projectId}/ai/app
  - Liste des apps déployées
