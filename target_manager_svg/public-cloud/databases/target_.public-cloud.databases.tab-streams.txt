# Cahier des Charges : public-cloud/databases - Onglet Flux de données

## 1. VUE D'ENSEMBLE
- **Page** : Databases - Onglet Flux de données
- **Type** : Page onglet listing
- **Objectif** : Configurer les flux de données vers Logs Data Platform

## 2. STRUCTURE PAGE
- Header OVH
- Sidebar avec service actif
- Breadcrumb avec "Flux"
- Onglets (Flux actif)
- Titre "Flux de données"
- Bannière informative bleue
- Bouton ajouter flux
- Zone vide (si aucun flux)

## 3. COMPOSANTS UI
- **OdsBreadcrumb** : Navigation
- **OdsTabs** : Onglets avec Flux actif
- **OdsMessage** : Bannière info (bleue)
- **OdsButton** : "+ Ajouter un flux de données" (outline)
- **OdsCard** : Zone vide ou tableau

## 4. TITRE
"Flux de données"

## 5. BANNIÈRE INFO
- Fond: Bleu clair (#EFF6FF)
- Bordure: Bleu (#3B82F6)
- Contenu:
  - "En vous abonnant aux audit logs de votre service, ils seront transférés dans Logs Data Platform et vous pourrez les centraliser avec vos autres logs, effectuer des recherches avancées, gérer leurs rétentions, définir des alertes ainsi que des tableaux de bord pour observer facilement vos clusters."

## 6. ÉTAT VIDE
- Fond: Blanc
- Bordure: Grise (#E5E7EB)
- Texte centré:
  - "Aucun flux de données configuré"
  - "Ajoutez un flux pour commencer à centraliser vos logs"

## 7. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/database/service/{serviceId}/logSubscription
  - Liste des flux de données
- POST /cloud/project/{projectId}/database/service/{serviceId}/logSubscription
  - Body: { ldpServiceId, streamId }
  - Créer un flux
- DELETE /cloud/project/{projectId}/database/service/{serviceId}/logSubscription/{subscriptionId}
  - Supprimer un flux
