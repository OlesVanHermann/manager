# Cahier des Charges : public-cloud/databases - Modal Supprimer nœud

## 1. VUE D'ENSEMBLE
- **Modal** : Supprimer un nœud
- **Type** : Modal confirmation avec estimation
- **Objectif** : Supprimer un nœud du cluster avec avertissement

## 2. STRUCTURE MODAL
- Overlay sombre
- Conteneur blanc 500x340px
- Header avec titre et bouton fermer
- Toggle affichage prix (Heure/Mois)
- Message confirmation avec économie
- Warning box rouge
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur
- **OdsToggle** : Affichage prix (Heure/Mois)
- **OdsText** : Message et économie
- **OdsMessage** : Warning box rouge
- **OdsButton** : Annuler, Valider (danger)

## 4. TITRE
"Supprimer un nœud"

## 5. TOGGLE AFFICHAGE PRIX
| Option | État |
|--------|------|
| Heure | Inactif |
| Mois | Actif (bleu) |

## 6. MESSAGE PRINCIPAL
"Vous disposez actuellement de **{count} nœud(s)**.
Voulez-vous vraiment en supprimer un ?

Vous réduirez votre facture de **{price} € HT** par mois."

### Variables
- `{count}` : Nombre actuel de nœuds (ex: 3)
- `{price}` : Économie mensuelle (ex: 45,00) - affiché en vert

## 7. WARNING BOX
- Fond: Rouge clair (#FEF2F2)
- Texte rouge (#991B1B):
  - "⚠ Attention : cette action est irréversible."
  - "Les données du nœud seront définitivement supprimées."

## 8. BOUTONS
| Bouton | Style | Position | Action |
|--------|-------|----------|--------|
| Annuler | Outline | Gauche | Fermer modal |
| Valider | Danger rouge | Droite | DELETE node |

## 9. CONTRAINTES
- Minimum de nœuds selon le plan (Essential: 1, Business: 3)
- Vérification avant suppression

## 10. APIS & ENDPOINTS
- DELETE /cloud/project/{projectId}/database/service/{serviceId}/node/{nodeId}
  - Supprimer un nœud
