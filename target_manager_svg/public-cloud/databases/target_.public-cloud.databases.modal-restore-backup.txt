# Cahier des Charges : public-cloud/databases - Modal Restaurer backup

## 1. VUE D'ENSEMBLE
- **Modal** : Restaurer votre backup
- **Type** : Modal formulaire
- **Objectif** : Restaurer le service à partir d'un point de restauration

## 2. STRUCTURE MODAL
- Overlay sombre
- Conteneur blanc 500x420px
- Header avec titre et avertissement
- Options de point de restauration (radio)
- Sélecteur de backup
- Footer avec bouton

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur
- **OdsText** : Avertissement perte de données
- **OdsRadio** : Point de restauration
- **OdsSelect** : Backup à restaurer
- **OdsButton** : Restaurer

## 4. TITRE
"Restaurer votre backup"

## 5. AVERTISSEMENT
"La restauration d'une sauvegarde écrase les informations présentes actuellement sur votre cluster. Cette opération peut provoquer une perte de données."

## 6. CHAMPS DU FORMULAIRE

### Point de restauration
| Option | Description |
|--------|-------------|
| Le plus récent | Point-in-time recovery le plus récent |
| Date spécifique | Restaurer à une date/heure précise |
| Backup | Sélectionner une sauvegarde existante (sélectionné) |

### Sélecteur Backup
| Champ | Type | Valeur exemple |
|-------|------|----------------|
| Backup | Select | 2024-12-15 10:30:00 (1.2 GB) |

### Format des options backup
- Date et heure de création
- Taille du backup

## 7. BOUTONS
| Bouton | Style | Position | Action |
|--------|-------|----------|--------|
| Restaurer | Primary | Droite | POST restore |

## 8. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/database/service/{serviceId}/backup
  - Liste des backups disponibles
- POST /cloud/project/{projectId}/database/service/{serviceId}/backup/{backupId}/restore
  - Restaurer à partir d'un backup
- POST /cloud/project/{projectId}/database/service/{serviceId}/restore
  - Body: { datetime? }
  - Point-in-time recovery
