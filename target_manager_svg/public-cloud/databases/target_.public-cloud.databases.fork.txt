# Cahier des Charges : public-cloud/databases - Page Fork

## 1. VUE D'ENSEMBLE
- **Page** : Databases - Dupliquer votre service de base de données (fork)
- **Type** : Page formulaire avec récapitulatif
- **Objectif** : Créer une copie du service à partir d'un point de restauration

## 2. STRUCTURE PAGE
- Header OVH
- Sidebar avec Bases de données actif
- Breadcrumb avec "Fork"
- Titre "Dupliquer votre service de base de données (fork)"
- Description explicative
- Formulaire 2 colonnes (formulaire + récapitulatif)

## 3. COMPOSANTS UI
- **OdsBreadcrumb** : Navigation
- **OdsCard** : Sections du formulaire
- **OdsRadio** : Point de restauration
- **OdsSelect** : Type de service/version
- **OdsInput** : Nom du service
- **OdsTile** : Sélecteur datacentre, offre, instance
- **OdsQuantity** : Nombre de nœuds, stockage
- **OdsButton** : "Commander" (primary)

## 4. TITRE
"Dupliquer votre service de base de données (fork)"

## 5. DESCRIPTION
"Le fork de votre sauvegarde s'effectuera sur un nouveau cluster."

| **Estimation** | ~150,00 € HT/mois |

## 8. BOUTONS
| Bouton | Style | Action |
|--------|-------|--------|
| Commander | Primary bleu | POST fork |

## 9. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/database/service/{serviceId}/backup
  - Liste des backups disponibles
- POST /cloud/project/{projectId}/database/service/{serviceId}/fork
  - Body: { restorePoint, engine, version, plan, region, flavor, nodeCount, additionalStorage, name }
  - Créer le fork
