# Cahier des Charges : public-cloud/cold-archive - Gestion Utilisateurs S3

## 1. VUE D'ENSEMBLE
- **Page** : Cold Archive - Gestion des credentials Object Storage
- **Type** : Page listing
- **Objectif** : Gérer les utilisateurs S3 pour Cold Archive

## 2. STRUCTURE PAGE
- Header OVH
- Sidebar avec "Utilisateurs S3" actif
- Titre et description
- Bouton ajouter utilisateur
- Barre recherche/filtres
- Tableau des utilisateurs
- ActionMenu ouvert avec dropdown
- Pagination

## 3. COMPOSANTS UI
- **OdsButton** : "+ Ajouter un utilisateur" (primaire)
- **OdsButton** : Refresh (↻) (outline)
- **OdsInput** : Recherche
- **OdsButton** : "Filtrer" (outline)
- **OdsTable** : Liste des utilisateurs
- **OdsActionMenu** : Actions par utilisateur

## 4. TITRE
"Gestion des credentials Object Storage"

## 5. DESCRIPTION
"Dans cette section, vous pouvez ajouter un nouvel utilisateur Object Storage, obtenir votre clé d'accès et votre clé secrète, et gérer les politiques d'accès."

## 6. TABLEAU UTILISATEURS
### Colonnes
| Colonne | Type | Description |
|---------|------|-------------|
| Nom d'utilisateur | Texte | Identifiant utilisateur |
| Date de création | Date | Format: JJ/MM/AAAA HH:MM |
| Description | Texte | Description de l'utilisateur |
| Clé d'accès | Monospace | Access Key (tronquée) |
| Actions | Menu ⋮ | ActionMenu |

### Données exemple
| Nom | Date création | Description | Clé d'accès |
|-----|---------------|-------------|-------------|
| user-s3-abc123 | 27/12/2024 14:30 | Cold Archive User | AKIAIOSFODNN7EXAMPLE |
| backup-user-xyz | 15/11/2024 09:15 | Backup automation | BKIAIOSFODNN8BACKUP |
| archive-reader | 01/10/2024 11:00 | Read-only access | CKIAIOSFODNN9READER |

## 7. ACTION MENU (ouvert)
| Action | Type |
|--------|------|
| Importer une politique d'accès (JSON) | Standard |
| Télécharger la politique d'accès (JSON) | Standard |
| Télécharger le fichier rclone | Standard |
| Voir la clé secrète | Standard |
| Supprimer | Danger (rouge) |

## 8. PAGINATION
"1-3 sur 3 résultats"

## 9. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/user
  - Liste des utilisateurs S3
- POST /cloud/project/{projectId}/user
  - Créer un utilisateur
- GET /cloud/project/{projectId}/user/{userId}/s3Credentials
  - Récupérer les credentials S3
- DELETE /cloud/project/{projectId}/user/{userId}
  - Supprimer un utilisateur
