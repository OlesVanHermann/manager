# Cahier des Charges : public-cloud/rancher - Créer

## 1. VUE D'ENSEMBLE
- **Page** : Créer un Managed Rancher Service
- **Type** : Formulaire wizard en 3 étapes
- **Objectif** : Créer un nouveau service Rancher managé

## 2. STRUCTURE PAGE
- Header OVHcloud
- Sidebar (Managed Rancher actif sous Containers)
- Breadcrumb
- Titre "Créer un Managed Rancher Service"
- Étapes numérotées (1-2-3)
- Formulaire
- Boutons footer

## 3. COMPOSANTS UI
- **OdsBreadcrumb** : Navigation
- **OdsStepper** : 3 étapes avec cercles numérotés
- **OdsInput** : Nom du service
- **OdsSelect** : Version Rancher
- **OdsTile** : Cartes offres avec radio
- **OdsButton** : Annuler (secondary), Créer (primary)

## 4. ÉTAPE 1 : NOM DU SERVICE
- **Label** : "Nom"
- **Input** : Champ texte
- **Placeholder** : "mon-rancher-service"
- **Helper** : "Le nom doit comporter entre 3 et 63 caractères alphanumériques."

## 5. ÉTAPE 2 : VERSION DE RANCHER
- **Select** : Dropdown avec versions disponibles
- **Valeur** : "v2.8.5 (dernière version stable)"

## 6. ÉTAPE 3 : OFFRE

### Standard
- **Description** : "Offre gratuite d'entrée de gamme"
- **Features** :
  - Jusqu'à 15 clusters downstream
  - Support communautaire
- **Prix** : Gratuit

### OVHcloud Edition (sélectionné)
- **Description** : "Pour les environnements de production"
- **Features** :
  - Clusters illimités
  - SLA 99.9%, support 24/7
- **Prix** : 0,05 €/vCPU/heure

## 7. BOUTONS
- Annuler (secondary)
- Créer (primary)

## 8. APIS & ENDPOINTS
- POST /cloud/project/{projectId}/rancher
  - Body: { name, version, plan }
  - Crée le service Rancher
- GET /cloud/project/{projectId}/rancher/version
  - Liste des versions disponibles
