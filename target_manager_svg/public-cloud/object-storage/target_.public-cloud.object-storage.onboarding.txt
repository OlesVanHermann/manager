# Cahier des Charges : public-cloud/object-storage - Onboarding

## 1. VUE D'ENSEMBLE
- **Page** : Object Storage - Onboarding (empty state)
- **Type** : Page d'accueil avant création
- **Objectif** : Présenter Object Storage et inciter à créer

## 2. STRUCTURE PAGE
- Header OVHcloud
- Sidebar (Object Storage actif)
- Breadcrumb
- Zone centrale avec illustration
- Titre et description
- Liste fonctionnalités
- CTA créer
- Cartes classes de stockage

## 3. COMPOSANTS UI
- **OdsBreadcrumb** : Navigation
- **OdsEmptyState** : Conteneur principal
- **OdsIcon** : ☁️ illustration
- **OdsText** : Titre, description
- **OdsList** : Fonctionnalités avec checkmarks
- **OdsButton** : "Créer un conteneur" (primary)
- **OdsCard** : Cartes classes de stockage

## 4. ILLUSTRATION
- Zone bleu clair avec icône cloud

## 5. TITRE ET DESCRIPTION
- **Titre** : "Object Storage"
- **Description** : "Hébergez votre catalogue de données dans notre solution de stockage d'objets compatible API S3."

## 6. FONCTIONNALITÉS
Liste à deux colonnes :
- ✓ Compatible API S3
- ✓ Stockage illimité
- ✓ Versioning des objets
- ✓ Classes de stockage multiples
- ✓ Réplication cross-région
- ✓ Chiffrement des données

## 7. CTA
- **Bouton** : "Créer un conteneur" (primary, 300px)

## 8. CLASSES DE STOCKAGE

| Classe | Badge | Description |
|--------|-------|-------------|
| Standard | Bleu | Accès fréquent aux données. Idéal pour contenus web, médias |
| Standard - Accès peu fréquent | Jaune | Données rarement consultées. Coût réduit, frais de récupération |
| Haute Performance | Vert | Très faible latence. Idéal pour applications critiques |

## 9. APIS & ENDPOINTS
- GET /cloud/project/{projectId}/storage
  - Vérifie si des conteneurs existent (pour afficher onboarding ou liste)
