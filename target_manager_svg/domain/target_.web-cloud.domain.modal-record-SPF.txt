# STRUCTURE MODAL
1. HEADER - Titre "Ajouter une entrée DNS - SPF" + badge SPF (fond OVH bleu)
2. CONTENT
   - Bannière info SPF
   - Sous-domaine + suffixe
   - TTL
   - Mécanismes SPF (checkboxes)
   - Includes additionnels (textarea)
   - Politique en cas d'échec (select)
   - Aperçu de l'enregistrement
3. FOOTER - Boutons Annuler / Ajouter l'entrée

# MODAL TYPE
- Type: Formulaire SPF assisté
- Dimensions: 700x640px
- Header: Fond OVH bleu (#000E9C)
- Badge: Bleu pour type SPF

# CHAMPS
| Champ | Type | Obligatoire | Description |
|-------|------|-------------|-------------|
| Sous-domaine | text | Non | @ pour racine |
| TTL | number | Non | Durée de cache |
| Mécanismes | checkboxes | Oui | mx, a, ip4, ip6, include |
| Includes | textarea | Non | Domaines tiers |
| Politique | select | Oui | -all, ~all, ?all, +all |

# MÉCANISMES SPF
| Mécanisme | Description |
|-----------|-------------|
| mx | Autoriser serveurs MX |
| a | Autoriser IP du domaine |
| ip4:x.x.x.x | Autoriser IPv4 spécifique |
| ip6:xxxx::xx | Autoriser IPv6 spécifique |
| include:domain | Inclure SPF externe |

# POLITIQUES (-all vs ~all)
| Politique | Symbole | Action |
|-----------|---------|--------|
| Fail | -all | Rejeter (strict) |
| SoftFail | ~all | Marquer spam (recommandé) |
| Neutral | ?all | Aucune politique |
| Pass | +all | Tout autoriser (dangereux) |

# CONSTRUCTION SPF
Format: v=spf1 [mécanismes] [includes] [politique]
Exemple: v=spf1 mx a include:mx.ovh.com ~all

# API ENDPOINTS
POST /domain/zone/{zone}/record - Créer enregistrement TXT avec SPF
POST /domain/zone/{zone}/refresh - Rafraîchir zone

# VALIDATION
- SPF doit commencer par v=spf1
- Maximum 10 lookups DNS (includes + mx + a)
- Longueur max 255 caractères par chaîne TXT
