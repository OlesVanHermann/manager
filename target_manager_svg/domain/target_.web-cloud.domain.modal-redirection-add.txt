# STRUCTURE MODAL
1. HEADER - Titre "Ajouter une redirection" (fond gris)
2. CONTENT
   - Bannière info
   - Sous-domaine source + suffixe
   - URL de destination
   - Type de redirection (radios)
   - Options iframe (conditionnelles)
   - Aperçu
3. FOOTER - Boutons Annuler / Ajouter la redirection

# MODAL TYPE
- Type: Formulaire redirection
- Dimensions: 700x700px
- Header: Fond gris (#F9FAFB)

# CHAMPS
| Champ | Type | Obligatoire | Description |
|-------|------|-------------|-------------|
| Sous-domaine source | text | Non | Préfixe ou vide pour racine |
| URL de destination | url | Oui | URL complète avec protocole |
| Type de redirection | radio | Oui | 301, 302, ou iframe |
| Titre | text | Non | Pour iframe uniquement |
| Mots-clés | text | Non | Pour iframe uniquement |
| Description | textarea | Non | Pour iframe uniquement |

# TYPES DE REDIRECTION
| Code | Nom | Usage | SEO |
|------|-----|-------|-----|
| 301 | Permanente | Déménagement définitif | Transfère le jus SEO |
| 302 | Temporaire | Redirection temporaire | URL source conservée |
| iframe | Visible | Affichage dans frame | URL source visible |

# OPTIONS IFRAME
Les champs titre, mots-clés et description ne sont disponibles que pour les redirections de type iframe. Ils sont utilisés pour le SEO de la page conteneur.

# VALIDATION
- L'URL de destination doit inclure le protocole (http:// ou https://)
- L'URL de destination doit être une URL valide
- Le sous-domaine source ne peut pas être un enregistrement existant conflictuel

# API ENDPOINTS
GET /domain/{domain}/redirection - Liste redirections
POST /domain/{domain}/redirection - Créer redirection
  Body: { subDomain, target, type, title, description, keywords }
