# STRUCTURE MODAL
1. HEADER - Titre "Ajouter une entrée DNS - SRV" + badge SRV (fond OVH bleu)
2. CONTENT
   - Bannière info SRV
   - Service + Protocole + Sous-domaine (3 colonnes)
   - TTL
   - Priorité + Poids + Port (3 colonnes)
   - Cible (serveur FQDN)
   - Aperçu de l'enregistrement
   - Services courants
3. FOOTER - Boutons Annuler / Ajouter l'entrée

# MODAL TYPE
- Type: Formulaire SRV
- Dimensions: 700x700px
- Header: Fond OVH bleu (#000E9C)
- Badge: Violet pour type SRV

# CHAMPS
| Champ | Type | Obligatoire | Description |
|-------|------|-------------|-------------|
| Service | text | Oui | _sip, _xmpp, _ldap... |
| Protocole | select | Oui | _tcp ou _udp |
| Sous-domaine | text | Non | Zone du service |
| TTL | number | Non | Durée de cache |
| Priorité | number | Oui | 0-65535 (basse = prioritaire) |
| Poids | number | Oui | 0-65535 (load balancing) |
| Port | number | Oui | 1-65535 |
| Cible | text | Oui | FQDN du serveur |

# FORMAT SRV
_service._protocol.name TTL IN SRV priority weight port target

# SERVICES COURANTS
| Service | Port standard | Usage |
|---------|---------------|-------|
| _sip._tcp | 5060 | VoIP SIP |
| _sip._udp | 5060 | VoIP SIP UDP |
| _xmpp-client._tcp | 5222 | Jabber/XMPP |
| _xmpp-server._tcp | 5269 | XMPP S2S |
| _ldap._tcp | 389 | LDAP |
| _caldav._tcp | 443 | CalDAV |
| _carddav._tcp | 443 | CardDAV |

# API ENDPOINTS
POST /domain/zone/{zone}/record - Créer enregistrement SRV
POST /domain/zone/{zone}/refresh - Rafraîchir zone

# PRIORITÉ ET POIDS
- Priorité: Les clients contactent d'abord les serveurs à faible priorité
- Poids: Si même priorité, le poids détermine la distribution du trafic
