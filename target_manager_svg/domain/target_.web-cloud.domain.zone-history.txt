# BREADCRUMB
Zone DNS › Historique

# NAV3 TABS
Zone DNS est l'onglet actif (même tab que zone.svg)

# TOOLBAR
| Élément | Action |
|---------|--------|
| ↻ Refresh | Recharger l'historique |
| 30 jours ▼ | Filtrer par période |
| ← Retour | Retour à la zone DNS |

# FILTRES PÉRIODE
| Période | Valeur |
|---------|--------|
| 7 jours | 7 |
| 30 jours | 30 (défaut) |
| 90 jours | 90 |
| 1 an | 365 |

# TIMELINE - STRUCTURE ENTRÉE
| Élément | Description |
|---------|-------------|
| Cercle | Bleu (#0050D7) pour récent, gris (#6B7280) pour ancien |
| Ligne verticale | Connecte les entrées |
| Carte | Détails de la modification |
| Action | Type d'opération |
| Détails | Sous-domaine → cible (TTL) |
| Auteur | Par: nichandle |
| Date | Date/heure de modification |
| Bouton | Restaurer cette version |

# TYPES D'ACTIONS
- Ajout enregistrement (A, AAAA, CNAME, MX, TXT, NS, SRV...)
- Modification enregistrement
- Suppression enregistrement
- Réinitialisation zone
- Import zone
- Modification TTL par défaut

# API ENDPOINTS
GET /domain/zone/{zone}/history - Historique modifications
  Params: {
    from: "2025-12-01",
    to: "2025-12-30"
  }
  Response: [
    {
      id: 12345,
      action: "addRecord",
      recordType: "A",
      subDomain: "www",
      target: "203.0.113.50",
      ttl: 3600,
      nichandle: "ab12345-ovh",
      date: "2025-12-30T14:30:00Z"
    }
  ]

POST /domain/zone/{zone}/history/{id}/restore - Restaurer version
  Response: { taskId: 123456 }
