# STRUCTURE MODAL
1. HEADER - Titre "Ajouter une entrée DNS - TXT" + badge TXT (fond OVH bleu)
2. CONTENT
   - Bannière info TXT
   - Sous-domaine + suffixe
   - TTL
   - Valeur (textarea)
   - Compteur de caractères
   - Utilisations courantes
3. FOOTER - Boutons Annuler / Ajouter l'entrée

# MODAL TYPE
- Type: Formulaire TXT générique
- Dimensions: 700x600px
- Header: Fond OVH bleu (#000E9C)
- Badge: Gris pour type TXT (générique)

# CHAMPS
| Champ | Type | Obligatoire | Description |
|-------|------|-------------|-------------|
| Sous-domaine | text | Non | Préfixe ou @ pour racine |
| TTL | number | Non | Durée de cache |
| Valeur | textarea | Oui | Contenu TXT libre |

# UTILISATIONS COURANTES
| Usage | Sous-domaine | Exemple de valeur |
|-------|--------------|-------------------|
| Vérification Google | @ | google-site-verification=xxx |
| Vérification Microsoft | @ | MS=ms12345678 |
| SPF | @ | v=spf1 include:mx.ovh.com ~all |
| DMARC | _dmarc | v=DMARC1; p=quarantine; ... |
| DKIM | selector._domainkey | v=DKIM1; k=rsa; p=... |
| BIMI | default._bimi | v=BIMI1; l=https://... |
| MTA-STS | _mta-sts | v=STSv1; id=... |

# LIMITES TXT
- Maximum 255 caractères par chaîne TXT (RFC)
- Les valeurs plus longues sont découpées automatiquement
- Les guillemets sont ajoutés automatiquement si nécessaires
- Caractères spéciaux échappés si nécessaire

# VALIDATION
- La valeur ne peut pas être vide
- Les retours à la ligne sont convertis en espaces
- Encodage UTF-8 supporté

# API ENDPOINTS
POST /domain/zone/{zone}/record - Créer enregistrement TXT
POST /domain/zone/{zone}/refresh - Rafraîchir zone
