# STRUCTURE MODAL
1. HEADER - Titre "Désactiver DNSSEC" (fond rouge clair)
2. CONTENT
   - Bannière warning (réduction sécurité)
   - Texte confirmation
   - Liste des conséquences
3. FOOTER - Boutons Annuler / Désactiver (danger)

# MODAL TYPE
- Type: Confirmation action dangereuse
- Dimensions: 550x400px
- Style: Warning/Danger (header rouge)

# CONSÉQUENCES DE LA DÉSACTIVATION
1. **Suppression des DS records** - Les enregistrements DS au registre
   seront supprimés
2. **Plus de validation** - Les résolveurs DNSSEC ne pourront plus
   valider les réponses DNS
3. **Propagation** - Le changement peut prendre jusqu'à 48h pour
   se propager complètement

# QUAND DÉSACTIVER DNSSEC ?
- Changement de serveurs DNS vers des serveurs ne supportant pas DNSSEC
- Migration vers un autre hébergeur DNS
- Problèmes de résolution causés par DNSSEC mal configuré
- Nécessité de modifier rapidement la zone (urgence)

# RISQUES
Sans DNSSEC, le domaine est vulnérable à :
- DNS spoofing
- Cache poisoning
- Man-in-the-middle DNS

# RÉACTIVATION
DNSSEC peut être réactivé à tout moment depuis la page DNSSEC
du domaine. La réactivation prend quelques minutes à quelques heures.

# API ENDPOINTS
DELETE /domain/{domain}/dsRecord - Désactiver DNSSEC
  Response: { status: "disabled" }

POST /domain/{domain}/deactivateDnssec - Alternative
  Response: { taskId: "12345" }
