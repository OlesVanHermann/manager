# Cahier des Charges : home/billing - SLA (Garantie de disponibilité)

## 1. VUE D'ENSEMBLE
- **Type** : Page de liste avec accordéon
- **URL** : /billing/sla
- **Objectif** : Consulter les incidents et demander des compensations SLA

## 2. DIMENSIONS
| Propriété | Valeur |
|-----------|--------|
| Largeur | 1200px |
| Hauteur | 700px |

## 3. COMPOSANTS UI
- **OdsHeader** : En-tête OVHcloud
- **OdsBreadcrumb** : Mon compte / Facturation / SLA
- **OdsMessage** : Info délai de demande
- **OdsCard** : Container principal
- **OdsAccordion** : Liste des incidents
- **OdsBadge** : Statuts (Demander SLA, Traité, En cours)
- **OdsDatagrid** : Services impactés (dans accordéon)
- **OdsPagination** : Navigation

## 4. HEADER PAGE
| Élément | Description |
|---------|-------------|
| Titre | "SLA - Garantie de disponibilité" |

## 5. MESSAGE INFO
| Type | Contenu |
|------|---------|
| Info (bleu) | "ℹ️ Vos demandes de compensation SLA peuvent être effectuées dans les 30 jours suivant l'incident." |

## 6. ACCORDÉON INCIDENTS

### Structure item accordéon
| Élément | Description |
|---------|-------------|
| Indicateur | ▼ (ouvert) / ▶ (fermé) |
| Date/heure | 15/01/2025 10:30 |
| Titre | "Incident réseau datacenter RBX" |
| Description | Texte italique explicatif |
| Badge action | Demander SLA / Traité / En cours |

### Contenu accordéon (ouvert)
| Élément | Description |
|---------|-------------|
| Info | "Vous avez jusqu'au 14/02/2025 pour demander une compensation." |
| Tableau services impactés | Liste des services avec détails |

### Colonnes tableau services
| Colonne | Description |
|---------|-------------|
| Service | Nom du service (ns1234567.ip-12-34-56.eu) |
| Description | Type de service (Serveur dédié Advance-2) |
| Plan SLA | Pourcentage garanti (99.99%) |
| Application | Compensation (Crédit mensuel 10%) |

## 7. BADGES STATUT
| Statut | Couleur | Action |
|--------|---------|--------|
| Demander SLA | Primary (bleu) | Bouton cliquable |
| Traité | Vert (#D1FAE5) | ✓ Traité |
| En cours | Jaune (#FEF3C7) | En cours |

## 9. APIS & ENDPOINTS
- GET /me/sla - Liste des incidents SLA
- GET /me/sla/{incidentId} - Détails incident
- GET /me/sla/{incidentId}/services - Services impactés
- POST /me/sla/{incidentId}/apply - Demander compensation
