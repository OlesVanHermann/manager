# Cahier des Charges : home/iam - Créer Politique

## 1. VUE D'ENSEMBLE
- **Type** : Page formulaire création politique
- **URL** : /iam/policies/create
- **Objectif** : Créer nouvelle politique IAM

## 2. STRUCTURE PAGE
- Breadcrumb
- Titre + description
- Formulaire carte blanche
- Footer actions

## 3. COMPOSANTS UI
- **OdsCard** : Conteneur formulaire
- **OdsInput** : Nom politique
- **OdsTextarea** : Description
- **OdsSelect** : Identités, Produits, Groupes
- **OdsCheckbox** : Actions
- **OdsButton** : Annuler, Créer

## 4. SECTIONS FORMULAIRE
| Section | Description |
|---------|-------------|
| Nom de la politique | Input texte requis |
| Description | Textarea optionnel |
| Identités | Select multi utilisateurs/groupes |
| Types de produits | Select catégorie produit |
| Ressources | Liste ressources (après sélection type) |
| Groupes de ressources | Select groupes existants |
| Conditions | Ajout conditions optionnelles |
| Actions | Checkboxes permissions |

## 5. CHAMP NOM
| Propriété | Valeur |
|-----------|--------|
| Label | "Nom de la politique" |
| Requis | Oui (*) |
| Largeur | 400px |
| Placeholder | "ma-nouvelle-politique" |

## 6. CHAMP DESCRIPTION
| Propriété | Valeur |
|-----------|--------|
| Label | "Description de la politique" |
| Requis | Non |
| Dimensions | 832x80px |

## 7. SECTION ACTIONS
Format des permissions :
```
{product}:apiovh:{resource}/{action}
```
Exemples :
- account:apiovh:me/get
- account:apiovh:me/edit
- dedicatedServer:apiovh:server/get

## 8. BOUTONS CONDITIONS
| Label | Style |
|-------|-------|
| "+ Ajouter une condition" | Secondary outline |

## 9. FOOTER ACTIONS
| Bouton | Style | Label |
|--------|-------|-------|
| Annuler | Secondary | "Annuler" |
| Créer | Primary | "Créer la politique" |

## 10. APIS & ENDPOINTS
- GET /me/identity/user - Liste utilisateurs
- GET /me/identity/group - Liste groupes
- GET /me/iam/resourceGroup - Groupes ressources
- GET /me/iam/action - Actions disponibles
- POST /me/iam/policy - Créer politique
