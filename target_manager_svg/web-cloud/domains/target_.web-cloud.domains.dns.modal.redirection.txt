# STRUCTURE MODAL
1. HEADER - Titre "Ajouter une redirection" (fond gris)
2. CONTENT
   - Source (sous-domaine)
   - Type de redirection (cards)
   - URL de destination
   - Options (checkbox)
   - Titre page (pour iFrame)
3. FOOTER - Boutons Annuler / Ajouter

# MODAL TYPE
- Type: Formulaire redirection web
- Dimensions: 700x500px
- Header: Fond gris (#F9FAFB)

# TYPES DE REDIRECTION
| Type | Code | Description |
|------|------|-------------|
| Visible 301 | 301 | Permanente - SEO conservé |
| Visible 302 | 302 | Temporaire - SEO non transmis |
| Invisible | iFrame | URL masquée dans barre adresse |

# CHAMPS
| Champ | Obligatoire | Description |
|-------|-------------|-------------|
| Source | Oui | Sous-domaine (www, @, blog...) |
| Type | Oui | 301, 302 ou iFrame |
| Destination | Oui | URL complète avec protocole |
| Paramètres | Non | Transmettre query string |
| Titre | Non | Pour iFrame uniquement |

# RECOMMANDATIONS SEO
- **301** : Utilisez pour les redirections permanentes (changement de domaine)
- **302** : Utilisez pour les redirections temporaires (maintenance)
- **iFrame** : Déconseillé pour le SEO (contenu dupliqué)

# OPTIONS
- **Inclure paramètres** : Les paramètres URL (?query=value) sont
  transmis à l'URL de destination

# LIMITATIONS iFrame
- Ne fonctionne pas avec les sites X-Frame-Options
- Problèmes de sécurité potentiels
- Non recommandé pour les sites avec formulaires

# API ENDPOINTS
POST /domain/zone/{zone}/redirection - Créer redirection
  Body: {
    subDomain: "www",
    type: "visible301" | "visible302" | "invisible",
    target: "https://www.exemple.com/page",
    preserveQueryString: true,
    title: "Titre page" (optional, for invisible)
  }
