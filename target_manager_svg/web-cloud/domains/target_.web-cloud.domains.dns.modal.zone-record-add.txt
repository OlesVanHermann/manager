# STRUCTURE MODAL
1. HEADER - Titre "Ajouter une entrée DNS" (fond gris)
2. CONTENT
   - Type d'enregistrement (select + grille raccourcis)
   - Sous-domaine + suffixe
   - TTL (select avec valeurs prédéfinies)
   - Cible (champs dynamiques selon type)
   - Aperçu de l'enregistrement
3. FOOTER - Boutons Annuler / Ajouter l'entrée

# MODAL TYPE
- Type: Formulaire générique ajout record
- Dimensions: 600x560px
- Header: Fond gris (#F9FAFB)

# CHAMPS
| Champ | Type | Obligatoire | Description |
|-------|------|-------------|-------------|
| Type | select | Oui | A, AAAA, CNAME, MX, TXT... |
| Sous-domaine | text | Non | Préfixe ou vide pour @ |
| TTL | select | Non | 300, 3600, 86400... |
| Cible | text | Oui | Varie selon le type |

# TYPES D'ENREGISTREMENT
Raccourcis affichés:
- A (IPv4)
- AAAA (IPv6)
- CNAME (alias)
- MX (mail)
- TXT (texte)
- SRV (service)
- Autres... (ouvre liste complète)

# TTL VALEURS PRÉDÉFINIES
| Valeur | Label |
|--------|-------|
| 60 | 1 minute |
| 300 | 5 minutes |
| 3600 | 1 heure (défaut) |
| 86400 | 1 jour |
| 604800 | 1 semaine |

# COMPORTEMENT DYNAMIQUE
Le champ "Cible" change selon le type sélectionné:
- A: Champ IP (validation IPv4)
- AAAA: Champ IP (validation IPv6)
- CNAME: Champ FQDN
- MX: Priority + Champ FQDN
- TXT: Textarea

# NOTE
Cette modale est générique. Pour les types complexes (DKIM, DMARC, SRV...),
l'utilisateur sera redirigé vers la modale spécifique au type.

# API ENDPOINTS
POST /domain/zone/{zone}/record - Créer enregistrement
  Body: { fieldType, subDomain, target, ttl }
POST /domain/zone/{zone}/refresh - Rafraîchir zone
