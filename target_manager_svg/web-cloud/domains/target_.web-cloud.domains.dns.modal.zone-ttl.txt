# STRUCTURE MODAL
1. HEADER - Titre "Modifier le TTL par défaut" (fond gris)
2. CONTENT
   - Bannière info (explication TTL)
   - Zone DNS (affichage)
   - Options TTL prédéfinies (radios)
   - Option personnalisée
   - Appliquer à tous (checkbox)
3. FOOTER - Boutons Annuler / Appliquer

# MODAL TYPE
- Type: Configuration TTL
- Dimensions: 600x540px
- Header: Fond gris (#F9FAFB)

# CHAMPS
| Champ | Type | Description |
|-------|------|-------------|
| TTL | radio | Valeur prédéfinie ou custom |
| Appliquer à tous | checkbox | Mettre à jour existants |

# OPTIONS TTL
| Valeur | Label | Usage |
|--------|-------|-------|
| 300 | 5 minutes | Changements fréquents, migration |
| 3600 | 1 heure | Recommandé, équilibre optimal |
| 86400 | 1 jour | Enregistrements stables |
| Custom | Personnalisé | Min 60, Max 604800 (7j) |

# COMPORTEMENT
| Option | Existants | Futurs |
|--------|-----------|--------|
| Appliquer à tous: Oui | Mis à jour | Nouveau TTL |
| Appliquer à tous: Non | Inchangés | Nouveau TTL |

# IMPACT TTL
- TTL court: Propagation rapide, plus de requêtes DNS
- TTL long: Moins de requêtes, propagation lente
- Conseil: Baisser le TTL avant une migration

# API ENDPOINTS
PUT /domain/zone/{zone} - Modifier TTL par défaut
  Body: { ttl: 3600 }
PUT /domain/zone/{zone}/record/bulk - Mettre à jour tous les TTL
POST /domain/zone/{zone}/refresh - Rafraîchir zone
