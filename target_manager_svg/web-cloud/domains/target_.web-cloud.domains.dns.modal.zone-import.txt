# STRUCTURE MODAL
1. HEADER - Titre "Importer une zone DNS" (fond OVH bleu)
2. CONTENT
   - Bannière info
   - Bannière warning (écrasement)
   - Zone de destination
   - Zone d'upload (drag & drop)
   - Formats acceptés
   - Options d'import (checkboxes)
3. FOOTER - Boutons Annuler / Importer

# MODAL TYPE
- Type: Import fichier avec options
- Dimensions: 600x580px
- Header: Fond OVH bleu (#000E9C)

# CHAMPS
| Champ | Type | Description |
|-------|------|-------------|
| Fichier | file upload | Fichier de zone |
| Conserver NS | checkbox | Garder les NS actuels |
| Valider syntaxe | checkbox | Vérifier avant import |
| Créer sauvegarde | checkbox | Backup zone actuelle |

# FORMATS ACCEPTÉS
- .zone (format BIND standard)
- .txt (fichier texte BIND)
- .bind (alias format BIND)
- Taille max: 1 Mo

# OPTIONS D'IMPORT
| Option | Par défaut | Description |
|--------|------------|-------------|
| Conserver NS | Non | Ne pas écraser les NS OVH |
| Valider syntaxe | Oui | Vérifier avant d'appliquer |
| Créer sauvegarde | Non | Sauvegarder l'état actuel |

# PROCESSUS D'IMPORT
1. Upload du fichier
2. Validation de la syntaxe (si activée)
3. Affichage des erreurs potentielles
4. Confirmation utilisateur
5. Application des changements
6. Rafraîchissement de la zone

# API ENDPOINTS
POST /domain/zone/{zone}/import - Importer fichier zone
  Body: multipart/form-data avec fichier
POST /domain/zone/{zone}/refresh - Rafraîchir après import
