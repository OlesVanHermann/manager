# STRUCTURE MODAL
1. HEADER - Titre "Ajouter une entrée DynHost" (fond gris)
2. CONTENT
   - Bannière info (explication DynHost)
   - Sous-domaine + suffixe
   - Adresse IP initiale + bouton détection
   - Informations accès DynHost
   - Aperçu de l'enregistrement
3. FOOTER - Boutons Annuler / Ajouter

# MODAL TYPE
- Type: Formulaire DynHost
- Dimensions: 600x500px
- Header: Fond gris (#F9FAFB)

# CHAMPS
| Champ | Type | Obligatoire | Description |
|-------|------|-------------|-------------|
| Sous-domaine | text | Oui | Préfixe DynHost |
| IP initiale | text | Oui | IPv4 de départ |

# FONCTIONNALITÉS
- Bouton "Détecter mon IP": Récupère l'IP publique actuelle
- Création automatique enregistrement DYNHOST

# CAS D'USAGE DYNHOST
- Serveur domestique avec IP dynamique
- NAS accessible depuis l'extérieur
- Caméra de surveillance
- Raspberry Pi/serveur personnel

# CLIENTS DYNDNS COMPATIBLES
- ddclient (Linux)
- DynDNS Update Client (Windows)
- No-IP DUC
- Routeurs avec DynDNS intégré (Fritz!Box, etc.)

# CONFIGURATION CLIENT
URL de mise à jour OVH:
```
https://www.ovh.com/nic/update?system=dyndns&hostname=HOSTNAME&myip=IP
```
Avec authentification login/password DynHost.

# API ENDPOINTS
POST /domain/zone/{zone}/dynHost/record - Créer entrée DynHost
  Body: { subDomain: "home", ip: "86.123.45.67" }
