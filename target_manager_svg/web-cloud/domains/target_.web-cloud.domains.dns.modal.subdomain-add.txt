# STRUCTURE MODAL
1. HEADER - Titre "Ajouter un sous-domaine" (fond gris)
2. CONTENT
   - Bannière info
   - Nom du sous-domaine + suffixe
   - Adresse IP de destination
   - Raccourcis (sélection rapide)
   - Aperçu de l'enregistrement DNS
3. FOOTER - Boutons Annuler / Ajouter

# MODAL TYPE
- Type: Formulaire sous-domaine simplifié
- Dimensions: 600x460px
- Header: Fond gris (#F9FAFB)

# CHAMPS
| Champ | Type | Obligatoire | Description |
|-------|------|-------------|-------------|
| Nom | text | Oui | Préfixe du sous-domaine |
| IP destination | text | Oui | IPv4 ou IPv6 cible |

# RACCOURCIS
Les boutons de raccourci permettent de pré-remplir l'IP:
| Bouton | Action |
|--------|--------|
| Hébergement OVH | IP de l'hébergement mutualisé associé |
| Serveur dédié | Liste des serveurs dédiés du compte |
| VPS OVH | Liste des VPS du compte |

# VALIDATION NOM
- Caractères autorisés: a-z, 0-9, -
- Pas de tiret au début ou à la fin
- Longueur: 1-63 caractères
- Nom unique (pas de conflit avec existant)

# ENREGISTREMENT CRÉÉ
Cette modale créera un enregistrement A (ou AAAA si IPv6):
- Type: A ou AAAA selon l'IP fournie
- TTL: 3600 secondes (par défaut)
- Sous-domaine: nom saisi
- Cible: IP fournie

# API ENDPOINTS
POST /domain/zone/{zone}/record - Créer enregistrement A/AAAA
  Body: { fieldType: "A", subDomain: "blog", target: "213.186.33.5", ttl: 3600 }
POST /domain/zone/{zone}/refresh - Rafraîchir zone
