# STRUCTURE MODAL
1. HEADER - Titre "Ajouter une entrée DNS - CAA" + badge CAA (fond OVH bleu)
2. CONTENT
   - Champ sous-domaine + suffixe domaine
   - Champ TTL
   - Champ Flags (0-255)
   - Sélecteur Tag (issue, issuewild, iodef)
   - Champ Cible (autorité de certification)
3. FOOTER - Boutons Annuler / Ajouter l'entrée

# MODAL TYPE
- Type: Formulaire d'ajout enregistrement CAA
- Dimensions: 650x500px
- Header: Fond OVH bleu (#000E9C)
- Badge: Vert pour type CAA

# CHAMPS
| Champ | Type | Obligatoire | Description |
|-------|------|-------------|-------------|
| Sous-domaine | text | Non | Préfixe (vide = @) |
| TTL | number | Non | Durée de cache |
| Flags | number | Oui | 0-255 (généralement 0) |
| Tag | select | Oui | issue, issuewild, iodef |
| Cible | text | Oui | CA autorisée |

# TAGS CAA
| Tag | Description |
|-----|-------------|
| issue | CA autorisée pour certificats standard |
| issuewild | CA autorisée pour wildcards |
| iodef | URL pour signaler violations |

# API ENDPOINTS
POST /domain/zone/{zone}/record - Créer enregistrement CAA
POST /domain/zone/{zone}/refresh - Rafraîchir zone après ajout
