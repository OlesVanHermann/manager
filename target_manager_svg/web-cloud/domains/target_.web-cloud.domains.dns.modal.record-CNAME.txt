# STRUCTURE MODAL
1. HEADER - Titre "Ajouter une entrée DNS - CNAME" + badge CNAME (fond OVH bleu)
2. CONTENT
   - Champ sous-domaine + suffixe domaine
   - Champ TTL (secondes)
   - Champ Cible (FQDN obligatoire)
   - Texte d'aide sur format FQDN
3. FOOTER - Boutons Annuler / Ajouter l'entrée

# MODAL TYPE
- Type: Formulaire d'ajout enregistrement CNAME
- Dimensions: 650x420px
- Header: Fond OVH bleu (#000E9C)
- Badge: Bleu pour type CNAME

# CHAMPS
| Champ | Type | Obligatoire | Description |
|-------|------|-------------|-------------|
| Sous-domaine | text | Non | Préfixe (alias) |
| TTL | number | Non | Durée de cache |
| Cible | text | Oui | FQDN terminé par un point |

# VALIDATION
- Sous-domaine: Ne peut pas être @ (racine)
- Cible: Doit être un FQDN valide
- FQDN: Terminé par un point (ex: www.exemple.com.)

# CAS D'USAGE CNAME
- Alias de sous-domaine (blog → www)
- Redirection vers CDN
- Services tiers (mail, stockage)

# API ENDPOINTS
POST /domain/zone/{zone}/record - Créer enregistrement CNAME
POST /domain/zone/{zone}/refresh - Rafraîchir zone après ajout
