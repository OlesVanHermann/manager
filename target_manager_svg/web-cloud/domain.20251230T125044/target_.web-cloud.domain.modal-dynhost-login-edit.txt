# Cahier des Charges : web-cloud/domain - Modal Modifier accès DynHost

## 1. VUE D'ENSEMBLE
- **Modal** : Modification d'un identifiant d'accès DynHost
- **Type** : Formulaire de modification
- **Objectif** : Modifier le sous-domaine et/ou le mot de passe d'un accès DynHost

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (✕)
- Description
- Champ sous-domaine
- Champ mot de passe (optionnel)
- Champ confirmation
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (550x480)
- **OdsInput** : Sous-domaine, Mot de passe, Confirmation
- **OdsButton** : Annuler, Modifier

## 4. TITRE
"Modifier un identifiant d'accès au Dynhost"

## 5. DESCRIPTION
"Modifier le sous-domaine d'un identifiant d'accès"

## 6. CHAMP SOUS-DOMAINE
| Élément | Valeur |
|---------|--------|
| Label | "Sous-domaine :" |
| Obligatoire | Oui (*) |
| Input | "dyn" |
| Suffixe (addon gris) | ".example.com" |

## 7. CHAMP MOT DE PASSE
| Élément | Valeur |
|---------|--------|
| Label | "Mot de passe :" |
| Obligatoire | Non |
| Placeholder | "Laisser vide pour ne pas modifier" |
| Aide | "Le mot de passe doit être composé de 8 à 32 caractères alphanumériques." |

## 8. CHAMP CONFIRMATION
| Élément | Valeur |
|---------|--------|
| Label | "Confirmation :" |
| Obligatoire | Oui (*) si mot de passe renseigné |
| Placeholder | "Confirmer le mot de passe" |

## 9. BOUTONS FOOTER
- Annuler (secondaire)
- Modifier (primaire)

## 10. APIS & ENDPOINTS
- PUT /domain/zone/{zoneName}/dynHost/login/{login} : Modifier accès
