# Cahier des Charges : web-cloud/domain - Modal Ajouter enregistrement CAA

## 1. VUE D'ENSEMBLE
- **Modal** : Création d'un enregistrement DNS de type CAA
- **Type** : Formulaire de création
- **Objectif** : Ajouter un enregistrement CAA (Certificate Authority Authorization)

## 2. STRUCTURE MODAL
- Header bleu OVH avec titre, badge CAA et bouton fermer (✕)
- Champ sous-domaine
- Champ TTL
- Champ Flags
- Champ Tag (select)
- Champ Cible
- Footer avec boutons

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (650x500)
- **OdsInput** : Sous-domaine, TTL, Flags, Cible
- **OdsSelect** : Tag
- **OdsBadge** : Type CAA (vert)
- **OdsButton** : Annuler, Ajouter l'entrée

## 4. TITRE
"Ajouter une entrée DNS - CAA" avec badge [CAA]

## 5. CHAMP SOUS-DOMAINE
| Élément | Valeur |
|---------|--------|
| Label | "Sous-domaine" |
| Input | "@" (placeholder) |
| Suffixe (addon gris) | ".exemple.com" |

## 6. CHAMP TTL
| Élément | Valeur |
|---------|--------|
| Label | "TTL" |
| Input | 3600 |

## 7. CHAMP FLAGS
| Élément | Valeur |
|---------|--------|
| Label | "Flags" |
| Obligatoire | Oui (*) |
| Input | 0 |
| Aide | "0-255" |

## 8. CHAMP TAG
| Élément | Valeur |
|---------|--------|
| Label | "Tag" |
| Obligatoire | Oui (*) |
| Type | Select dropdown |
| Valeur | "issue" |
| Options | issue, issuewild, iodef |

## 9. CHAMP CIBLE
| Élément | Valeur |
|---------|--------|
| Label | "Cible" |
| Obligatoire | Oui (*) |
| Input | "letsencrypt.org" |

## 10. BOUTONS FOOTER
- Annuler (secondaire)
- Ajouter l'entrée (primaire)

## 11. APIS & ENDPOINTS
- POST /domain/zone/{zoneName}/record : Créer enregistrement CAA
  - fieldType: "CAA"
  - subDomain: string
  - target: "flags tag value" format
  - ttl: number
