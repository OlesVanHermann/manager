## API ENDPOINTS

### Liste utilisateurs
GET /sms/{serviceName}/users
Response: [{ login, quotaLimit, quotaUsed, allowedIps, callbackUrl, status }]

### Créer utilisateur
POST /sms/{serviceName}/users
Body: { login, password, quotaLimit, allowedIps?, callbackUrl? }

### Modifier utilisateur
PUT /sms/{serviceName}/users/{login}
Body: { quotaLimit?, allowedIps?, callbackUrl?, status? }

### Supprimer utilisateur
DELETE /sms/{serviceName}/users/{login}

### Régénérer mot de passe
POST /sms/{serviceName}/users/{login}/password
Response: { password }

### Stats utilisation
GET /sms/{serviceName}/users/{login}/usage
Query: { period: '7d' }
Response: [{ date, count }]

================================================================================

## REGLES METIER

- Login unique par compte
- Quota quotidien réinitialisé à minuit UTC
- IPs au format CIDR ou adresse unique
- Callback optionnel, doit être HTTPS
- Mot de passe visible uniquement à la création
- Maximum 20 utilisateurs par compte
