et gestion des variables personnalisables.

## API ENDPOINTS

### Liste modèles
GET /sms/{serviceName}/templates
Query: { category }
Response: [{
  id, name, content, category,
  variables: [], createdAt, updatedAt
}]

### Créer modèle
POST /sms/{serviceName}/templates
Body: { name, content, category }

### Modifier modèle
PUT /sms/{serviceName}/templates/{id}
Body: { name, content, category }

### Supprimer modèle
DELETE /sms/{serviceName}/templates/{id}

### Dupliquer modèle
POST /sms/{serviceName}/templates/{id}/duplicate

### Catégories
GET /sms/{serviceName}/templates/categories
Response: ['Rendez-vous', 'Sécurité', 'Marketing', 'Commandes', ...]

## VARIABLES SUPPORTÉES

| Variable       | Description                     | Exemple         |
|----------------|---------------------------------|-----------------|
| {prenom}       | Prénom du destinataire          | Jean            |
| {nom}          | Nom du destinataire             | Dupont          |
| {date}         | Date formatée                   | 28/12/2025      |
| {heure}        | Heure formatée                  | 14:30           |
| {code}         | Code aléatoire/promo            | ABC123          |
| {lien}         | URL courte                      | ovh.to/xyz      |
| {montant}      | Montant formaté                 | 49,99€          |
| {numero}       | Numéro de référence             | CMD-12345       |

## REGLES METIER

- Max 50 modèles par compte
- Nom unique obligatoire
- Variables auto-détectées {xxx}
- 160 chars = 1 SMS standard
- 70 chars si Unicode
- Catégorie obligatoire
