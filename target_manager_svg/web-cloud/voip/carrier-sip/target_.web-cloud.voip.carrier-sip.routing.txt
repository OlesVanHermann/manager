# STRUCTURE PAGE
1. NAV1 - Header OVH (Web Cloud actif)
2. NAV2 - Web Cloud (VoIP actif)
3. LEFT PANEL - Liste des trunks
4. RIGHT PANEL
   - Header: Identifiant trunk + badge statut + description
   - NAV3: Général | Paramètres | [Routage] | CDR | Consommation
   - Toolbar: Refresh, Recherche, + Ajouter règle
   - Bannière info: Explication ordre de priorité
   - Table règles de routage: Priorité, Nom, Pattern, Action, Destination, Statut
   - Card "Destinations configurées": Nombre gateways
   - Card "Statistiques routage": Appels routés/bloqués 24h

# COLONNES TABLE
| Colonne | Largeur | Contenu |

# INTERACTIONS
- Clic NAV3 tab → Navigation onglet
- Clic ↻ → Rafraîchir liste
- Clic "+ Ajouter règle" → Modal création règle
- Clic ▲/▼ priorité → Réordonner règle
- Clic ✎ → Modal édition règle
- Clic ✕ → Modal confirmation suppression
- Clic "Gérer destinations" → Page gestion gateways

# API ENDPOINTS
GET /telephony/{ba}/trunk/{sn}/routes              → Liste règles routage
POST /telephony/{ba}/trunk/{sn}/routes             → Créer règle
PUT /telephony/{ba}/trunk/{sn}/routes/{id}         → Modifier règle
DELETE /telephony/{ba}/trunk/{sn}/routes/{id}      → Supprimer règle
PUT /telephony/{ba}/trunk/{sn}/routes/{id}/priority → Modifier priorité
GET /telephony/{ba}/trunk/{sn}/destinations        → Liste gateways
GET /telephony/{ba}/trunk/{sn}/routing/stats       → Statistiques routage
