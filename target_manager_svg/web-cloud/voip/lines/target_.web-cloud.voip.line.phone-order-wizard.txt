# Cahier des Charges : telecom/line - Wizard commande telephone

## 1. RESUME EXECUTIF

### Perimetre
Wizard de commande d'un telephone VoIP.

### Objectif fonctionnel
- Selectionner le materiel
- Choisir les accessoires
- Definir la quantite

---

## 2. LAYOUT

### Structure wizard avec stepper 4 etapes et grille selection

---

## 3. COMPOSANTS UI

### Header page
| Element | Description |
|---------|-------------|
| Retour | ← Retour au telephone |
| Titre | Commander un telephone |

### Stepper
| Etape | Etat | Nom |
|-------|------|-----|
| 1 | Active (bleu) | Materiel |
| 2 | Todo (gris) | Livraison |
| 3 | Todo (gris) | RMA |
| 4 | Todo (gris) | Recapitulatif |

### Carte selection materiel
| Element | Description |
|---------|-------------|
| Titre | Choisir le materiel |

### Grille telephones (3 colonnes)
| Element | Description |
|---------|-------------|
| Image | Zone grise avec emoji |
| Nom | Modele telephone |
| Description | Lignes SIP, ecran |
| Prix | XX,XX € HT (vert) |
| Checkbox | Cercle selection |

### Carte telephone selectionnee
| Etat | Style |
|------|-------|
| Selected | Fond bleu clair, bordure bleue, check bleu |
| Normal | Fond blanc, bordure grise |

### Section Accessoires
| Element | Description |
|---------|-------------|
| Titre | Accessoires (optionnel) |
| Liste | Checkboxes avec prix |

### Checkbox accessoire
| Etat | Style |
|------|-------|
| Checked | Fond bleu, check blanc |
| Unchecked | Fond blanc, bordure grise |

### Input quantite
| Element | Description |
|---------|-------------|
| Stepper | - / valeur / + |
| Min | 1 |
| Max | 10 |

### Footer
| Element | Description |
|---------|-------------|
| Total | Total : XXX,XX € HT |
| Bouton | Suivant → (Primary) |

---

## 4. APIS & DONNEES

### Endpoints
```
GET /telephony/{billingAccount}/line/{serviceName}/phone/catalog
GET /telephony/{billingAccount}/line/{serviceName}/phone/accessories
```

### Modele
```typescript
interface PhoneCatalog {
  id: string;
  model: string;
  brand: string;
  description: string;
  priceHT: number;
  lines: number;
  features: string[];
}

interface Accessory {
  id: string;
  name: string;
  priceHT: number;
  compatible: string[];
}
```
