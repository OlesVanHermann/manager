# Cahier des Charges : web-cloud/zimbra - Modal ajout alias

## 1. RESUME EXECUTIF

### Perimetre
Modal d'ajout d'un alias a un compte Zimbra.

### Objectif fonctionnel
- Definir l'adresse alias
- Selectionner le domaine
- Valider la creation

---

## 2. LAYOUT

### Structure modale avec header + formulaire + footer

---

## 3. COMPOSANTS UI

### Modal header
| Element | Description |
|---------|-------------|
| Titre | Ajouter un alias |
| Bouton fermer | X |

### Description
| Element | Description |
|---------|-------------|
| Texte | Creer un alias pour le compte {email} |

### Formulaire
| Champ | Type | Description |
|-------|------|-------------|
| Alias * | Input | Nom de l'alias |
| @ | Texte fixe | Separateur |
| Domaine | Select | Selectionner un nom de domaine |

### Regles de validation
| Regle | Description |
|-------|-------------|
| Caracteres | Lettres, chiffres, tirets, points |
| Debut/fin | Pas de tiret ou point |
| Longueur | 1 a 64 caracteres |

### Footer
| Element | Description |
|---------|-------------|
| Bouton annuler | Secondary |
| Bouton confirmer | Primary |

---

## 4. APIS & DONNEES

### Endpoints
```
GET /zimbra/platform/{platformId}/domain
POST /zimbra/platform/{platformId}/account/{accountId}/alias
```

### Modele
```typescript
interface CreateAlias {
  alias: string;
  domain: string;
}
```

## 6. REGLES METIER

- Alias sur domaines de la plateforme uniquement
- Validation format email
- Un alias ne peut pointer que vers un seul compte
- Alias unique sur toute la plateforme
