# STRUCTURE MODAL

- Dimensions: 500x520px
- Type: Formulaire avec validation
- Header: Icône + titre
- Indicateur force mot de passe
- Checklist des exigences

# CHAMPS

| Champ              | Type     | Requis | Description               |
|--------------------|----------|--------|---------------------------|
| Nouveau MDP        | password | Oui    | Nouveau mot de passe      |
| Confirmer MDP      | password | Oui    | Doit matcher le nouveau   |

# EXIGENCES MOT DE PASSE

| Règle                  | Validation                |
|------------------------|---------------------------|
| Longueur minimum       | ≥ 8 caractères            |
| Majuscule              | Au moins 1 [A-Z]          |
| Minuscule              | Au moins 1 [a-z]          |
| Chiffre                | Au moins 1 [0-9]          |
| Caractère spécial      | Au moins 1 (!@#$%^&*...)  |

# INDICATEUR FORCE

| Niveau  | Couleur | Largeur |
|---------|---------|---------|
| Faible  | Rouge   | 25%     |
| Moyen   | Orange  | 50%     |
| Bon     | Jaune   | 75%     |
| Fort    | Vert    | 100%    |

# OPTIONS

| Option                          | Par défaut | Description                |
|---------------------------------|------------|----------------------------|
| Envoyer par email               | Non        | Email avec nouveau MDP     |
| Forcer déconnexion              | Oui        | Déco toutes les sessions   |

# INTERACTIONS

- Saisie MDP → Mise à jour indicateur force
- Toggle visibilité → Affiche/masque le MDP
- Clic "Enregistrer" → Validation + envoi
- Clic "Annuler" → Ferme modal

# VALIDATION

- Les deux champs doivent matcher
- Toutes les exigences doivent être validées
- Bouton "Enregistrer" désactivé si invalide

# API ENDPOINTS

PUT /email/domains/{domain}/accounts/{account}/password

## Payload
```json
{
  "password": "NewSecureP@ss123",
  "sendByEmail": false,
  "forceLogout": true
}
```
