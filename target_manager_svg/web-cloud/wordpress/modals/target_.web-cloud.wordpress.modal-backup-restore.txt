DESCRIPTION
Modal de restauration d'une sauvegarde WordPress.
Permet de choisir entre restaurer tout, fichiers uniquement ou BDD uniquement.
Crée automatiquement une sauvegarde avant restauration.

================================================================================

DONNÉES DYNAMIQUES
| Exemple SVG              | Source API                                          |
|--------------------------|-----------------------------------------------------|
| 27/12/2025 15:42         | GET /managedCMS/resource/{sn}/backups/{id} → date   |
| Manuel                   | GET /managedCMS/resource/{sn}/backups/{id} → type   |
| 242 Mo                   | GET /managedCMS/resource/{sn}/backups/{id} → size   |
| Note: Avant mise...      | GET /managedCMS/resource/{sn}/backups/{id} → note   |

================================================================================

STRUCTURE MODAL
1. Header: Titre + bouton fermer
2. Sauvegarde sélectionnée: Date + type + taille + note
3. Options restauration: Radio buttons (tout/fichiers/BDD)
4. Bannière warning: Avertissement remplacement données
5. Footer: Annuler | Restaurer

================================================================================

INTERACTIONS
- Clic "×" → Ferme modal
- Clic "Annuler" → Ferme modal
- Radio buttons → Sélection type restauration
- Clic "Restaurer" → POST /managedCMS/resource/{sn}/backups/{id}/restore

================================================================================

API ENDPOINTS
POST /managedCMS/resource/{serviceName}/backups/{backupId}/restore
  Body: { type: "all" | "files" | "database" }
  Response: { taskId, status: "pending" }
