DESCRIPTION
Modal de confirmation pour supprimer définitivement un site WordPress.
Demande de taper "SUPPRIMER" pour confirmer l'action irréversible.

================================================================================

DONNÉES DYNAMIQUES
| Exemple SVG              | Source API                                    |
|--------------------------|-----------------------------------------------|
| mon-blog.fr              | GET /managedCMS/resource/{sn} → displayName   |
| WordPress 6.4.2          | GET /managedCMS/resource/{sn} → wordpressVersion |
| Pro                      | GET /managedCMS/resource/{sn} → offer         |
| 245 Mo                   | GET /managedCMS/resource/{sn} → diskUsage     |

================================================================================

STRUCTURE MODAL
1. Header: Titre + bouton fermer
2. Bannière error: Avertissement action irréversible
3. Site concerné: Nom + version + offre + espace
4. Liste conséquences: Fichiers, BDD, sauvegardes, SSL/CDN
5. Champ confirmation: Taper "SUPPRIMER"
6. Footer: Annuler | Supprimer (danger)

================================================================================

INTERACTIONS
- Clic "×" → Ferme modal
- Clic "Annuler" → Ferme modal
- Saisie confirmation → Vérifie "SUPPRIMER"
- Bouton "Supprimer" désactivé tant que confirmation incorrecte
- Clic "Supprimer" → DELETE /managedCMS/resource/{sn}

================================================================================

API ENDPOINTS
DELETE /managedCMS/resource/{serviceName}
  Response: { taskId, status: "deleting" }
