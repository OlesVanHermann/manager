# STRUCTURE PAGE
1. NAV1 - Header OVH (Web Cloud actif, x=510)
2. NAV2 - Web Cloud (VoIP actif, x=500)
3. LEFT PANEL - Liste des lignes du groupe
4. RIGHT PANEL
   - Header: Numéro + description + modèle téléphone
   - NAV3: 7 tabs (Général, Téléphone, **Appels**, Transferts, Messagerie, Consommation, Options)
   - NAV4: 3 sous-tabs (Historique, **Filtrage entrant**, Filtrage sortant)
   - Bloc: Configuration du filtrage entrant (mode, options)
   - Bloc: Ajouter un filtre (input + select + bouton)
   - Table: Règles de filtrage (numéro, type, date, statut, actions)
   - Bannière info: Explication joker *

# INTERACTIONS
- Clic NAV3 tab → Navigation vers autre section ligne
- Clic NAV4 sous-tab → Bascule Historique/Filtrage entrant/Filtrage sortant
- Select mode filtrage → Change entre liste noire/blanche/désactivé
- Checkbox → Active/désactive option (anonymes, ne pas déranger)
- Input numéro + Ajouter → Crée nouvelle règle
- ✎ dans table → Modal édition règle
- × dans table → Supprime règle (confirmation)
- Lien "Appliquer à plusieurs lignes" → Modal sélection lignes

# API ENDPOINTS
GET    /telephony/{ba}/line/{sn}/options                    # Options ligne (dont screening)
PUT    /telephony/{ba}/line/{sn}/options                    # Modifier options
GET    /telephony/{ba}/screen/{sn}/screenLists              # Liste règles filtrage
POST   /telephony/{ba}/screen/{sn}/screenLists              # Ajouter règle
PUT    /telephony/{ba}/screen/{sn}/screenLists/{id}         # Modifier règle
DELETE /telephony/{ba}/screen/{sn}/screenLists/{id}         # Supprimer règle
