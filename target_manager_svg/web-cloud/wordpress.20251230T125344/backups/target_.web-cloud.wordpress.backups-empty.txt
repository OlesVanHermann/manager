DESCRIPTION
Page des sauvegardes en état vide, avant la première sauvegarde automatique.
Affiche un message d'onboarding avec bouton pour créer une sauvegarde manuelle.

| mon-blog.fr              | GET /managedCMS/resource/{sn} → displayName   |
| Pro                      | GET /managedCMS/resource/{sn} → offer         |
| 14 jours                 | GET /managedCMS/resource/{sn} → backupRetention |

================================================================================

STRUCTURE PAGE
1. NAV1 - Header OVH (Web Cloud actif)
2. NAV2 - Web Cloud (Wordpress actif)
3. LEFT PANEL - Liste services (1 site)
4. RIGHT PANEL
   - Header: Titre site
   - NAV3: 6 tabs (Sauvegardes actif)
   - Empty state: Illustration + texte + bouton CTA
   - Bannière info: Politique de sauvegarde de l'offre

================================================================================

INTERACTIONS
- Clic tab → Navigation
- Clic "Créer une sauvegarde" → Ouvre modal-backup-create

================================================================================

API ENDPOINTS
GET /managedCMS/resource/{serviceName}/backups
  Response: [] (liste vide)

POST /managedCMS/resource/{serviceName}/backups
  Response: { taskId, status: "pending" }
