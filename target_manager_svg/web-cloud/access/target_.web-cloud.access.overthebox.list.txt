# Cahier des Charges : web-cloud/access/overthebox - Liste

## 1. VUE D'ENSEMBLE
- **Écran** : Liste des services OverTheBox
- **Objectif** : Afficher tous les services OTB du compte avec possibilité de commander
- **Contexte** : Page d'entrée du module OverTheBox

## 2. LAYOUT & STRUCTURE
- Header OVHcloud avec menu Telecom
- Titre "OverTheBox" + bouton Changelog
- Datagrid avec liste des services

## 3. COMPOSANTS UI
- **OdsButton** : "Commander" (secondaire)
- **OdsButton** : "Changelog" (secondaire, aligné droite)
- **OdsDatagrid** : Liste des services
- **OdsBadge** : Statut (Actif, Création)
- **OdsActionMenu** : Menu contextuel (⋮)

## 4. DONNÉES & COLONNES
| Colonne | Type | Description |
|---------|------|-------------|
| Nom | link | Identifiant service (otb-xxx), cliquable |
| Description | text | Nom personnalisé ou "(non défini)" |
| Statut | badge | Actif (vert), Création (jaune) |
| Actions | menu | Menu contextuel |

## 5. ACTIONS & INTERACTIONS
- Bouton "Commander" → overthebox.order
- Clic sur nom → overthebox (dashboard)
- Menu ⋮ → Actions contextuelles
- Pagination en bas

## 6. APIS & ENDPOINTS
- GET /overthebox : Liste des services
- Réponse : Array<{serviceName, description, status}>

## 7. ÉTATS
- Liste vide → Afficher onboarding
- Liste avec données → Afficher datagrid
- Statuts : active, creating, suspended
