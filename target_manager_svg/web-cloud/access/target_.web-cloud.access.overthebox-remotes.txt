# STRUCTURE PAGE

1. NAV1 - Header OVH (Web Cloud actif)
2. NAV2 - Section (Accès actif)
3. LEFT PANEL - Liste OverTheBox
4. RIGHT PANEL
   - Header: Nom OTB
   - NAV3: 6 tabs (Remotes actif)
   - Contenu:
     - BANNER INFO: Explication accès distant
     - TOOLBAR: Bouton + Ajouter un remote
     - ACCÈS DISTANTS (table):
       - Nom
       - Type (SSH/HTTP/RDP/RTSP/Custom)
       - IP locale
       - Port local
       - URL d'accès public
       - Statut
       - Actions
     - DOCUMENTATION: Lien guide

# TYPES REMOTE
| Type | Port défaut | Usage |
|------|-------------|-------|
| SSH | 22 | Terminal distant |
| HTTP/HTTPS | 80/443 | Interface web |
| RDP | 3389 | Bureau Windows |
| VNC | 5900 | Bureau Linux/Mac |
| RTSP | 554 | Flux vidéo |
| Custom | Variable | Personnalisé |

# FONCTIONNEMENT
1. L'OverTheBox crée un tunnel chiffré vers le service OVH
2. Une URL publique unique est générée
3. L'accès se fait via cette URL sans ouvrir de port
4. Authentification requise (OVH SSO)

# INTERACTIONS
- + Ajouter un remote → Modal création
- URL d'accès → Clic = copie, ouvre dans nouveau tab
- Modifier → Modal édition
- Supprimer → Confirmation + DELETE
- Documentation → Guide externe

# API ENDPOINTS
GET  /overthebox/{id}/remotes           - Liste remotes
POST /overthebox/{id}/remotes           - Créer remote
PUT  /overthebox/{id}/remotes/{remId}   - Modifier remote
DELETE /overthebox/{id}/remotes/{remId} - Supprimer remote
