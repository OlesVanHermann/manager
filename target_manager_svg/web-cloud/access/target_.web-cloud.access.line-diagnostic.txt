# STRUCTURE PAGE

1. NAV1 - Header OVH (Web Cloud actif)
2. NAV2 - Section (Accès actif)
3. LEFT PANEL - Liste connexions
4. RIGHT PANEL
   - Header: Nom connexion
   - NAV3: 7 tabs (Ligne actif)
   - NAV4: 4 sous-tabs (Diagnostic actif)
   - Contenu:
     - Header avec date dernier diag + bouton nouveau
     - 6 Status Cards (2 rows × 3):
       - Incident OVH (vert/rouge)
       - Statut connexion (vert/rouge)
       - Passerelle LNS (vert/rouge)
       - Ping (vert/rouge)
       - Synchronisation (vert/rouge)
       - Qualité ligne (vert/jaune/rouge)
     - Bloc DÉTAILS LIGNE (profil, débits, latence, perte)
     - Bloc ASSISTANT RÉSOLUTION (radio buttons + CTA)
     - Lien historique

# CODES COULEUR STATUS CARDS
- Vert (#DEF7EC): Test OK
- Jaune (#FEF3C7): Warning, dégradation
- Rouge (#FEE2E2): Erreur, problème

# INTERACTIONS
- Clic "Nouveau diagnostic" → POST /line/diagnostic
- Clic radio → Sélectionne problème
- Clic "Lancer assistant" → Démarre wizard
- Clic "Voir historique" → Liste diagnostics passés

# API ENDPOINTS
GET  /access/connections/{id}/line/diagnostic     - Dernier diagnostic
POST /access/connections/{id}/line/diagnostic     - Lancer nouveau
GET  /access/connections/{id}/line/diagnostic/history - Historique
