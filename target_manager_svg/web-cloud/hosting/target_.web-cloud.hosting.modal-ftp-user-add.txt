STRUCTURE MODAL
1. Header avec titre et bouton fermer
2. Banniere info
3. Formulaire creation utilisateur FTP
   - Nom utilisateur (avec prefixe)
   - Mot de passe + confirmation
   - Repertoire racine
   - Option SSH
   - Quota disque
4. Footer avec boutons

CHAMPS FORMULAIRE
| Champ              | Type        | Validation                    |
|--------------------|-------------|-------------------------------|
| Nom utilisateur    | Input       | Requis, prefixe + alphanum    |
| Mot de passe       | Password    | Min 8, complexite requise     |
| Confirmation       | Password    | Doit correspondre             |
| Repertoire racine  | Select/Input| Chemin valide                 |
| SSH                | Toggle      | Optionnel (si offre compatible)|
| Quota              | Select      | Optionnel                     |

OPTIONS QUOTA
| Option       | Description                       |
|--------------|-----------------------------------|
| Illimite     | Pas de limite (defaut)            |
| 1 Go         | 1 Go maximum                      |
| 5 Go         | 5 Go maximum                      |
| 10 Go        | 10 Go maximum                     |
| Personnalise | Valeur personnalisee              |

INTERACTIONS
- Clic &#128065; → Afficher/masquer mot de passe
- Clic &#128193; → Parcourir repertoires
- Clic toggle SSH → Activer/desactiver SSH
- Clic Annuler → Fermer sans action
- Clic Creer → Creer l'utilisateur FTP

VALIDATIONS
- Nom utilisateur unique
- Mot de passe complexe
- Repertoire existe
- Quota utilisateurs non atteint

API ENDPOINTS
GET /hosting/web/{serviceName}                     → Info prefixe
GET /hosting/web/{sn}/user                         → Liste users existants
POST /hosting/web/{sn}/user                        → Creer utilisateur
GET /hosting/web/{sn}/attachedDomain               → Repertoires dispo
