STRUCTURE MODAL
1. Header orange (warning)
2. Banniere info explicative
3. Attachement actuel
4. Liste consequences (positives)
5. Checkbox confirmation
6. Footer avec boutons

DETACHEMENT vs RESILIATION
| Action       | Effet                           |
|--------------|---------------------------------|
| Detacher     | Dissocie, service reste actif   |
| Resilier     | Supprime completement           |

CONSEQUENCES DETACHEMENT
| Type     | Description                      |
|----------|----------------------------------|
| Positif  | Emails conserves                 |
| Positif  | Service toujours actif           |
| Positif  | Gestion independante possible    |
| Attention| DNS a mettre a jour              |

APRES DETACHEMENT
- Service visible dans Emails (manager)
- Facturation separee
- Peut etre rattache ailleurs
- DNS MX a verifier

BOUTONS FOOTER
| Bouton   | Action                     | Etat            |
|----------|----------------------------|-----------------|
| Annuler  | Fermer sans detacher       | Toujours actif  |
| Detacher | POST detach                | Si checkbox     |

INTERACTIONS
- Checkbox → Active bouton Detacher
- Clic Detacher → POST detach

API ENDPOINTS
GET /hosting/web/{sn}/email                       → Info service
POST /hosting/web/{sn}/email/detach               → Detacher
