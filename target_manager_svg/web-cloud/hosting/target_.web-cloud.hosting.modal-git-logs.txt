| Commit hash                  | deployment → commitHash                       |
| Message commit               | deployment → commitMessage                    |

STRUCTURE MODAL
1. Header avec titre
2. Barre filtre (recherche, statut, refresh)
3. Table deploiements
4. Zone logs selectionne
5. Pagination
6. Footer

COLONNES TABLE
| Colonne  | Description                        |
|----------|------------------------------------|
| Date     | Date et heure du deploiement       |
| Commit   | Hash court du commit (7 chars)     |
| Message  | Message du commit                  |
| Duree    | Temps d'execution                  |
| Statut   | Succes/Echec/En cours              |
| Logs     | Lien vers logs detailles           |

STATUTS DEPLOIEMENT
| Statut    | Couleur | Badge      |
|-----------|---------|------------|
| success   | Vert    | Succes     |
| error     | Rouge   | Echec      |
| running   | Jaune   | En cours   |

ZONE LOGS
- Terminal style (fond sombre)
- Timestamps pour chaque etape
- Code couleur (vert succes, rouge erreur)
- Duree totale en fin

INTERACTIONS
- Clic refresh → Actualiser liste
- Clic filtre statut → Filtrer
- Clic ligne → Afficher logs dans zone inferieure
- Clic icone logs → Telecharger logs complets
- Pagination → Naviguer

API ENDPOINTS
GET /hosting/web/{sn}/attachedDomain/{d}/git/deployments           → Liste
GET /hosting/web/{sn}/attachedDomain/{d}/git/deployments/{id}      → Details
GET /hosting/web/{sn}/attachedDomain/{d}/git/deployments/{id}/logs → Logs
