STRUCTURE MODAL
1. Header avec titre et bouton fermer
2. Banniere info
3. Formulaire creation variable
   - Cle (nom)
   - Type (string/password/boolean)
   - Valeur
4. Bloc variables reservees
5. Footer avec boutons

CHAMPS FORMULAIRE
| Champ  | Type        | Validation                    |
|--------|-------------|-------------------------------|
| Cle    | Input       | Requis, format UPPER_SNAKE    |
| Type   | Radio cards | string, password, boolean     |
| Valeur | Textarea    | Requis                        |

TYPES DE VARIABLES
| Type     | Description                       |
|----------|-----------------------------------|
| string   | Texte libre, visible              |
| password | Valeur masquee (***) dans l'UI    |
| boolean  | Valeur true ou false uniquement   |

VARIABLES RESERVEES
- PATH, HOME, USER, SHELL, PWD, TERM
- PHP_*, NODE_*, PYTHON_*, HTTP_*, SERVER_*
- OVH_*, HOSTING_*

INTERACTIONS
- Clic carte type → Selection du type
- Clic Annuler → Fermer sans action
- Clic Ajouter → Creer la variable

VALIDATIONS
- Cle unique (non existante)
- Cle non reservee
- Format cle valide (UPPER_SNAKE_CASE)
- Valeur non vide

API ENDPOINTS
GET /hosting/web/{sn}/envVar                       → Variables existantes
POST /hosting/web/{sn}/envVar                      → Creer variable
GET /hosting/web/{sn}/envVar/{key}                 → Verifier existence
