# STRUCTURE PAGE
1. NAV1 - Header OVH (Web Cloud actif)
2. NAV2 - Hébergement actif
3. LEFT PANEL - Liste hébergements
4. RIGHT PANEL
   - Header: Nom service
   - NAV3: 16 tabs (Boost actif)
   - TITRE
   - BANDEAU INFO: Disponibilité offres Performance
   - BLOC STATUT: État actuel du boost
   - TABLE HISTORIQUE: Historique des activations

# BANDEAU INFO
- Message: Boost réservé aux offres Performance
- Bouton: "Changer d'offre" → Page migration

# BLOC STATUT
| Champ            | Contenu                           |
|------------------|-----------------------------------|
| Statut           | Badge Activé/Désactivé            |
| Offre actuelle   | Nom de l'offre (Pro 2024, etc.)   |
| Boost disponible | Oui/Non selon éligibilité         |

# TABLE HISTORIQUE
| Colonne     | Largeur | Contenu                          |
|-------------|---------|----------------------------------|
| Compte      | 230px   | Nom de l'hébergement             |
| Offre       | 200px   | Offre au moment de l'action      |
| Offre Boost | 250px   | Badge Performance 1/2/Désactivé  |
| Date        | 200px   | Date/heure de l'action           |

# ÉTATS BOOST
| État           | Badge couleur |
|----------------|---------------|
| Désactivé      | Gris #F3F4F6  |
| Performance 1  | Vert #DEF7EC  |
| Performance 2  | Vert #DEF7EC  |

# INTERACTIONS
- Clic "Changer d'offre" → Page migration/upgrade
- Clic "Activer le boost" → Modal activation (si éligible)

# API ENDPOINTS
GET /hosting/web/{sn}/boost              - Statut boost actuel
GET /hosting/web/{sn}/boost/history      - Historique des boosts
POST /hosting/web/{sn}/boost             - Activer boost
DELETE /hosting/web/{sn}/boost           - Désactiver boost
