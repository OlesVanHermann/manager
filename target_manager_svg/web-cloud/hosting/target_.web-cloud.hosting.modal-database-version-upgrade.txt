STRUCTURE MODAL
1. Header avec titre
2. Warning migration
3. Affichage version actuelle avec statut
4. Selection nouvelle version (cartes)
5. Resultats verification compatibilite
6. Info processus de migration
7. Checkboxes confirmation (2)
8. Footer avec boutons

VERSION ACTUELLE
| Element    | Description                        |
|------------|------------------------------------|
| Version    | Numero de version complete         |
| Badge      | Support actif/etendu/EOL           |
| Fin support| Date fin de support                |

VERSIONS DISPONIBLES
| Version | Badge       | Description             |
|---------|-------------|-------------------------|
| 8.0     | Recommande  | Support actif long terme|
| 8.4     | Derniere    | Nouvelles fonctionnalites|

VERIFICATION COMPATIBILITE
| Element                | Resultat           |
|------------------------|--------------------|
| Types colonnes         | Compatible (vert)  |
| Fonctions SQL          | Compatible (vert)  |
| Taille base            | Compatible (vert)  |
| Mode SQL strict        | A verifier (orange)|

PROCESSUS MIGRATION
1. Sauvegarde automatique
2. Migration (quelques minutes)
3. Verification et validation

CONFIRMATIONS REQUISES
- Sauvegarde recente existante
- Acceptation interruption service

INTERACTIONS
- Clic version → Selectionner
- Checkboxes → Activer bouton Migrer
- Clic Migrer → Lancer migration

API ENDPOINTS
GET /hosting/web/{sn}/database/{name}                  → Version actuelle
GET /hosting/web/{sn}/database/availableVersions       → Versions dispo
POST /hosting/web/{sn}/database/{name}/checkCompatibility → Verifier
POST /hosting/web/{sn}/database/{name}/changeVersion   → Migrer
