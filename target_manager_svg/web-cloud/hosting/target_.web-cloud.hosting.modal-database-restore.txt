STRUCTURE MODAL
1. Header avec titre
2. Warning banniere
3. Base de donnees (lecture seule)
4. Dropdown selection dump
5. Liste dumps disponibles (radio)
6. Checkbox confirmation obligatoire
7. Footer avec boutons (Restaurer desactive sans confirmation)

VALIDATION
- L'utilisateur doit cocher la case pour activer le bouton Restaurer
- Selection d'un dump obligatoire

INTERACTIONS
- Clic radio dump → Selectionner dump a restaurer
- Checkbox confirmation → Active/desactive bouton
- Clic Restaurer → Lancer la restauration

API ENDPOINTS
GET /hosting/web/{sn}/database/{name}             → Details BDD
GET /hosting/web/{sn}/database/{name}/dump        → Liste dumps
POST /hosting/web/{sn}/database/{name}/restore    → Restaurer dump
