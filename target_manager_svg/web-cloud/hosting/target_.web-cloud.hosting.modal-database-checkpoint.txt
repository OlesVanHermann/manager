STRUCTURE MODAL
1. Header avec titre
2. Info base de donnees
3. Champ nom du point
4. Champ description (optionnel)
5. Liste points existants avec suppression
6. Banniere info retention
7. Footer avec boutons

CHAMPS
| Champ       | Type     | Required | Description              |
|-------------|----------|----------|--------------------------|
| Nom         | Input    | Oui      | Identifiant unique       |
| Description | Textarea | Non      | Note explicative         |

LISTE CHECKPOINTS
| Colonne      | Description                    |
|--------------|--------------------------------|
| Nom          | Identifiant du point           |
| Date         | Date de creation               |
| Taille       | Volume de la sauvegarde        |
| Action       | Bouton supprimer               |

LIMITES
| Offre       | Max checkpoints |
|-------------|-----------------|
| Start       | 2               |
| Performance | 5               |
| Pro         | 10              |

RETENTION
- 30 jours maximum
- Suppression automatique anciens

BOUTONS FOOTER
| Bouton   | Action                        |
|----------|-------------------------------|
| Annuler  | Fermer sans creer             |
| Creer    | POST checkpoint               |

INTERACTIONS
- Clic Supprimer → DELETE checkpoint
- Clic Creer → POST nouveau checkpoint

API ENDPOINTS
GET /hosting/web/{sn}/database/{name}               → Info base
GET /hosting/web/{sn}/database/{name}/checkpoint    → Liste points
POST /hosting/web/{sn}/database/{name}/checkpoint   → Creer point
DELETE /hosting/web/{sn}/database/{name}/checkpoint/{id} → Supprimer
