STRUCTURE MODAL
1. Header avec titre
2. Banniere statut snapshot
3. Informations detaillees
4. Contenu du snapshot (fichiers, BDD, emails)
5. Options de restauration (3 cartes)
6. Note de precaution
7. Footer avec boutons

INFORMATIONS SNAPSHOT
| Champ          | Description                         |
|----------------|-------------------------------------|
| Type           | Automatique quotidien / Manuel      |
| Statut         | Disponible / En cours / Expire      |
| Date creation  | Date et heure de creation           |
| Age            | Temps ecoule depuis creation        |
| Taille fichiers| Espace occupe par les fichiers      |
| Taille BDD     | Espace occupe par la base           |
| Expiration     | Date de suppression automatique     |

CONTENU SNAPSHOT
- Fichiers FTP (checkmark si inclus + taille + nombre)
- Base de donnees (checkmark si inclus + taille + nom)
- Emails (generalement non inclus)

OPTIONS RESTAURATION
| Option              | Description                        |
|---------------------|-------------------------------------|
| Fichiers uniquement | Restaure espace FTP seulement       |
| BDD uniquement      | Restaure base de donnees seulement  |
| Restauration complete| Restaure fichiers + BDD           |

INTERACTIONS
- Clic carte option → Selectionner type restauration
- Clic Restaurer → Lancer restauration
- Clic Fermer → Fermer modal

API ENDPOINTS
GET /hosting/web/{sn}/dump/{id}                        → Details snapshot
POST /hosting/web/{sn}/dump/{id}/restore               → Restaurer
