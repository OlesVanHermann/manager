STRUCTURE MODAL
1. Header avec titre
2. Banniere info (URL acces logs)
3. Champ identifiant
4. Champ mot de passe + icones
5. Champ confirmation mot de passe
6. Toggle ownLogsOnly
7. Footer avec boutons

CHAMPS
| Champ           | Type     | Validation                    |
|-----------------|----------|-------------------------------|
| Identifiant     | Text     | 3-32 chars, alphanum + tiret  |
| Mot de passe    | Password | Min 8, complexite requise     |
| Confirmation    | Password | Doit correspondre             |
| ownLogsOnly     | Toggle   | Optionnel                     |

INTERACTIONS
- Saisie identifiant → Validation format
- Clic oeil → Afficher/masquer mot de passe
- Clic generateur → Generer mot de passe aleatoire
- Toggle ownLogsOnly → Restreindre acces
- Clic Creer → Creer l'utilisateur

API ENDPOINTS
GET /hosting/web/{sn}/userLogs                    → Liste utilisateurs logs
POST /hosting/web/{sn}/userLogs                   → Creer utilisateur logs
