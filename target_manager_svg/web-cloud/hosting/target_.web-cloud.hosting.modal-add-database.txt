| Prefixe BDD                  | GET /hosting/web/{sn} → dbPrefix              |

================================================================================

STRUCTURE MODAL
1. Header avec titre et bouton fermer
2. Banniere info (temps de creation)
3. Formulaire creation BDD
   - Selection type (MySQL, PostgreSQL, MariaDB)
   - Nom de la base
   - Version
   - Nom utilisateur
   - Mot de passe + confirmation
4. Footer avec boutons Annuler/Creer

CHAMPS FORMULAIRE
| Champ              | Type        | Validation                    |
|--------------------|-------------|-------------------------------|
| Type BDD           | Radio cards | Requis                        |
| Nom BDD            | Input       | Requis, prefixe + max 16 char |
| Version            | Select      | Requis                        |
| Nom utilisateur    | Input       | Requis, prefixe + alphanum    |
| Mot de passe       | Password    | Min 8, maj, chiffre, special  |
| Confirmation       | Password    | Doit correspondre             |

TYPES DISPONIBLES
| Type       | Versions                          |
|------------|-----------------------------------|
| MySQL      | 5.7, 8.0                          |
| PostgreSQL | 13, 14, 15                        |
| MariaDB    | 10.6, 10.11                       |

INTERACTIONS
- Clic type → Selection du type de BDD
- Clic &#128065; → Afficher/masquer mot de passe
- Clic Annuler → Fermer modal sans action
- Clic Creer → Valider et creer la BDD
- Clic &#10005; → Fermer modal

VALIDATIONS
- Nom BDD unique
- Mot de passe complexe
- Quota BDD non atteint

API ENDPOINTS
GET /hosting/web/{sn}/databaseAvailableType        → Types disponibles
GET /hosting/web/{sn}/databaseAvailableVersion     → Versions
POST /hosting/web/{sn}/database                    → Creer BDD
GET /hosting/web/{sn}/database                     → Verifier quota
