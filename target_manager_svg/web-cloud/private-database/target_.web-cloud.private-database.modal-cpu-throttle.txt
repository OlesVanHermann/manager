# Cahier des Charges : web-cloud/private-database - Modal Saturation CPU

## 1. VUE D'ENSEMBLE
- **Modal** : Détails des événements de saturation CPU
- **Type** : Modal informative avec tableau
- **Objectif** : Afficher l'historique des saturations CPU

## 2. STRUCTURE MODAL
- Header avec titre et bouton fermer (×)
- Description et seuil critique
- Tableau des événements
- Bandeau warning recommandation
- Bouton Fermer

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur (700x580)
- **OdsText** : Description
- **OdsTable** : Événements CPU
- **OdsBadge** : "En cours" (orange)
- **OdsMessage** : Warning orange
- **OdsButton** : Fermer

## 4. TITRE
"Saturation CPU"

## 5. DESCRIPTION
- "Événements de saturation CPU sur les 7 derniers jours."
- "Seuil critique : 50% du temps en saturation."

## 6. COLONNES DU TABLEAU
| Colonne | Description | Exemple |
|---------|-------------|---------|
| Début de l'événement | Date/heure début | 22 déc. 2024 14:30:00 |
| Fin de l'événement | Date/heure fin ou badge | 22 déc. 2024 14:45:00 |
| Durée | Temps total | 15 min |

## 7. BADGE EN COURS
Badge orange (#FEF3C7) "En cours" pour événement actif

## 8. BANDEAU WARNING
Fond orange (#FFFBEB) avec bordure (#F59E0B) :
- "⚠ Votre serveur passe beaucoup de temps en saturation CPU."
- "Envisagez d'optimiser vos requêtes ou d'augmenter les ressources."

## 9. BOUTONS
| Bouton | Style | Action |
|--------|-------|--------|
| Fermer | Outline | Ferme la modal |

## 10. APIS & ENDPOINTS
- GET /hosting/privateDatabase/{serviceName}/cpuThrottle
  - Retourne: events[] (startDate, endDate, duration)
