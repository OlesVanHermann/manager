# Cahier des Charges : network/cloud-connect - Modal diagnostic BGP Peering

## 1. VUE D'ENSEMBLE
- **Type** : Modal de diagnostic
- **Contexte** : Page diagnostics Cloud Connect
- **Objectif** : Lancer un diagnostic du peering BGP

## 2. DIMENSIONS
| Propriété | Valeur |
|-----------|--------|
| Modal largeur | 500px |
| Modal hauteur | 340px |

## 3. COMPOSANTS UI
- **OdsModal** : Container principal
- **OdsText** : Description
- **OdsRadio** : Options de diagnostic
- **OdsButton** : Annuler, Lancer le diagnostic

## 4. HEADER MODAL
| Propriété | Valeur |
|-----------|--------|
| Titre | "Diagnostic BGP Peering" |
| Fond | Bleu (#0050D7) |
| Couleur texte | Blanc |
| Bouton fermer | × |

## 5. CONTENU

### Description
| Texte |
|-------|
| "Sélectionnez le type de diagnostic à exécuter :" |

### Options radio
| Option | Description | État |
|--------|-------------|------|
| Diagnostic par défaut | Vérification standard de l'état du peering BGP | Sélectionné |
| Routes reçues | Analyse des routes BGP reçues depuis le peer | - |
| Routes annoncées | Analyse des routes BGP annoncées vers le peer | - |

## 6. FOOTER MODAL
| Bouton | Style | Action |
|--------|-------|--------|
| Annuler | Secondary | Ferme modal |
| Lancer le diagnostic | Primary | Lance le diagnostic sélectionné |

## 7. APIS & ENDPOINTS
- POST /ovhCloudConnect/{serviceName}/diagnostic/bgp - Lancer diagnostic BGP
