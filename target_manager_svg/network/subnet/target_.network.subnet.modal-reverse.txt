# Cahier des Charges : network/ip - Modal Modifier Reverse DNS

## 1. VUE D'ENSEMBLE
- **Type** : Modal de modification
- **Déclencheur** : Action "Modifier le reverse DNS" dans le menu IP
- **Objectif** : Modifier l'enregistrement PTR de l'IP

## 2. STRUCTURE MODAL
- Overlay sombre (50% opacité)
- Modal centrée (600x400px)
- Header avec titre et bouton fermer
- Message d'information
- Formulaire
- Footer avec boutons action

## 3. COMPOSANTS UI
- **OdsModal** : Conteneur modal
- **OdsMessage** : Information reverse DNS (info)
- **OdsInput** : Adresse IP (readonly), Reverse DNS (editable)
- **OdsText** : Labels et aide
- **OdsButton** : Annuler/Confirmer

## 4. HEADER MODAL
| Élément | Valeur |
|---------|--------|
| Titre | "Modifier le reverse DNS" |
| Bouton fermer | ✕ (coin droit) |
| Séparateur | Ligne grise |

## 5. MESSAGE INFORMATION
| Propriété | Valeur |
|-----------|--------|
| Type | Info (bleu) |
| Fond | #E8F0FE |
| Bordure | #0050D7 |
| Icône | ℹ️ |
| Texte L1 | "Le reverse DNS permet d'associer un nom de domaine" |
| Texte L2 | "à votre adresse IP (enregistrement PTR)." |

## 6. CHAMPS DU FORMULAIRE

### Adresse IP (readonly)
| Propriété | Valeur |
|-----------|--------|
| Type | Input readonly |
| Fond | Gris (#F3F4F6) |
| Largeur | 540px |
| Exemple | 123.45.67.89 |

### Reverse DNS
| Propriété | Valeur |
|-----------|--------|
| Type | Input text |
| Largeur | 540px |
| Exemple | server1.example.com |
| Focus | Bordure bleue 2px |
| Aide | "Le nom de domaine doit pointer vers cette IP (enregistrement A ou AAAA)." |

## 7. FOOTER MODAL
| Bouton | Style | Position |
|--------|-------|----------|
| Annuler | Outline (bordure grise) | Gauche |
| Confirmer | Primary (bleu) | Droite |

## 8. VALIDATION
| Règle | Description |
|-------|-------------|
| Format FQDN | Nom de domaine valide |
| Résolution | Le domaine doit pointer vers l'IP |

## 9. APIS & ENDPOINTS
- GET /ip/{ip}/reverse - Reverse actuel
- PUT /ip/{ip}/reverse/{ip} - Modifier reverse
